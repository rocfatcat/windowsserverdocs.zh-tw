# 部署資料夾重新導向與離線檔案

>適用於： Windows 10，Windows 7，Windows 8、 Windows 8.1、 Windows Server 2008 R2、 Windows Server 2012、 Windows Server 2012 R2，Windows Server 2016 中，Windows Vista

本主題說明如何使用 Windows Server 資料夾重新導向與離線檔案部署到 Windows 用戶端電腦。

如需本主題最近的變更清單，請參閱[變更歷程記錄](#change-history)。

>[!IMPORTANT]
>安全性所做的變更[MS16 072](https://support.microsoft.com/en-us/help/3163622/ms16-072-security-update-for-group-policy-june-14-2016)中，因為我們更新[步驟 3： 建立適用於資料夾重新導向的 GPO](#step-3:-create-a-gpo-for-folder-redirection)本主題，Windows 可以正確地套用資料夾重新導向原則 （並不會還原受影響的電腦上重新導向的資料夾）。

## 必要條件

### 硬體需求

資料夾重新導向需要 x64 或 x86 為主的電腦。它不支援 Windows® RT

### 軟體需求

資料夾重新導向具有下列軟體需求：

- 若要管理資料夾重新導向，您必須為網域系統管理員安全性群組、 企業版系統管理員安全性群組，或群組原則建立者擁有者安全性群組成員登入。
- 用戶端電腦必須執行 Windows 10、 Windows 8.1、 Windows 8、 Windows 7，Windows Server 2012 R2、 Windows Server 2012、 Windows Server 2008 R2 或 Windows Server 2008。
- 用戶端電腦必須加入 Active Directory 網域服務 (AD DS) 所管理。
- 電腦必須是可用於群組原則管理及安裝 Active Directory 系統管理中心。
- 在檔案伺服器必須為可用來裝載重新導向的資料夾。
    - 如果檔案共用使用 DFS 命名空間，DFS 資料夾 （連結） 必須以防止使用者進行衝突的編輯不同伺服器上的單一目標。
    - 如果檔案共用使用 DFS 複寫來複寫內容與另一部伺服器，使用者必須能夠存取只在來源伺服器以防止使用者在不同的伺服器上進行衝突的編輯。
    - 當使用叢集的檔案共用，請停用若要避免使用資料夾重新導向與離線檔案的效能問題的檔案共用上的持續可用性。 此外，離線檔案可能未轉換成離線模式 3 到 6 分鐘之後使用者遺失持續可用的檔案共用，無法不便還不使用離線檔案的永遠離線瀏覽模式的使用者存取。

>[!NOTE]
>某些較新的功能，在資料夾重新導向有額外的用戶端電腦和 Active Directory 結構描述需求。 如需詳細資訊，請參閱[部署主要電腦](deploy-primary-computers.md)、[停用離線檔案的資料夾](disable-offline-files-on-folders.md)、[啟用永遠離線瀏覽模式](enable-always-offline.md)，並[啟用最佳化的資料夾移動](enable-optimized-moving.md)。

## 步驟 1： 建立資料夾重新導向安全性群組

如果您的環境不已經設定使用資料夾重新導向，第一個步驟是建立安全性群組，其中包含您想要套用資料夾重新導向原則設定要在其中的所有使用者。

以下是如何建立安全性群組，如資料夾重新導向：

1. 已安裝在電腦上開啟的伺服器管理員，Active Directory 系統管理中心。
2. 在 [**工具**\] 功能表，選取 [ **Active Directory 系統管理中心**。 \[Active Directory 系統管理中心\] 隨即顯示。
3. 適當的網域或 OU 上按一下滑鼠右鍵，選取 [**新增]**，然後選取**群組**。
4. 在 **\[建立群組\]** 視窗內的 **\[群組\]** 區段中，指定下列設定：
    - 在 [**群組名稱**輸入安全性群組的名稱，例如：**資料夾重新導向的使用者**。
    - 在 [**群組領域**中，選取**安全性**，然後選取 **[通用**。
5. 在 [**成員**] 區段中，選取 [**新增**。 \[選取使用者、連絡人、電腦、服務帳戶或群組\] 對話方塊隨即顯示。
6. 輸入使用者或您想以部署資料夾重新導向，選取 **[確定]**，然後再次選取 **[確定]** 群組的名稱。

## 步驟 2： 建立適用於重新導向的資料夾的檔案共用

如果您還沒有重新導向的資料夾的檔案共用，請使用下列程序來執行 Windows Server 2012 的伺服器上建立的檔案共用。

>[!NOTE]
>部分功能可能會不同，或如果您執行另一個版本的 Windows Server 的伺服器上建立檔案共用，而無法使用。

以下是如何在 Windows Server 2012 和 Windows Server 2016 上建立的檔案共用：

1. 在伺服器管理員瀏覽窗格中，選取 [**檔案和存放服務**]，然後選取 [**共用**，以顯示 \ [共用] 頁面。
2. 在**共用**磚中，選取 [**工作**，然後選取**新的共用**。 新的共用精靈會出現。
3. 在**選取設定檔**頁面上，選取 [ **SMB 共用 – 快速**。 如果您已安裝的檔案伺服器資源管理員，並使用資料夾管理內容，請改為選取**SMB 共用-進階**。
4. 在**共用的位置**] 頁面上，選取您要建立共用的伺服器與磁碟區。
5. **共用名稱**在頁面上，輸入**共用名稱**] 方塊中的共用 （例如，**使用者 $**） 的名稱。
    >[!TIP]
    >建立共用時，隱藏共用投放```$```共用名稱之後。 這將會隱藏隨性的瀏覽器中的共用。
6. 在**其他設定**頁面上，清除啟用持續可用性核取方塊，如果出現，表示，而您可以選擇性選取 [**啟用存取型列舉**和**加密的資料存取**的核取方塊。
7. 在**權限**頁面上，選取 [ **...自訂權限**。 進階安全性設定] 對話方塊隨即顯示。
8. 選取 [**停用繼承**，，然後選取 [**轉換繼承的權限到此物件上的明確權限**。
9. 為所述的表 1，並顯示在圖 1 中，移除權限適用於未列出的群組和帳戶，然後將特殊權限新增至您在步驟 1 中建立資料夾重新導向的使用者群組設定的權限。
    
    ![設定適用於資料夾重新導向的共用的權限](media/deploy-folder-redirection/setting-the-permissions-for-the-redirected-folders-share.png)
    
    **圖 1**設定適用於資料夾重新導向的共用的權限
10. 如果您選擇的**SMB 共用-進階**設定檔，在**管理屬性**頁面上，選取的**使用者檔案**資料夾使用量值。
11. 如果您選擇的**SMB 共用-進階**的設定檔，在**配額**頁面上，選擇性地選取的配額套用至共用的使用者。
12. 在**確認**頁面上，選取**建立。**

## 步驟 3： 建立適用於資料夾重新導向的 GPO

如果您還沒有建立資料夾重新導向設定的 GPO，請使用下列程序來建立一個。

以下說明如何建立資料夾重新導向的 GPO:

1. 已安裝在電腦上開啟的伺服器管理員，使用群組原則管理。
2. 從 [**工具**\] 功能表，選取 [**群組原則管理**。
3. 網域或您要設定資料夾重新導向的 OU 上按一下滑鼠右鍵，然後選取 [**建立這個網域中的 GPO 並連結到此處**。
4. 在 [**新增 GPO\** ] 對話方塊中，輸入 （例如，**資料夾重新導向設定**） 會將 GPO 的名稱，然後選取 **[確定]**。
5. 以滑鼠右鍵按一下新建立的 GPO，然後清除 [**啟用連結**的核取方塊。 如此可防止將 GPO 套用直到完成設定它。
6. 選取的 GPO。 在 [**範圍**] 索引標籤的 [**安全性篩選**] 區段中，選取 [ **Authenticated Users\**]，然後選取**移除**以防止將 GPO 套用到所有人。
7. 在 [**安全性篩選**」 區段中，選取 [**新增**。
8. 在 [**選取使用者、 電腦或群組**] 對話方塊中，輸入您在步驟 1 （例如，**資料夾重新導向的使用者**），建立安全性群組的名稱，然後選取 **[確定]**。
9. 選取**委派**] 索引標籤，選取 [**加入**、 輸入**已驗證的使用者**，選取 **[確定]**，，然後選取一次以接受預設的讀取權限的 **[確定]** 。
    
    這是必要步驟因為[MS16 072](https://support.microsoft.com/help/3163622/ms16-072-security-update-for-group-policy-june-14-2016)中所做的安全性變更。

>[!IMPORTANT]
>因為[MS16 072](https://support.microsoft.com/help/3163622/ms16-072-security-update-for-group-policy-june-14-2016)，您現在中所做的變更必須提供的安全性 Authenticated Users 群組 「 受委派的讀取權限給資料夾重新導向 GPO-否則您將不會取得將 GPO 套用到使用者，或如果您已經套用，做為 GPO 中移除時，重新導向回到在本機電腦的資料夾。 如需詳細資訊，請參閱[部署群組原則安全性更新 MS16-072](https://blogs.technet.microsoft.com/askds/2016/06/22/deploying-group-policy-security-update-ms16-072-kb3163622/)。

## 步驟 4： 設定資料夾重新導向與離線檔案

建立資料夾重新導向設定所需的 GPO 之後, 編輯群組原則設定，來啟用並設定資料夾重新導向，如下所述的下列程序。

>[!NOTE]
>離線檔案是預設為重新導向的資料夾，在 Windows 用戶端電腦上啟用和停用在執行 Windows Server 的電腦上，除非使用者變更。 若要使用群組原則控制是否啟用離線檔案，請使用**允許或禁止使用離線檔案功能**原則設定。
> 一些其他的離線檔案群組原則設定的相關資訊，請參閱[啟用進階離線檔案的功能](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-r2-and-2012/dn270369(v%3dws.11)>)，以及[設定離線檔案的群組原則](<https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2003/cc759721(v%3dws.10)>)。

以下是如何在群組原則中設定資料夾重新導向：

1. 在群組原則管理，您建立 （例如，**資料夾重新導向設定**），將 GPO 上按一下滑鼠右鍵，然後選取 [**編輯**。
2. 在群組原則管理編輯器 \] 視窗中，瀏覽到**使用者設定**，則**原則**，則**Windows 設定**，而且**資料夾重新導向**。
3. 以滑鼠右鍵按一下您想要重新導向 （例如，**文件**），然後選取 [**內容**的資料夾。
4. 在 [**屬性**] 對話方塊中，從 [**設定**] 方塊中，選取**基本-將重新導向每個人的資料夾的相同位置**。

    > [!NOTE]
    > 若要套用到執行 Windows XP 或 Windows Server 2003 的用戶端電腦的資料夾重新導向，選取 [**設定**] 索引標籤，然後選取**也重新導向將原則套用到 Windows 2000、 Windows 2000 Server、 Windows XP 和 Windows Server 2003 操作系統**核取方塊。
5. 在 [**目標資料夾位置**] 區段中，選取 [**建立每個使用者在根路徑的資料夾**，然後在**根路徑**] 方塊中，鍵入儲存重新導向的資料夾，例如檔案共用的路徑︰ **\\\fs1.corp.contoso.com\\使用者 $**
6. 選取 [**設定**] 索引標籤，然後在**原則移除**區段中，您可以選擇性選取**回本機使用者設定檔的位置原則遭到移除時將資料夾重新導向**（這項設定可協助使用者更有更多的行為的資料夾重新導向如預期般再適用於 adminisitrators 和使用者）。
7. 選取 **[確定]**，並再警告對話方塊中選取 **[是]** 。

## 步驟 5： 啟用資料夾重新導向的 GPO

一旦您已完成設定資料夾重新導向群組原則設定下, 一個步驟是啟用的 GPO 中，允許它可以被套用到受影響的使用者。

>[!TIP]
>如果您計劃實作主要電腦支援或其他原則設定，請立即執行，啟用 GPO 之前，先。 如此可防止使用者資料之前會啟用主要電腦支援複製到非主要的電腦。

以下是如何啟用做為資料夾重新導向的 GPO:

1. 開啟 [群組原則管理。
2. 您已建立，將 GPO 上按一下滑鼠右鍵，然後選取 [**啟用] 連結**。 核取方塊會出現的功能表項目旁邊。

## 步驟 6： 測試資料夾重新導向

若要測試資料夾重新導向，使用登入電腦設定資料夾重新導向的使用者帳戶。 然後請確認這些資料夾和設定檔會重新導向。

以下是如何測試資料夾重新導向：

1. 使用登入主要的電腦 （如果您啟用主要電腦支援） 已經啟用資料夾重新導向的使用者帳戶。
2. 如果使用者先前登入到電腦，開啟提升權限的命令提示字元中，然後輸入下列命令，以確保最新的群組原則設定會套用至用戶端電腦：
    
    ```PowerShell
    gpupdate /force
    ```
3. 開啟檔案總管 \]。
4. 重新導向的資料夾 （例如，[我的文件資料夾中的文件庫），以滑鼠右鍵按一下，然後選取 [**屬性**。
5. 選取 [**位置**] 索引標籤，並確認路徑會顯示您指定而不是本機路徑的檔案共用。
