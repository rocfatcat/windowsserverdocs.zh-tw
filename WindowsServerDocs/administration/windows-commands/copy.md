# <a name="copy"></a>copy



將一或多個檔案複製到另一個位置。

如需如何使用此命令的範例，請參閱[範例](#BKMK_examples)。

## <a name="syntax"></a>語法

```
copy [/d] [/v] [/n] [/y | /-y] [/z] [/a | /b] <Source> [/a | /b] [+<Source> [/a | /b] [+ ...]] [<Destination> [/a | /b]]
```

## <a name="parameters"></a>參數

|參數|描述|
|---------|-----------|
|/d|可讓加密的檔案複製到儲存為目的地的解密檔案。|
|/v|確認新的檔案會正確地寫入。|
|/n|如果有的話，將複製的檔案名稱超過八個字元，或副檔名的檔案超過三個字元時，使用短檔名。|
|/y|不要提示您確認您想要覆寫現有目的地檔案。|
|/-y|會提示您確認您想要覆寫現有目的地檔案。|
|/z|網路將檔案複製可重新啟動模式。|
|/a|表示 ASCII 文字檔。|
|/b|表示二進位檔。|
|\<來源 >|必要。 指定您要從中複製檔案或一組檔案的位置。 *來源*可包含磁碟機代號與冒號、 目錄名稱、 檔案名稱或這些項目的組合。|
|\<目的地 >|必要。 指定您要複製檔案或一組檔案的位置。 *目的地*可包含磁碟機代號與冒號、 目錄名稱、 檔案名稱或這些項目的組合。|
|/?|在命令提示字元顯示說明。|

## <a name="remarks"></a>備註

-   您可以複製 ASCII 文字檔，使用檔案結尾字元 (CTRL + Z)，以指出檔案的結尾。
-   使用  **/a**

    當 **/a**前面或後面清單的命令列上的檔案，它會套用至之前所列的所有檔案**複製**遇到 **/b**。 在此情況下， **/b**會套用到檔案前面**b**。

    效果 **/a**取決於在命令列字串中的位置。 當 **/a**遵循*來源*，**複製**視為 ASCII 檔案的檔案，並複製資料之前的第一個檔案結尾字元 (CTRL + Z)。

    當 **/a**遵循*目的地*，**複製**新增檔案結尾字元 (CTRL + Z) 做為檔案的最後一個字元。
-   使用  **/b**

    **b**指示命令直譯器，讀取目錄中的檔案大小所指定的位元組數目。 **/ b**是預設值**複製**，除非**複製**結合了檔案。

    當 **/b**前面或後面清單的命令列上的檔案，它適用於所有列出的檔案，直到**複製**遇到 **/a**。 在此情況下， **/a**會套用到檔案前面 **/a**。

    效果**b**取決於在命令列字串中的位置。 當 **/b**遵循*來源*，**複製**複製整個檔案，包括任何檔案結尾字元 (CTRL + Z)。

    當 **/b**遵循*目的地*，**複製**不會新增為檔案結尾字元 (CTRL + Z)。
-   使用 **/v**

    如果出現錯誤訊息無法驗證的寫入作業。 雖然很少發生錄製錯誤**複製**，您可以使用 **/v**若要確認已正確地寫入重要資料。 **/V**命令列選項也會減慢**複製**命令，因為必須檢查記錄在磁碟上每個磁區。
-   使用 **/y**和 **/-y**

    如果 **/y**會預設在 COPYCMD 環境變數中，您可以使用覆寫此設定 **/-y**在命令列。 根據預設，系統會提示您時您將此設定，除非**複製**批次指令碼中執行命令。
-   附加檔案

    若要附加的檔案，指定單一檔案，以*目的地*，但多個檔案*來源*(使用萬用字元或*File1* + *File2*+*File3*格式)。
-   使用 **/z**

    如果連線複製期間 （例如，如果離線的伺服器將會中斷連接），將會中斷**複製 /z**之後重新建立連接時繼續執行。 **/z**也會顯示完成每個檔案的複製作業的百分比。
-   複製與裝置

    您可以使用裝置名稱的一或多個相符項目*來源*或是*目的地*。
-   使用或省略**b**複製到裝置時

    當*目的地*是一種裝置 （例如 Com1 或 Lpt1） **b**將資料複製到二進位模式中的裝置。 在二進位模式中，**複製/b** （包括特殊字元，例如 CTRL + C 鍵，CTRL + S、 CTRL + Z、 ENTER） 的所有字元都複製到做為資料的裝置。 不過，如果您省略**b**，資料會複製到 ASCII 模式中的裝置。 在 ASCII 模式中，特殊字元可能會導致複製程序期間合併的檔案。
-   使用預設的目的地檔案

    如果您未指定目的地檔案，複製會建立具有相同名稱、 修改日期，和修改時間為原始的檔案。 新的複本會儲存在目前的磁碟機上目前的目錄。 如果原始程式檔是在目前的磁碟機上，並在目前的目錄，而且您未指定不同的磁碟機或目的地檔案的目錄**複製**命令會停止，並顯示下列錯誤訊息：

    `File cannot be copied onto itself`

    `0 File(s) copied`
-   合併檔案

    如果您指定多個檔案中的*來源*，**複製**將它們合併成單一檔案使用檔案名稱中指定所有*目的地*。 **複製**合併的檔案是 ASCII 檔案，除非您使用會假設 **/b**選項。
-   複製的零長度檔案

    **複製**不會複製檔案的位元組數為 0。 使用**xcopy**複製這些檔案。
-   變更檔案的日期與時間

    如果您想要指派為目前時間和日期到檔案，而不需要修改檔案，請使用下列語法：  
    ```
    copy /b <Source> +,,
    ```  
    逗號表示省略*目的地*參數。
-   在子目錄中複製檔案

    若要複製的所有目錄的檔案和子目錄，請使用**xcopy**命令。
-   **複製**命令，使用不同的參數，可從修復主控台。

## <a name="BKMK_examples"></a>範例

若要複製稱為 Memo.doc Letter.doc 到目前的磁碟機中的檔案，以及確認檔案結尾字元 (CTRL + Z) 會在複製的檔案結尾處，輸入：
```
copy memo.doc letter.doc /a
```
若要複製名 Robin.typ 為從目前的磁碟機和目錄中，名為鳥位於 C 磁碟機上的現有目錄的檔案，請輸入：
```
copy robin.typ c:\birds
```
如果鳥目錄不存在，檔案 Robin.typ 會複製到名為鳥位於 c 磁碟機的磁碟上的根目錄中的檔案

若要結合 Mar89.rpt、 Apr89.rpt 和 May89.rpt，其位於目前的目錄，並將它們放在名為 （也是在目前的目錄） 的報表，請輸入：
```
copy mar89.rpt + apr89.rpt + may89.rpt Report
```
當您合併檔案**複製**標示目的地檔案的目前日期和時間。 如果您省略*目的地*，檔案會進行結合並儲存在清單中的第一個檔案的名稱。 比方說，若要結合報表中的所有檔案，檔案，名為 「 報表已經存在時，請輸入：
```
copy report + mar89.rpt + apr89.rpt + may89.rpt
```
若要結合目前的目錄中副檔名 the.txt 到單一檔案的具名 Combined.doc 的所有檔案，請輸入：
```
copy *.txt Combined.doc 
```
如果您想要結合成一個檔案的多個二進位檔，使用萬用字元，包括**b**。 這可防止 Windows CTRL + Z 視為檔案結尾字元。 例如，輸入：
```
copy /b *.exe Combined.exe
```

> [!CAUTION]
> 如果您合併二進位檔案時，所產生的檔案可能無法使用，因為內部格式。

在下列範例中，**複製**結合每個已使用其對應的.ref 檔案副檔名為.txt 的檔案。 結果是相同的檔案名稱，但副檔名為.doc 的檔案。 **複製**結合使用 File1.ref File1.txt 表單 File1.doc，然後**複製**File2.txt 結合 File2.ref 表單 File2.doc，依此類推。 例如，輸入：
```
copy *.txt + *.ref *.doc
```
若要結合副檔名為.txt 的所有檔案，並再.ref 副檔名的所有檔案都都將併入一個名為 Combined.doc 檔案，請輸入：
```
copy *.txt + *.ref Combined.doc
```
