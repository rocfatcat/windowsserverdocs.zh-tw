# <a name="winnt32"></a>winnt32

>適用於：Windows Server (半年通道)、Windows Server 2016、Windows Server 2012 R2、Windows Server 2012

執行安裝或升級至 Windows Server 2003 中的產品。 您可以在執行 Windows 95、Windows 98、Windows Millennium edition、Windows NT、Windows 2000、Windows XP 或 Windows Server 2003 產品的電腦上，于命令提示字元中執行**winnt32.exe** 。 如果您在執行 Windows NT 4.0 版的電腦上執行**winnt32.exe** ，則必須先套用 Service Pack 5 或更新版本。
## <a name="syntax"></a>語法
```
winnt32 [/checkupgradeonly] [/cmd: <CommandLine>] [/cmdcons] [/copydir:{i386|ia64}\<FolderName>] [/copysource: <FolderName>] [/debug[<Level>]:[ <FileName>]] [/dudisable] [/duprepare: <pathName>] [/dushare: <pathName>] [/emsport:{com1|com2|usebiossettings|off}] [/emsbaudrate: <BaudRate>] [/m: <FolderName>]  [/makelocalsource] [/noreboot] [/s: <Sourcepath>] [/syspart: <DriveLetter>] [/tempdrive: <DriveLetter>] [/udf: <ID>[,<UDB_File>]] [/unattend[<Num>]:[ <AnswerFile>]]
```

## <a name="remarks"></a>備註
如果您要在用戶端電腦上部署 Windows XP，可以使用 Windows XP 隨附的 winnt32.exe 版本。 部署 Windows XP 的另一種方法是使用 winnt32.exe，它可以透過 Windows Installer，這是一組可執行檔技術。 如需用戶端部署的詳細資訊，請參閱 Windows Server 2003 部署套件，如[使用 Windows 部署和資源套件](https://technet.microsoft.com/library/cc779317(v=ws.10).aspx)中所述。

在 Itanium 型電腦上， **winnt32.exe**可以從可延伸韌體介面（EFI）或 windows Server 2003 Enterprise、windows Server 2003 R2 Enterprise、windows Server 2003 R2 Datacenter 或 windows Server 2003 datacenter 執行。 此外，在以 Itanium 架構為基礎的電腦上， **/cmdcons**和 **/syspart**無法使用，而且無法使用與升級相關的選項。
如需硬體相容性的詳細資訊，請參閱[硬體相容性](https://technet.microsoft.com/library/cc757927(v=ws.10).aspx)。
如需有關使用動態更新及安裝多個用戶端的詳細資訊，請參閱 Windows Server 2003 部署套件，如[使用 Windows 部署和資源套件](https://technet.microsoft.com/library/cc779317(v=ws.10).aspx)中所述。
如需修改開機設定的相關資訊，請參閱 windows Server 2003 的 Windows 部署與資源套件。 如需詳細資訊，請參閱[使用 Windows 部署和資源套件](https://technet.microsoft.com/library/cc779317(v=ws.10).aspx)。
使用 **/unattend**命令列選項，自動執行安裝 affirms，您已閱讀並接受 Windows Server 2003 的 Microsoft 授權合約。 在使用此命令列選項代替您自己的組織來安裝 Windows Server 2003 之前，您必須確認使用者（不論是個人或單一實體）是否已接收、閱讀及接受 Microsoft 授權條款該產品的合約。  Oem 不能在銷售給終端使用者的電腦上指定此金鑰。
