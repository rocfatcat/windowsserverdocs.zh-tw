# <a name="step-4-configure-group-policy-settings-for-automatic-updates"></a>步驟 4：設定自動更新的群組原則設定

>適用於：Windows Server （半年通道），Windows Server 2016 中，Windows Server 2012 R2 中，Windows Server 2012

在 active directory 環境中，您可以使用群組原則來定義電腦和使用者 （稱為本文件中做為 WSUS 用戶端） 與 Windows 更新，以取得自動更新從 Windows Server Update Services (WSUS) 可以互動方式。

本主題包含兩個主要區段：

[WSUS 用戶端更新的群組原則設定](4-configure-group-policy-settings-for-automatic-updates.md#BKMK_PolSettings)、 提供規範指引和行為的相關詳細資料的 Windows 更新和維護排程器設定群組原則 」 的 Windows Update 與 WSUS 用戶端可以互動方式控制若要取得自動更新。

[補充資訊](4-configure-group-policy-settings-for-automatic-updates.md#BKMK_Supplemental)具有下列各節：

-   [存取 Windows Update 設定，群組原則中的](4-configure-group-policy-settings-for-automatic-updates.md#BKMK_OpenGPO)，以提供一般指引是有關使用群組原則管理編輯器] 和 [存取的更新服務原則 」 擴充功能和中的維護排程器設定的相關資訊群組原則。

-   [變更 WSUS 與本指南為](4-configure-group-policy-settings-for-automatic-updates.md#BKMK_changes)： 熟悉 WSUS 3.2 和先前版本的系統管理員，本節會提供目前和過去版本的 WSUS 與本指南的主要差異的簡短摘要。

-   [詞彙和定義](4-configure-group-policy-settings-for-automatic-updates.md#BKMK_Terms)： 與 WSUS 和更新本指南中所使用的服務相關的各種詞彙的定義。

#### <a name="BKMK_comp4"></a>自動更新偵測頻率
指定 Windows 用來判斷檢查可用更新之前的等待時間，以小時為單位。 確實等待時間的決定方式，是使用這裡指定的時數減去所指定時數的 0 到 20%。 比方說，如果此原則用來指定 20 小時偵測頻率，要套用此原則的所有用戶端會檢查更新 16 個與 20 小時之間的任何位置。

|支援的作業：|排除：|
|---------|-------|
|Windows 作業系統，仍在其[Microsoft 產品支援週期](https://support.microsoft.com/gp/lifeselect)。|Windows RT|

>
> 如果已停用 「 設定自動更新 」 原則設定，此原則沒有任何作用。

> [!NOTE]
> 在 Windows RT 上不支援此原則 啟用此原則並不會有任何執行 Windows RT 的電腦上的作用

|||
|-|-|
|**原則設定狀態**|**行為**|
|**未設定**|指定 Windows 會檢查可用的更新，22 小時的預設間隔。|
|**已啟用**|指定 Windows 會檢查可用的更新，在指定的間隔。|
|**已停用**|指定 Windows 會檢查可用的更新，22 小時的預設間隔。|

**選項：** 如果啟用此設定時，您可以指定 Windows Update 檢查更新前所等待的時間間隔 （以小時為單位）。

### <a name="BKMK_OpenGPO"></a>存取 Windows Update 設定，在 群組原則
接下來的程序描述如何開啟您的網域控制站上 GPMC。 然後程序會說明如何進行編輯，開啟現有網域層級群組原則物件 (GPO)，或建立新的網域層級 GPO，並開啟它進行編輯。

> [!NOTE]
> 您必須是隸屬**Domain Admins**群組或同等權限才能執行此程序。

##### <a name="to-open-or-add-and-open-a-group-policy-object"></a>若要開啟或新增，然後開啟 群組原則物件

1.  網域控制站，請移至**伺服器管理員**，>**工具**，>**群組原則管理**。 群組原則管理主控台隨即開啟。

2.  在左窗格中，展開您的樹系。 例如，按兩下**樹系： example.com**。

3.  在左窗格中，按兩下**網域**，然後按兩下您要管理的群組原則物件的網域。 例如，按兩下**example.com**。

4.  執行下列其中一項：

    -  **若要開啟現有的網域層級 GPO 進行編輯**，按兩下包含您想要管理，以滑鼠右鍵按一下您想要管理，然後按一下 網域原則的群組原則物件的網域**編輯**。 此時會開啟 群組原則管理編輯器 」 (GPME)。

    -  **若要建立新的群組原則物件，並開啟進行編輯**:
        1.  以滑鼠右鍵按一下您要建立新的群組原則物件，並按一下 的網域**在這個網域中建立 GPO 並連結到**。

        2.  在 **新的 GPO**，請在**名稱**，輸入新群組原則物件的名稱，然後按一下**確定**。

        3.  新的群組原則物件，以滑鼠右鍵按一下，然後按一下**編輯**。 此時會開啟 GPME。

##### <a name="to-open-the-windows-update-or-maintenance-scheduler-extensions-of-group-policy"></a>若要開啟 群組原則的 Windows 更新或維護排程器擴充功能

1.  在 [群組原則管理編輯器] 中，執行下列其中一項動作：

    -   **開啟 電腦設定 > Windows Update 群組原則延伸**。 瀏覽至：*PolicyName* > **電腦組態** > **原則** / **系統管理範本** > **Windows 元件** > **Windows Update**。

    -   **開啟 [使用者設定] > 的群組原則的 Windows 更新延伸模組**。 瀏覽至：*PolicyName* > **使用者設定** > **原則** > **系統管理範本** >  **Windows 元件** > **Windows Update**。

    -   **開啟 [電腦設定] > 的群組原則的維護排程器延伸模組**。 在 GPOE，瀏覽至*PolicyName* > **電腦組態** > **原則** > **系統管理範本** > **Windows 元件** > **維護排程器**。

如需有關群組原則的詳細資訊，請參閱[群組原則概觀](https://technet.microsoft.com/library/hh831791.aspx(v=ws.12))。

> [!TIP]
> 開啟您想要的群組原則延伸模組之後，您可以使用下列步驟來啟用、 停用，或設定之間瀏覽：

##### <a name="to-configure-group-policy-settings"></a>設定群組原則設定

1.  在  *ExtensionOfGroupPolicy*，連按兩下您想要檢視或修改的設定。

2.  若要設定的設定，執行下列其中一項：

    -   保留預設值未指定的設定，選取的狀態**未設定**。

    -   若要啟用此設定，請選取**已啟用**。

    -   若要停用此設定，請選取**已停用**。

3.  在 **選項**，如果未列出任何選項，請保留預設值，或視需要加以修改。

4.  執行下列其中一項：

    -   若要儲存變更，並繼續進行下一步 設定，按一下**套用**，然後按一下**下一步 設定**。

    -   若要儲存變更並關閉對話方塊，按一下**確定**。

    -   若要捨棄所有未儲存的變更並關閉對話方塊，按一下**取消**。
