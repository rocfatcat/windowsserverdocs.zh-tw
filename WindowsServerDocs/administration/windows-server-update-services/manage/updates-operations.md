# <a name="updates-operations"></a>更新作業

>適用於：Windows Server （半年通道），Windows Server 2016 中，Windows Server 2012 R2 中，Windows Server 2012

更新已同步處理至 WSUS 伺服器之後，它們會自動掃描伺服器的用戶端電腦的適用性。 不過，您必須核准更新，才能在部署至您網路上的電腦。 當您核准更新時，即表示 WSUS 要用來執行工作項目 (您可以選擇**安裝**或是**拒絕**新的更新)。 您可以核准的更新**的所有電腦**群組或子群組。 如果不核准更新，其核准狀態會維持**未核准**，和您的 WSUS 伺服器可讓用戶端評估是否需要更新。

如果您的 WSUS 伺服器複寫模式中執行，您將無法核准您的 WSUS 伺服器上的更新。 如需複寫模式的詳細資訊，請參閱[執行的 WSUS 複本模式](running-wsus-replica-mode.md)。

## <a name="approving-updates"></a>核准更新
您可以核准安裝的更新，WSUS 網路中的所有電腦或不同的電腦群組。 一經核准更新，您可以執行其中一個 （或以上） 的下列：

-   如果有的話，請將此核准套用到子群組。

-   設定自動安裝的期限。 當您選取此選項時，您設定特定的時間和日期，以安裝更新，覆寫用戶端電腦上的任何設定。 此外，您可以指定期限的過去的日期，如果您想要核准的更新立即 （安裝在用戶端電腦連線到 WSUS 伺服器的下一次）。

-   如果該更新支援的移除，請移除已安裝的更新。

有兩個謹記在心的重要考量：

-   首先，您無法設定自動安裝更新的期限，使用者輸入是否需要 （例如，指定與更新相關的設定）。 若要判斷更新是否會需要使用者輸入，看看**可能會要求使用者輸入**欄位中更新顯示在更新內容**更新**頁面。 也請檢查中的訊息**核准更新** 方塊中，"**所選擇的更新需要使用者輸入，而且不支援的安裝期限**。 」

-   如果有更新至 WSUS 伺服器元件，您無法核准用戶端系統的其他更新，直到已核准 WSUS 更新。 您會看到這則警告訊息在 [核准更新] 對話方塊中："There are 尚未核准的 WSUS 更新。 您應該核准 WSUS 更新核准此更新之前。 」 在此情況下，您應該按一下 [WSUS 更新] 節點，並確定，所有在該檢視中的更新皆已獲核准，然後傳回給一般更新。

#### <a name="to-approve-updates"></a>若要核准更新

1.  在 WSUS 系統管理主控台中，按一下**更新**，然後按一下**所有更新**。

2.  在更新清單中，選取的更新，您想要核准，並以滑鼠右鍵按一下 （或移至 [動作] 窗格），並在核准更新] 對話方塊中，選取 [電腦群組，您要核准該更新，然後按一下它旁邊的箭號。

3.  選取 **已核准安裝**，然後按一下**核准**。

4.  **核准進度**視窗會顯示進度完成核准。 此程序完成時，**關閉**按鈕隨即顯示。 按一下 **關閉**。

5.  您可以選取期限更新上按一下滑鼠右鍵、 選取適當的電腦群組，按一下它，旁邊的箭號，然後按一下**期限**。

    -   您可以選取其中一個標準的截止日期 （一週、 兩週，1 個月），或者您可以按一下**自訂**指定日期和時間。

    -   如果您想要盡速用戶端電腦請連絡伺服器以安裝更新時，請按一下**自訂**，然後將 日期和時間，目前的日期和時間，或在過去的其中一個。

#### <a name="to-approve-multiple-updates"></a>核准多個更新

1.  在 WSUS 系統管理主控台中，按一下**更新**，然後按一下**所有更新**。

2.  若要選取多個連續的更新，請按**shift**選取更新時。 若要選取多個不連續的更新，請按下並按住**CTRL**選取更新時。

3.  以滑鼠右鍵按一下選取項目，然後按一下 **核准**。 **核准更新** 對話方塊隨即開啟與**核准狀態**設定為**保留現有核准**而**確定**已停用按鈕。

4.  您可以變更為個別的群組，核准，但是這麼做，就不會影響子核准。 選取您要變更的核准的群組，然後按一下其左側的箭號。 在快顯功能表中，按一下**已核准安裝**。

5.  變更所選群組的核准**安裝**。 如果有任何子群組，仍會保持其核准**保留現有核准**。 若要變更子群組的核准，按一下群組，並按一下其左側的箭號。 在快顯功能表中，按一下**套用至子系**。

6.  若要設定從父系繼承其所有核准的特定子系，按一下 子系，並按一下其左側的箭號。 在快顯功能表中，按一下**與父系相同**。 如果您設定子系繼承的核准，但並不會變更父系核准時，子系會繼承父系的現有核准。

7.  如果您想要變更所有子系的核准行為時，核准**的所有電腦**，然後選擇**套用到子系**。

8.  按一下 **確定**設定您的核准之後。 **核准進度**視窗會顯示進度完成核准。 此程序完成時，**關閉**按鈕可以使用。 按一下 **關閉**。

## <a name="declining-updates"></a>拒絕更新
如果您選取此選項時，更新會從清單中移除預設的 可用的更新，WSUS 伺服器將不提供給用戶端，用於評估或安裝的更新。 您可以透過此選項，藉由選取更新或群組的更新，以滑鼠右鍵按一下或移至 [動作] 窗格。 拒絕的更新會出現在 [更新] 清單中，只有當您選取**已拒絕**核准清單時指定下方的更新清單的篩選條件中**檢視**。

#### <a name="to-decline-updates"></a>若要拒絕更新

1.  在 WSUS 系統管理主控台中，按一下**更新**，然後按一下**所有更新**。

2.  在更新清單中，選取一或多個您想要拒絕的更新。

3.  選取 **拒絕**，然後按一下**是**上確認訊息。

## <a name="cleaning-up-declined-updates"></a>清除 已拒絕的更新
已拒絕的更新會繼續耗用部分的 WSUS 伺服器資源。 您應該執行 [伺服器清理精靈] 移除拒絕的更新從 WSUS 資料庫。 請參閱：[伺服器清理精靈](the-server-cleanup-wizard.md)，如需詳細資訊。

## <a name="reinstating-declined-updates"></a>恢復已拒絕的更新
已拒絕的更新之後，您仍然可以將其恢復。

#### <a name="to-reinstate-declined-updates"></a>若要重新啟用已拒絕的更新

1.  在 WSUS 系統管理主控台中，按一下**更新**，然後按一下**所有更新**。

2.  變更**核准**要**已拒絕**然後按一下**重新整理**。 已拒絕的更新清單載入。

3.  在更新清單中，選取一或多個您想要恢復的已拒絕的更新。

4.  若要恢復特定更新，以滑鼠右鍵按一下更新，然後選取**核准**。 在 [**核准更新**] 對話方塊中，按一下**確定**來重新套用預設的 「 未核准 」 核准狀態。 更新會顯示在清單中，作為**未核准**而不是已拒絕。

已拒絕的更新已清除藉由使用 WSUS 伺服器清理精靈之後，它將會刪除從 WSUS 伺服器，並不會再出現在 [所有更新] 檢視中。 您可以重新匯入已拒絕，從 Microsoft Update 類別目錄的清除更新。 如需詳細資訊，請參閱[WSUS 和類別目錄站台](wsus-and-the-catalog-site.md)。

## <a name="change-an-approved-update-to-not-approved"></a>變更未核准的核准更新
如果已核准更新，而且您決定不將它安裝在此階段中，而是要將它儲存在未來時間，您可以變更更新到未核准的狀態。 這表示更新仍會在可用的更新的預設清單並將報告用戶端合規性，但將不會安裝在用戶端上。

#### <a name="to-change-an-update-from-approved-to-not-approved"></a>若要變更的更新核准未核准

1.  在 WSUS 系統管理主控台中，按一下**更新**，然後按一下**所有更新**。

2.  在更新清單中，選取一或多個您想要將變更為未核准的核准的更新。

3.  在快顯功能表中或**動作**窗格中，選取**未核准**，然後按一下**是**上確認訊息。

## <a name="approving-updates-for-removal"></a>移除核准更新
您可以核准移除的更新 (亦即，若要解除安裝已安裝的更新)。 更新已安裝，而且支援移除時，才可以使用此選項。 您可以指定要在解除安裝更新的期限，或指定的期限過去的日期，如果您想要移除的更新立即 （下一步 用戶端電腦連線到 WSUS 伺服器的時間）。

很重要，更何況，並非所有更新都支援移除。 您可以看到更新是否支援選取個別更新，並查看移除**詳細資料**窗格。 底下**其他詳細資料**，您會看到**卸除式**類別目錄。 如果無法透過 WSUS 移除更新，在某些情況下就可以移除與**新增或移除程式**從**控制台**。

#### <a name="to-approve-updates-for-removal"></a>若要核准移除的更新

1.  在 WSUS 系統管理主控台中，按一下**更新**，然後按一下**所有更新**。

2.  在更新清單中，選取您想要移除的核准，並以滑鼠右鍵按一下它們的一或多個更新 (或移至**動作**窗格)。

3.  在 [**核准更新**] 對話方塊中，選取您要從中移除更新，電腦群組，然後按一下它旁邊的箭號。

4.  選取 [**核准移除**，然後按一下**移除**] 按鈕。

5.  完成移除核准之後，您可以藉由再次以滑鼠右鍵按一下更新、 選取適當的電腦群組，然後按一下它旁邊的箭號來選取期限。 然後選取**期限**。 您可以選取其中一個標準的截止日期 （一週、 兩週，1 個月），或者您可以按一下**自訂**來選取特定日期和時間。

6.  如果您想要盡速用戶端電腦請連絡伺服器以移除更新，請按一下**自訂**，並設定在過去的日期。

## <a name="approving-updates-automatically"></a>自動核准更新
您可以設定您的 WSUS 伺服器的特定更新的自動核准。 您也可以指定可供使用時的自動核准現有更新的修訂。 這個選項預設會選取。 修訂會更新已變更的版本 （例如，它可能已過期，或其適用性規則可能已變更）。 如果您未選擇要自動核准更新的修訂的版本，WSUS 會使用較舊的版本，而且您必須手動核准更新的修訂。

您可以建立您的 WSUS 伺服器會自動套用同步處理期間的規則。 您指定您要安裝、 更新分類、 產品和電腦群組的自動核准哪些的更新。 只適用於新的更新，而不是修訂的更新。 您也可以指定設定的天數和供應項目之前核准的更新期限安裝特定時間的更新核准期限。 這些設定位於**選項**窗格下方**自動核准**。

#### <a name="to-automatically-approve-updates"></a>若要自動核准更新

1.  在 WSUS 管理主控台中，按一下**選項**，然後按一下**自動核准**。

2.  在 [更新規則] 中，按一下 [新增規則]。

3.  在**新增規則** 對話方塊底下**步驟 1： 選取 屬性**，選取是否要使用**當更新處於某個特定分類**或**有更新時在特定產品中**（或兩者） 做為準則。 （選擇性） 選取是否要**設定的期限**核准。

4.  在 **步驟 2： 編輯屬性**按一下加底線的屬性，以選取分類、 產品和您想要自動核准，適用的電腦群組。 （選擇性） 選擇 更新核准期限日期和時間。

5.  在 **步驟 3:指定 名稱 方塊**，輸入規則的唯一名稱。

6.  按一下 [確定] 。

自動核准規則不會套用到需要終端使用者授權合約 (EULA)，尚未接受在伺服器上的更新。 如果您發現，套用自動核准規則不會導致所有相關的更新核准，您應該手動核准這些更新。

## <a name="automatically-approving-revisions-to-updates-and-declining-expired-updates"></a>自動核准更新的修訂和拒絕已過期的更新
[選項] 窗格的 [自動核准] 區段包含自動核准已核准更新的修訂的預設選項。 您也可以設定您的 WSUS 伺服器會自動拒絕已過期的更新。 如果您選擇不自動核准更新的修訂的版本，您的 WSUS 伺服器會使用較舊的修訂版，因此您必須手動核准更新的修訂。

> [!NOTE]
> 修訂為已變更的更新版本 （例如，它可能已過期或已更新的適用性規則）。

#### <a name="to-automatically-approve-revisions-to-updates-and-decline-expired-updates"></a>若要自動核准更新的修訂和拒絕過期的更新

1.  在 WSUS 管理主控台中，按一下**選項**，然後按一下**自動核准**。

2.  在上**進階**索引標籤上，請確定兩者**自動核准新修訂的已核准更新**和**自動拒絕更新，當新的修訂，使它們到期**已選取。

3.  按一下 [確定]。

    > [!NOTE]
    > 保留預設值，這些選項，可讓您在 WSUS 網路上保有良好效能。 如果不想使用過期的更新，會自動被拒絕，您應該確定定期執行以手動方式加以拒絕。

## <a name="automatically-declining-superseded-updates"></a>自動拒絕被取代的更新
當您核准新的更新，取代現有的更新自動核准時，被取代的更新會變成 「 不適用 」 至電腦或裝置一旦已安裝較新的更新。 您可以在 WSUS 主控台確認所有的電腦已不適用的更新。 當這種情況時，則可以安全地拒絕更新。 此外，可能會自動拒絕更新時執行的 WSUS 伺服器清理精靈。

若要搜尋已取代的更新，您可以選取 「 已取代 」 旗標資料行中的所有更新 檢視中和在該資料行的排序。 會有四個群組：

-   更新已永遠不會取代 （空白圖示）。

-   更新已被取代，但永遠不會取代其他更新 （在底部的藍色方框圖示）。

-   更新以被取代，並已取代其他更新 （在中間的藍色方框圖示）。

-   已取代其他更新 （在頂端的藍色方框圖示） 的更新。

Windows Server Update Services，會自動拒絕被取代更新核准的較新的更新沒有任何功能。 建議您先設定 「 未核准 」，以核准，然後使用 已滿足所有相關的條件時，會自動拒絕更新的 伺服器清理精靈。 如需詳細資訊，請參閱：[伺服器清理精靈](the-server-cleanup-wizard.md)。

## <a name="approving-superseding-or-superseded-updates"></a>核准取代或已取代的更新
通常，會取代其他更新的更新會一或多個項目：

-   增強、 改進或新增一或多個先前發行的更新所提供的修正程式。

-   可改善其如果的安裝已核准的更新，會將用戶端電腦安裝的更新檔封裝的效率。 例如，被取代的更新可能包含不再相關的修正程式或作業系統現在支援新的更新，因此不會取代的更新檔案封裝中包含這些檔案的檔案。

-   更新較新版本的作業系統。 它也是作業系統的值得注意，取代更新可能不支援舊版。

相反地，由其他更新取代的更新會執行下列作業：

-   修正的問題類似於的更新，取代它。 不過，取代更新，它可能會強化修正被取代的更新提供。

-   更新舊版的作業系統。 在某些情況下，這些版本的作業系統不再更新所取代的更新。

在個別的更新詳細資料窗格中，參考的圖示，然後在頂端的訊息指出它取代，或由其他更新所取代。 此外，您可以判斷哪些更新取代或更新取代藉由查看**取代此更新的更新**並**由此更新取代的更新**中的項目**其他詳細資料**一節**屬性**。 更新的詳細資料窗格會顯示以下的更新清單。

WSUS 不會自動拒絕被取代的更新，並建議您不要不假設，被取代的更新應拒絕而採納新，取代更新。 之前拒絕被取代的更新，請確定它已不再需要由任何用戶端電腦。 以下是範例的情況下，您可能需要安裝被取代的更新：

-   如果取代的更新只支援較新版的作業系統，而且某些用戶端電腦執行舊版作業系統。

-   如果取代的更新有適用性限制比多更新它取代了，使它不適用於某些用戶端電腦。

-   如果更新不再取代先前發行的更新，因為新的變更。 可以，透過在每個版本的變更，更新不再取代其原本該取代舊版本中的更新。 在此案例中，您仍會看到的訊息被取代的更新，即使並不會更新已取代的更新，取代它。
