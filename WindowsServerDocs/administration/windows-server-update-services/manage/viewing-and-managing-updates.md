# <a name="viewing-and-managing-updates"></a>檢視和管理更新

>適用於：Windows Server （半年通道），Windows Server 2016 中，Windows Server 2012 R2 中，Windows Server 2012

您可以使用 WSUS 主控台來檢視和管理更新。

## <a name="viewing-updates"></a>檢視更新
在 [**更新**] 頁面上，您可以執行下列動作：

-   檢視更新狀態。 更新概觀會顯示已更新來源的資料同步到您的 WSUS 伺服器，並可供核准的更新。

-   篩選更新。 在預設檢視中，您可以篩選更新的核准狀態] 和 [安裝狀態。 預設值為未核准的更新時所需的某些用戶端，或在某些用戶端上已安裝失敗。 您可以變更此檢視，以變更的核准狀態和安裝狀態篩選器，然後按一下**重新整理**。

-   建立新的更新檢視。 在 **動作**窗格中，按一下**新的更新檢視**。 您可以篩選依照分類、 產品、 在有已核准、 群組和同步處理日期的更新。 您可以按一下標題列中的適當的資料行標題來排序清單。

-   搜尋更新。 您可以依標題、 描述、 知識庫文章或更新的 Microsoft Security Response Center 編號搜尋個別的更新或一組更新。

-   檢視詳細資料、 狀態，以及每個更新的修訂歷程記錄。

-   核准和拒絕更新。

#### <a name="to-view-updates"></a>若要檢視更新

1.  在 WSUS 管理主控台中，展開 [更新] 節點，然後按一下所有更新。

2.  根據預設，更新會顯示其標題、 與分類、 已安裝/不適用的百分比，核准狀態。 如果您想要顯示更多或不同的更新內容，以滑鼠右鍵按一下欄標題列，然後選取適當的資料行。

3.  若要排序依據不同準則，例如下載狀態 」、 「 標題 」、 「 分類 」、 「 發行日期或 「 核准狀態中，按一下適當的資料行標題。

#### <a name="to-filter-the-list-of-updates-displayed-on-the-updates-page"></a>若要篩選顯示在 [更新] 頁面上的更新清單

1.  在 WSUS 管理主控台中，依序展開**更新**，然後按一下**所有更新**。

2.  在中央窗格旁**核准**，選取 [所需的核准狀態] 和 [下的一步]**狀態**選取所需的安裝狀態。 按一下 **重新整理**。

#### <a name="to-create-a-new-update-view-on-wsus"></a>若要建立新的更新檢視，請在 WSUS

1.  在 WSUS 管理主控台中，依序展開**更新**，然後按一下**所有更新**。

2.  在 **動作**窗格中，按一下**新的更新檢視**。

3.  在 **新增更新的檢視**視窗底下**步驟 1： 選取 屬性**，選取您要篩選更新檢視的屬性：

    -   選取更新處於特定分類來篩選更新屬於其中一或多個更新分類。

    -   選取 有更新一或多個產品或產品系列篩選特定產品的更新。

    -   選取要篩選的一個或多個電腦群組核准更新的特定群組核准更新。

    -   選取要篩選更新同步處理在特定時間在特定期間內同步處理更新。

    -   選取更新是做為篩選條件 WSUS 更新 WSUS 更新。

4.  底下**步驟 2： 編輯屬性**，按一下加底線的文字，挑選您想要的值。

5.  在下**步驟 3:指定的名稱**，提供您的新檢視的名稱。

6.  按一下 [確定] 。

您的新檢視會出現在樹狀檢視窗格下方的更新。 它將會顯示，例如當您選取中間窗格中的標準檢視。

#### <a name="to-search-for-an-update"></a>若要搜尋更新

1.  選取 **更新**節點 （或其下的任何節點）。

2.  在 **動作**窗格中，按一下**搜尋**。

3.  在 [**搜尋**] 視窗，請在**更新**索引標籤上，輸入您的搜尋準則。 您可以使用文字**標題**，**描述**，並**Microsoft 知識庫 (KB) 文章編號**欄位。 每一個項目是屬性列於**詳細資料**更新內容 索引標籤。

#### <a name="to-view-the-properties-for-an-update"></a>若要檢視更新的內容

1.  在 WSUS 管理主控台中，依序展開**更新**，然後按一下**所有更新**。

2.  在更新清單中，按一下您想要檢視的更新。

3.  在下方窗格中，您會看到不同的屬性區段：

    -   標題列會顯示更新的標題例如，安全性更新的 Windows Media Player 9 (KB911565)。

    -   [狀態] 區段會顯示安裝更新的狀態 （它必須在其安裝的電腦、 電腦它已安裝但發生錯誤，它已安裝或不適用，電腦和未回報的電腦狀態更新），以及 （KB，MSRC 編號發行日期等等） 的一般資訊。

    -   [描述] 部分顯示更新的簡短描述。

    -   其他詳細資料 區段會顯示下列資訊：

        -   安裝更新的行為 （已卸除式、 要求重新啟動、 需要使用者輸入，或必須單獨安裝）。

        -   更新具有 Microsoft 軟體授權條款

        -   要套用更新的產品

        -   取代此更新的更新

        -   此更新取代的更新

        -   更新所支援的語言

        -   更新識別碼

請注意，您可以在只有一個更新一次執行此程序。 如果您選取多個更新，只在清單中的第一個更新會顯示在**屬性**窗格。

## <a name="managing-updates-with-wsus"></a>使用 WSUS 管理更新
更新用來更新，或提供的電腦安裝軟體的完整檔案取代項目。 每個更新，可在 Microsoft Update 是兩個元件組成：

-   中繼資料：提供關於更新的資訊。 例如，中繼資料會提供屬性的更新，如此可讓您找出的項目更新是很有用的資訊。 中繼資料也包含 Microsoft 軟體授權條款。 下載更新的中繼資料封裝是通常遠低於實際的更新檔案封裝。

-   更新檔案：實際的檔案，才能在電腦上安裝更新。

當將更新同步處理至 WSUS 伺服器時，中繼資料和更新檔案會儲存在兩個不同的位置。 中繼資料會儲存在 WSUS 資料庫中。 在您的 WSUS 伺服器上或 Microsoft Update 伺服器上，根據您的同步處理選項的設定方式，就可以儲存更新檔案。 如果您選擇儲存在 Microsoft Update 伺服器上的更新檔案，當時的同步處理; 下載僅中繼資料核准 WSUS 主控台中，透過更新，然後用戶端電腦取得直接從 Microsoft Update 在安裝期間更新檔案。 如需儲存更新選項的詳細資訊，請參閱下一節[1.3。選擇 WSUS 儲存策略](../plan/plan-your-wsus-deployment.md#BKMK_1.3.)的步驟 1:準備部署 WSUS，WSUS 部署指南中。

您要設定並執行同步處理期間，新增電腦及電腦群組與部署定期更新。 下列清單提供您可能執行的 wsus 更新電腦的一般工作的範例。

-   決定根據您的網路拓撲和頻寬、 公司的需求，以及組織結構的整體更新管理方案。

    -   是否要設定 WSUS 伺服器階層，以及結構階層的方式。

    -   哪些電腦群組來建立，以及如何將電腦指派給 （伺服器端或用戶端目標）。

    -   要用於更新中繼資料 （例如，Windows Internal Database、 SQL Server） 的資料庫。

    -   是否自動和時間，更新就應該同步處理。

-   設定同步處理選項，例如更新來源、 產品和更新分類、 語言、 連線設定、 儲存體位置和同步處理排程。

-   取得更新和相關聯的中繼資料在您的 WSUS 伺服器上透過同步處理從 Microsoft Update 或上游 WSUS 伺服器。

-   核准或拒絕更新。 您可以選擇允許使用者安裝更新本身 （如果它們是在其用戶端電腦上的本機系統管理員）。

-   設定自動核准。 您也可以設定是否要啟用自動核准現有更新的修訂，或手動核准的修訂。 如果您選擇手動核准的修訂，WSUS 伺服器會繼續使用舊的版本，直到您手動核准新修訂。

-   檢查更新的狀態。 您可以檢視更新狀態、 列印狀態報表，或設定規則的狀態報告的電子郵件。

## <a name="update-products-and-classifications"></a>更新產品和分類
可透過 Microsoft Update 取得更新的差別在於產品 （或產品系列） 和分類。

產品即為特定版本的作業系統或應用程式，比方說，Windows Server 2012。 產品系列是指衍生出個別產品的基底作業系統或應用程式。 產品系列的範例是 Microsoft Windows，Windows Server 2012 的成員。 您可以選取的產品或產品系列，您會想要同步處理更新伺服器。 您可以指定產品系列或系列中的個別產品。 選取任何產品或產品系列，將取得產品的目前和未來版本的更新。

更新分類代表更新類型。 對於任何特定的產品或產品系列，更新可能可在多個更新分類 （例如，Windows 7 系列重大更新和安全性更新）。 下表列出的更新分類。

| 更新分類  | 描述   |
|--|--|
|重大更新|廣泛發行的特定問題解決重大、 非安全性相關的 bug 的修正程式。|
|定義更新|更新至病毒或其他定義檔案。|
|驅動程式|設計來支援新硬體的軟體元件。|
|Feature pack|新的功能版本，通常會納入下一版的產品。|
|安全性更新|廣泛發行的特定產品，解決安全性問題的修正程式。|
|Service Pack|所有的 hotfix、 安全性更新、 重大更新積存組合和更新的產品發行以來所建立。 Service pack 也可能包含有限的數目的客戶要求設計變更或功能。|
|工具|公用程式或功能來協助您完成工作或一組工作。|
|更新彙總套件|Hotfix、 安全性更新、 重大更新和其他封裝在一起以便於部署的更新累計集。 彙總套件通常以特定的區域中，例如安全性或特定元件，例如網際網路資訊服務 (IIS) 為目標。|
|更新|廣泛發行的定址非重大 」、 「 非安全性相關的 bug 的特定問題的修正程式。|
