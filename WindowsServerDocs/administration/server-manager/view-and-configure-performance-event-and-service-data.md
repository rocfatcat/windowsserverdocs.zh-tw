# <a name="view-and-configure-performance-event-and-service-data"></a>檢視與設定效能、事件及服務資料

>適用於：Windows Server （半年通道），Windows Server 2016 中，Windows Server 2012 R2 中，Windows Server 2012

本主題描述如何檢視與設定事件記錄項目、 效能計數器，以及針對本機和遠端伺服器在 [伺服器管理員] 中顯示的服務警示。  

事件、 服務及效能記錄資料會顯示在 [伺服器管理員] 主控台，在 Windows Server 中的兩個地方。  

-   在儀表板，您可以按一下 **事件**，**效能**，並**Services**設定事件、 效能及服務記錄檔資料，您想要查看的縮圖的資料列角色、 整個伺服器管理員伺服器集區、 使用者建立的伺服器群組和本機伺服器。 按一下超文字列，開啟**詳細資料檢視**對話方塊，可讓您指定您想要收到儀表板中的資料。 您設定事件、 服務及您想要在儀表板縮圖中反白顯示的效能記錄資料之後，符合您所指定條件的記錄項目會列在底部**詳細資料檢視**對話方塊。  

-   [事件]、[服務] 及 [效能] 磚都是角色和群組首頁的一部分。 這些磚的 [工作] 功能表上的命令讓您能夠指定要從受管理伺服器收集的資料。 磚包含篩選與查詢，可以視需要進一步限制要在磚中顯示的記錄檔項目。  

本主題涵蓋下列各節。

## <a name="BKMK_thumb"></a>縮圖有哪些？  
*縮圖*每個角色 （角色的縮圖會反映資料收集的關於所有伺服器管理員集區中執行該角色的伺服器），在伺服器管理員儀表板會顯示每個伺服器群組，如**所有伺服器**群組 （所有的伺服器管理員集區中的伺服器），以及本機伺服器。 伺服器管理員會從受管理伺服器取得資料之後，縮圖會自動建立伺服器集區中的伺服器上執行的角色。  

如果伺服器管理員 主控台正在用戶端電腦上做為遠端伺服器管理工具的一部分，則任何**本機伺服器**縮圖。  

縮圖會顯示角色、伺服器及伺服器群組的狀態與管理性的快速檢視。 縮圖標題列就會變更色彩 （並反白顯示的數字會顯示在左邊界） 當事件、 效能計數器、 最佳做法分析程式結果、 服務或一般管理性問題符合您所設定準則中**詳細資料檢視**對話方塊開啟按一下縮圖列。 下表描述縮圖中顯示的資料。  

|縮圖列|描述|  
|---------|--------|

|效能|您可以設定**效能**列來顯示角色或伺服器群組的警示，當出現符合資源類型、 伺服器或您在中指定的時間週期的效能警示**效能詳細資料檢視**  對話方塊。<br /><br />預設會關閉效能計數器。 受管理的伺服器執行的作業系統比 Windows Server 2003，而且效能計數器尚未啟動，通常會顯示管理性狀態錯誤**未啟動效能計數器的上線-** 在 **伺服器**角色或群組頁面的圖格。 若要開啟效能計數器受管理的伺服器上**所有伺服器**頁面上，以滑鼠右鍵按一下中的項目**效能**圖格，以顯示**計數器狀態**值**關閉**，然後按一下**啟動效能計數器**。 您也可以以滑鼠右鍵按一下項目中的伺服器啟動效能計數器**伺服器**的角色或群組頁面，然後按一下圖格**啟動效能計數器**。|

## <a name="BKMK_events"></a>檢視和設定事件  
在本節中，了解如何設定事件記錄檔收集哪些資料是從伺服器管理員伺服器集區中的伺服器，以及您要縮圖中反白顯示哪些事件。  

> [!NOTE]  
> 您要警示哪些縮圖中的事件會指示伺服器管理員，從受管理伺服器收集的事件總數子集。 雖然變更事件條件**設定事件資料**] 對話方塊中的**事件**圖格可以變更您看到 [伺服器管理員儀表板中，變更中的事件警示準則的警示數目縮圖會有任何作用，從受管理伺服器收集的事件記錄檔資料。  

#### <a name="to-configure-the-events-collected-from-managed-servers"></a>設定從受管理伺服器收集的事件  

1.  在 [伺服器管理員] 主控台中，開啟儀表板以外的任何頁面。 您可以在角色、伺服器群組或本機伺服器頁面上的 [事件]  磚中，設定要從受管理伺服器收集的事件。  

2.  在 [事件]  磚的 [工作]  功能表上，按一下 [設定事件資料] 。  

3.  選取您想要從所選群組中的伺服器收集的事件嚴重性層級。 預設會選取 [嚴重]、[錯誤] 及 [警告] 嚴重性層級。  

4.  指定事件發生的時間間隔。 預設的事件保留時間為 24 小時。  

5.  選取您想要收集事件的事件記錄檔。 預設值為 [應用程式] 、[設定] 及 [系統] 。  

6.  若要儲存您的變更，請按一下 [確定]  來關閉 [設定事件資料]  對話方塊。 儲存變更之後，事件資料會自動重新整理。  

#### <a name="to-configure-the-events-highlighted-in-thumbnails"></a>在縮圖中設定要反白顯示的事件  

1.  如果已經開啟伺服器管理員，請移至下一個步驟。 如果尚未開啟伺服器管理員，請執行下列其中一項動作來將它開啟。  

    -   在 Windows 桌面上，按一下 Windows 工作列中的 [伺服器管理員] 來啟動 [伺服器管理員]。  

    -   在 Windows **\[開始\]** 畫面上，按一下 \[伺服器管理員\] 磚。  

2.  在儀表板頁面上，於 [角色及伺服器群組]  磚的縮圖中，按一下 [事件]  列。  

3.  在 **事件詳細資料檢視**對話方塊方塊中，將嚴重性層級新增至您想要顯示的事件。 根據預設，縮圖只會顯示針對嚴重事件進行反白顯示的警示。 請注意，在顯示的事件數目**詳細資料檢視**對話方塊會增加，當您將新增您想要接獲通知的嚴重性層級。  

4.  在 [事件來源] 欄位中，選取您要警示的事件來源。 預設為 **All**。  

5.  如果這個縮圖是由多部伺服器或一群多部伺服器已安裝的角色，您可以選取要事件警示中的伺服器**伺服器**下拉式清單。  

6.  在 **時段**欄位中，指定一段時間，最多為 1440年分鐘、 24 小時或 1 天。  

7.  在 [事件識別碼] 欄位中，輸入您要警示的特定事件的事件識別碼。 您可以輸入某個範圍的事件識別碼並以虛線 (**-**) 分隔，並且在您要排除的事件識別碼或事件識別碼範圍之前輸入虛線，來從範圍中排除事件識別碼。 例如，值 **1,3,5-99,-76** 表示會針對事件識別碼 1 和 3，以及 5 到 99 之間的所有事件識別碼 (但事件識別碼 76 除外) 引發警示。  

8.  當您變更顯示警示的條件時，對話方塊底部的結果窗格中顯示的事件警示數目可能會變更。 在清單中選取項目，然後按一下**隱藏警示**，防止它們影響來源縮圖上顯示的警示計數。 當您選取警示時按住 **Ctrl**，可以一次選取多個警示。 您可以針對符合事件警示準則但不需要看見的警示執行這個動作。  

9. 按一下 [顯示全部]  ，讓隱藏的警示重新出現在清單中。  

10. 按一下  **確定**以儲存您的變更，關閉**詳細資料檢視**對話方塊，然後檢視來源縮圖中的事件警示變更。  

## <a name="BKMK_perf"></a>檢視和設定效能記錄資料  
在本節中，了解如何設定從伺服器管理員伺服器集區中的伺服器收集的效能記錄資料，以及哪些效能計數器警示您要在縮圖中的 反白顯示。  

預設會關閉效能計數器。 受管理的伺服器執行的作業系統比 Windows Server 2003，而且效能計數器尚未啟動，通常會顯示管理性狀態錯誤**未啟動效能計數器的上線-** 在 **伺服器**角色或群組頁面的圖格。 若要開啟效能計數器受管理的伺服器上**所有伺服器**頁面上，以滑鼠右鍵按一下中的項目**效能**圖格，以顯示**計數器狀態**值**關閉**，然後按一下**啟動效能計數器**。 您也可以以滑鼠右鍵按一下項目中的伺服器啟動效能計數器**伺服器**的角色或群組頁面，然後按一下圖格**啟動效能計數器**。  

> [!NOTE]  
> 您檢視縮圖中的效能警示是您指示從受管理伺服器收集的伺服器管理員總數的效能計數器資料的子集。 雖然變更效能警示準則**設定效能警示**] 對話方塊中的**效能**圖格可以變更您看到 [伺服器管理員儀表板上，變更的警示數目縮圖中的效能警示準則已從受管理伺服器收集的效能記錄檔資料無效。  
>   
> 因此，您可以在縮圖中顯示效能資料的存留期上限，不可大於 [設定效能警示]  對話方塊中設定的圖形顯示期間上限。 例如，如果**圖形顯示期間**中的值**設定效能警示**是**1 天**，最大值**時段**欄位**效能詳細資料檢視**可以是您從 伺服器管理員儀表板開啟的對話方塊**1 天**， **24 小時**，或**1,440 分鐘**。  

#### <a name="to-configure-the-performance-log-data-collected-from-managed-servers"></a>設定從受管理伺服器收集的效能記錄檔資料  

1.  在 [伺服器管理員] 主控台中，開啟儀表板以外的任何頁面。 您可以在角色、伺服器群組或本機伺服器頁面上的 [效能] 磚中，設定要從受管理伺服器收集的效能資料。  

2.  若要從受管理的伺服器收集效能記錄檔資料，必須開啟效能計數器。 如果效能計數器已關閉，以滑鼠右鍵按一下中的項目**效能** 清單中的磚，然後按一下**啟動效能計數器**。 根據收集資料的伺服器數目以及可用的網路頻寬而定，效能計數器資料收集可能需要花費一些時間。 檢視 **[計數器狀態]** 欄中的狀態。  

3.  在 [效能]  磚的 [工作]  功能表上，按一下 [設定效能警示] 。  

4.  在選取的群組，或是伺服器正在執行選取的角色，指定要收集伺服器管理員的效能計數器警示的 CPU 使用量百分比。 預設值為 85%。  

5.  指定其餘可用的記憶體 (MB)，此為在您想要收集效能計數器警示之前，伺服器必須具備的記憶體。 預設值為 2 MB。  

6.  指定要在選取的頁面上，依照 [效能]  磚中資源 [CPU 使用率]  和 [可用記憶體]  的圖表來顯示的時間間隔。 預設期間為一天。 按一下 [儲存] 。  

    請注意，在您按一下 [儲存]  之後，可以變更 [效能] 磚中的效能警示數目，以及透過圖形顯示的一段時間的警示對應。  

    > [!NOTE]  
    > 有的虛擬機器[動態記憶體](https://technet.microsoft.com/library/ff817651.aspx)開啟，提高效能警示閾值會產生誤判警示。  

7.  若要重新整理從伺服器收集的效能警示清單，可以在 [工作] 功能表上，按一下 [重新整理]。  

#### <a name="to-configure-the-performance-alerts-highlighted-in-thumbnails"></a>在縮圖中設定要反白顯示的效能警示  

1.  在儀表板頁面上，於 [角色及伺服器群組]  磚的縮圖中，按一下 [效能]  列。  

2.  在 **效能詳細資料檢視**對話方塊中，選取或清除核取方塊，針對您想要接獲通知中的資源效能閾值**資源類型**欄位。 請注意，在顯示的效能警示數目**詳細資料檢視**新增您想要接獲通知的資源效能閾值時，可以增加 對話方塊。  

3.  如果這個縮圖是由多部伺服器或一群多部伺服器已安裝的角色，您可以選取要效能警示中的伺服器**伺服器**下拉式清單。  

4.  在 **時段**欄位中，指定一段時間，最多為 1440年分鐘、 24 小時或 1 天。  

5.  當您變更顯示警示的條件時，對話方塊底部的結果窗格中顯示的警示數目可能會變更。 按一下 [隱藏警示]  ，隱藏早於目前時間的所有警示，防止它們對來源縮圖上顯示的警示計數產生影響。  

6.  按一下 [顯示全部]  ，讓隱藏的警示重新出現在清單中。  

7.  按一下  **確定**以儲存您的變更，關閉**詳細資料檢視**對話方塊，然後檢視來源縮圖中的效能警示變更。  

#### <a name="to-view-the-properties-of-performance-alerts"></a>檢視效能警示的內容  

1.  執行下列其中一項。  

    -   在儀表板頁面上，於 [角色及伺服器群組]  磚的縮圖中，按一下 [效能]  列。  

    -   開啟角色或群組首頁，並找到該角色或群組的 [效能] 磚。  

2.  在清單中按兩下效能警示，以檢視它的內容。 或者，您可以在效能警示上按一下滑鼠右鍵，然後按一下 [檢視內容]。  

3.  在 [效能警示屬性] 對話方塊中，選取記錄項目，以檢視與 [處理程序] 區域中的項目相關聯的處理程序相關資訊。  

4.  當您完成效能警示內容的檢視時，請關閉對話方塊。

## <a name="BKMK_services"></a>管理服務，並設定服務警示  
在本節中，了解如何啟動、 停止、 重新啟動、 暫停或繼續服務，會顯示在**Services**上角色及伺服器群組頁面在 [伺服器管理員] 中的圖格。 您也可以在 伺服器管理員儀表板的縮圖中，設定您要警示的服務。  

> [!NOTE]  
> 您無法變更服務、 服務相依性、 復原選項或其他服務中的屬性在 [伺服器管理員] 中的 [服務] 圖格的啟動類型。 若要變更服務狀態以外的服務內容，請開啟 [服務] 嵌入式管理單元。 若要開啟快顯**服務**嵌入式管理單元並用於**工具**功能表在 [伺服器管理員] 中。  

#### <a name="to-start-stop-restart-pause-or-resume-a-service"></a>啟動、停止、重新啟動、暫停或繼續服務  

1.  在 [伺服器管理員] 主控台中，開啟儀表板 （也就是說，任何角色或群組首頁） 以外的任何頁面。  

2.  在角色或群組的 [服務]  磚中，用滑鼠右鍵按一下服務。  

3.  在操作功能表中，按一下您要在服務上執行的動作。 如果服務已停止，唯一可以執行的動作就是啟動服務。 同樣地，如果服務已暫停，則唯一可以執行的動作就是繼續服務。  

4.  請注意，當您啟動、停止、重新啟動、暫停或繼續服務時，[狀態]  欄的值會針對 [服務]  磚中的服務進行變更。  

#### <a name="to-configure-service-alerts-highlighted-in-thumbnails"></a>在縮圖中設定要反白顯示的服務警示  

1.  在儀表板頁面上，於 [角色及伺服器群組] 磚的縮圖中，按一下 [服務] 列。  

2.  在 **服務詳細資料檢視**對話方塊方塊中，選取您想要接獲通知的服務啟動類型。 根據預設，**自動 （延遲開始）** 並**自動**已選取。  

3.  選取您想要接獲通知的服務狀態。 預設會選取 [全部]  。  

4.  選取您想要接獲通知的服務。 預設會選取 [全部]  。  

5.  選取您要接收警示的相關服務群組的角色相關聯的伺服器。 預設會選取 [全部]  。  

6.  當您變更顯示警示的條件時，對話方塊底部的結果窗格中顯示的警示數目可能會變更。 按一下 [隱藏警示]  ，隱藏早於目前時間的所有警示，防止它們對來源縮圖上顯示的警示計數產生影響。  

7.  按一下 [顯示全部]  ，讓隱藏的警示重新出現在清單中。  

8.  按一下  **確定**以儲存您的變更，關閉**詳細資料檢視**對話方塊，然後檢視來源縮圖中的服務警示變更。  

## <a name="BKMK_copy"></a>檢視和複製事件、 服務或效能項目  
您可以複製事件、 服務或效能項目內容中都**詳細資料檢視**對話方塊並**事件**並**效能**角色或群組的圖格。 事件或效能項目，以滑鼠右鍵按一下，然後按一下**複製**。  

[事件]  磚也可以讓您透過選取清單中的事件，在磚的下半部中預覽事件內容。 若要複製預覽中顯示的屬性，以滑鼠右鍵按一下 [預覽] 窗格中，，然後按一下**複製**。
