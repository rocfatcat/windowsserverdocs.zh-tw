# <a name="manage-the-local-server-and-the-server-manager-console"></a>Manage the Local Server and the Server Manager Console

>適用於：Windows Server （半年通道），Windows Server 2016 中，Windows Server 2012 R2 中，Windows Server 2012

在 Windows Server 中，伺服器管理員可讓您同時管理本機伺服器 （如果您在 Windows Server 上並不是在以 Windows 為基礎的用戶端作業系統上執行伺服器管理員） 和執行 Windows Server 2008 和較新版 Windows 的遠端伺服器伺服器作業系統。

**本機伺服器**頁面在 [伺服器管理員] 中顯示伺服器屬性、 事件、 服務和效能計數器資料，並對本機伺服器的 Best Practices Analyzer (BPA) 結果。 事件、服務、BPA 和效能磚的運作方式與在角色和伺服器群組頁面上相同。 如需關於設定這些磚中所顯示資料的詳細資訊，請參閱[檢視與設定效能、事件及服務資料](view-and-configure-performance-event-and-service-data.md)以及[執行最佳做法分析程式掃描與管理掃描結果](run-best-practices-analyzer-scans-and-manage-scan-results.md)。

功能表命令和伺服器管理員主控台標題列中的設定全域套用至您的伺服器集區中的所有伺服器，並可讓您使用伺服器管理員來管理整個伺服器集區。

本主題涵蓋下列各節。

-   [關閉本機伺服器](#BKMK_shutdown)

-   [設定伺服器管理員屬性](#BKMK_props)

-   [管理伺服器管理員主控台](#BKMK_managesm)

-   [自訂 [工具] 功能表中所顯示的工具](#BKMK_tools)

-   [管理角色首頁上的角色](#BKMK_roles)

## <a name="BKMK_shutdown"></a>關閉本機伺服器
**任務**本機伺服器中的功能表**屬性**圖格可讓您在本機伺服器上，啟動 Windows PowerShell 工作階段中開啟**管理的電腦**mmc 嵌入式管理單元，或開啟mmc 嵌入式管理單元的角色或本機伺服器所安裝的功能。 您也可以使用這個 [工作] 功能表中的 [關閉本機伺服器] 命令來關閉本機伺服器。 您也可以在 [所有伺服器]  頁面的 [伺服器]  磚中或在本機伺服器代表的任何角色或群組頁面中，找到供本機伺服器使用的  [關閉本機伺服器] 命令。

關閉本機伺服器，使用此方法，不同的是從 Windows Server 2016**開始**畫面上，開啟**向下 Windows 關閉**對話方塊中，可讓您指定關機的原因在 **關機事件追蹤器**區域。

> [!NOTE]
> 只有 Administrators 群組的成員才能關閉或重新啟動伺服器。 標準使用者無法關閉或重新啟動伺服器。 按一下 [關閉本機伺服器]  命令會將標準使用者登出伺服器工作階段。 這與標準使用者從伺服器桌面執行 **Alt+F4** 關機命令的經驗相符。

## <a name="BKMK_managesm"></a>管理伺服器管理員主控台
標題列頂端的 [伺服器管理員] 主控台視窗中，找到將套用至整個的 [伺服器管理員] 主控台中，並已新增至伺服器管理員伺服器集區的所有遠端伺服器的全域設定。

### <a name="add-servers-to-server-manager"></a>將伺服器新增到伺服器管理員
開啟的命令**將伺服器加入** 對話方塊中，並可讓您新增遠端實體或虛擬伺服器至伺服器管理員的伺服器集區，會在**管理**功能表的 伺服器管理員 主控台。 如需如何新增伺服器的詳細資訊，請參閱[將伺服器新增到伺服器管理員](add-servers-to-server-manager.md)。

### <a name="refresh-data-that-is-displayed-in-server-manager"></a>重新整理伺服器管理員中顯示的資料
您可以設定資料重新整理間隔上所顯示在 [伺服器管理員**伺服器管理員屬性**] 對話方塊中，您可以從開啟**管理**功能表。

##### <a name="to-configure-the-refresh-interval-in-server-manager"></a>在伺服器管理員中設定重新整理間隔

1.  在 **管理**功能表中 伺服器管理員 主控台中，按一下 **伺服器管理員屬性**。

2.  在 **伺服器管理員屬性**對話方塊方塊中，指定一段時間，以分鐘為單位，會顯示在 伺服器管理員 中的資料重新整理之間經過時間量。 預設值是 10 分鐘。 完成後，請按一下 [確定]。

#### <a name="refresh-limitations"></a>重新整理限制
重新整理會全域套用至資料的已加入伺服器管理員的伺服器集區的所有伺服器。 您無法針對個別伺服器、角色或群組重新整理資料或設定不同的重新整理間隔。

當叢集中的伺服器加入至 [伺服器管理員] 中，無論是實體電腦或虛擬機器時，第一次重新整理資料可能會失敗，或是只顯示資料之主機伺服器的叢集物件。 後續重新整理就會顯示伺服器叢集中實體或虛擬伺服器的正確資料。

遠端桌面服務、 IP 位址管理以及檔案和存放服務角色首頁中 伺服器管理員中顯示的資料不會自動重新整理。 重新整理以手動的方式，藉由按下顯示這些頁面中的資料**F5**或按一下**重新整理**在伺服器管理員主控台標題，當您在這些頁面上。

在 [伺服器管理員] 中，角色和功能資料會顯示在系統中，也稱為系統預設 GUI 語言或在作業系統安裝期間選取的語言的基礎語言。

### <a name="create-server-groups"></a>建立伺服器群組

### <a name="prevent-server-manager-from-opening-automatically-at-logon"></a>避免登入時自動開啟伺服器管理員
**在登入時不要自動啟動伺服器管理員**中的核取方塊**伺服器管理員屬性**對話方塊可讓您控制是否伺服器管理員會自動開啟在登入時的成員在本機伺服器上的系統管理員群組。 在 Windows 10 上執行遠端伺服器管理工具的一部分時，此設定就不會影響伺服器管理員的行為。 如需有關如何進行這項設定的詳細資訊，請參閱 <<c0> [ 伺服器管理員](server-manager.md)。

### <a name="zoom-in-or-out"></a>放大或縮小
若要放大或縮小您的伺服器管理員主控台的檢視，您可以使用**縮放**上的命令**檢視**功能表，或是按下**Ctrl + 加號 （+）** 拉近與**Ctrl + 減號 （-）** 縮小。

## <a name="BKMK_tools"></a>自訂 [工具] 功能表中所顯示的工具
**工具**功能表在 伺服器管理員 中所包含的捷徑的軟連結**系統管理工具**中的資料夾**控制台/系統及安全性**。 **系統管理工具**資料夾包含一份捷徑或 LNK 檔可用的管理工具，例如 mmc 嵌入式管理單元。伺服器管理員會填入**工具**功能表的連結，這些捷徑，並將複製的資料夾結構**系統管理工具**資料夾**工具**功能表。 根據預設，[系統管理工具] 資料夾中的工具會是依類型或名稱排序的平面清單。 在 [伺服器管理員**工具**] 功能表中，只依名稱，不會依類型排序項目。

若要自訂 [工具]  功能表，請將您要使用的工具或指令碼捷徑複製到 [系統管理工具]  資料夾。 您也可以管理資料夾中的捷徑，這樣會在 [工具]  功能表中建立串聯功能表。 此外，如果您想要限制存取自訂工具**工具** 功能表中，您可以設定使用者存取權限，在 系統管理工具 這兩個您自訂工具資料夾上或直接在原始工具或指令碼檔。

不建議您重新組織系統和系統管理工具，以及與本機伺服器上安裝的角色和功能相關的任何管理工具。 移動角色和功能管理工具可在必要時避免成功解除安裝這些管理工具。 解除安裝角色或功能之後，捷徑已移除之工具的無作用連結可能仍留在 [工具] 功能表中。 如果您重新安裝角色，則會在 [工具] 功能表中建立相同工具的重複連結，但是這兩個連結的其中一個將無法作用。

不過，在以 Windows 用戶端為基礎的電腦上安裝為遠端伺服器管理工具一部分的角色和功能工具，可以組織到自訂資料夾中。 解除安裝父系角色或功能沒有任何作用，執行 Windows 10 的遠端電腦上可用的工具捷徑。

下列程序描述如何建立名為的範例資料夾*MyTools*，和兩個 Windows PowerShell 指令碼捷徑移動到資料夾，然後從 [伺服器管理員工具] 功能表可存取。

#### <a name="to-customize-the-tools-menu-by-adding-shortcuts-in-administrative-tools"></a>在系統管理工具中新增捷徑以自訂工具功能表

1.  建立新資料夾，稱為*MyTools*在方便的位置。

    > [!NOTE]
    > 由於 [系統管理工具]  資料夾有存取權限限制，因此您無法直接在 [系統管理工具]  資料夾中建立新資料夾；您必須在別處建立新資料夾 (例如在 [桌面] 上)，然後將新資料夾複製到 [系統管理工具]  資料夾中。

2.  移動或複製*MyTools*要**控制台/系統及安全性/系統管理工具**。 根據預設，您必須是電腦上 Administrators 群組的成員才能對 [系統管理工具]  資料夾進行變更。

3.  如果您不需要限制使用者存取權限，自訂工具捷徑，，移至步驟 6。 否則，以滑鼠右鍵按一下工具檔案 (或 *MyTools* 資料夾)，然後按一下 [內容]。

4.  在上**安全性**索引標籤上的檔案**屬性**] 對話方塊中，按一下 [**編輯**。

5.  針對您要限制其工具存取的使用者，清除核取方塊**讀取 & 執行**，**讀取**，並**撰寫**權限。 這些權限會由 [系統管理工具] 資料夾中的工具捷徑繼承。

    如果當使用者使用伺服器管理員 時編輯使用者的存取權限 （或伺服器管理員 時開啟），則您的變更不會顯示**工具**直到使用者重新啟動伺服器管理員 功能表。

    > [!NOTE]
    > 如果您對複製到 系統管理工具的整個資料夾限制存取，受限制的使用者可以看到資料夾和其內容在 伺服器管理員都不**工具**功能表。
    > 
    > 編輯中資料夾的權限**系統管理工具**資料夾。 因為系統管理工具中隱藏的檔案和資料夾一律會顯示在 [伺服器管理員**工具**] 功能表中，請勿使用**Hidden**上的檔案或資料夾的設定**屬性**對話方塊來限制使用者存取自訂工具捷徑。
    > 
    > [拒絕] 權限一律覆寫 [允許] 權限。

6.  以滑鼠右鍵按一下原始工具、 指令碼或您要新增項目上的可執行檔**工具**功能表，然後再按一下**建立快顯**。

7.  移至捷徑*MyTools*系統管理工具中的資料夾。

8.  重新整理或重新啟動伺服器管理員，如有必要，請參閱中的自訂工具捷徑**工具**功能表。

## <a name="BKMK_roles"></a>管理角色首頁上的角色
您將伺服器新增到伺服器管理員的伺服器集區，以及伺服器管理員會收集有關集區中的清查資料之後，伺服器管理員會將頁面加入至受管理伺服器所發現角色瀏覽窗格中。 角色頁面上的 [伺服器] 磚會列出執行該角色的受管理伺服器。 根據預設，[事件] 、[最佳做法分析程式] 、[服務] 及 [效能]  磚會顯示執行該角色的所有伺服器資料；選取 [伺服器]  磚中的特定伺服器會讓事件、服務、效能計數器和 BPA 結果的範圍僅限於選取的伺服器。 管理工具時通常可在伺服器管理員主控台**工具** 功能表中，角色或功能已安裝或在受管理的伺服器上探索到之後。 您也可以用滑鼠右鍵按一下 [伺服器] 磚中某個角色或群組的伺服器項目，然後啟動您要使用的管理工具。

在 Windows Server 2016 中，下列角色和功能的管理工具已整合至伺服器管理員 主控台中的頁面。

-   **檔案和存放服務。** [檔案和存放服務] 頁面包含自訂磚和命令，可用來管理磁碟區、共用、iSCSI 虛擬磁碟及儲存集區。 當您開啟檔案和存放服務角色首頁中 伺服器管理員，一個撤銷窗格隨即開啟，顯示自訂的管理頁面檔案和存放服務。 如需部署與管理檔案和存放服務的詳細資訊，請參閱 [檔案和存放服務](https://go.microsoft.com/fwlink/p/?LinkId=241530)。

-   **遠端桌面服務。** [遠端桌面服務] 頁面包含自訂磚和命令，可用來管理工作階段、授權、閘道和虛擬桌面。 如需有關部署和管理遠端桌面服務的詳細資訊，請參閱 <<c0> [ 遠端桌面服務 (rdS)](https://go.microsoft.com/fwlink/p/?LinkId=241532)。

-   **IP 位址管理 (IPAM)。** IPAM 角色頁面包含自訂的 [歡迎使用] 磚，其中包含常用 IPAM 設定和管理工具的連結，包括佈建 IPAM 伺服器的精靈。 IPAM 首頁還包括檢視受管理網路、設定摘要及排程工作的磚。

    有一些限制 IPAM 管理在 [伺服器管理員] 中。 與一般角色和群組頁面不同，IPAM 沒有 [伺服器]、[事件]、[效能]、[最佳做法分析程式] 或 [服務] 磚。 沒有可供 IPAM; 使用的最佳做法分析程式模型不支援在 IPAM 上的進行最佳做法分析程式掃描。 若要存取伺服器集區中執行 IPAM 的伺服器，請建立這些執行 IPAM 之伺服器的自訂群組，然後從自訂群組頁面上的 [伺服器] 磚存取伺服器清單。 或者，從 [所有伺服器] 群組頁面的 [伺服器] 磚存取 IPAM 伺服器。

    與其他角色和群組的縮圖相比，儀表板縮圖只能顯示有限的 IPAM 列。 按一下 IPAM 縮圖列，您就可以檢視執行 IPAM 之伺服器的事件、效能資料及管理性狀態警示。 與 IPAM 相關的服務可以從包含 IPAM 伺服器的伺服器群組頁面加以管理，例如 [所有伺服器]  群組的頁面。

    如需有關部署與管理 IPAM 的詳細資訊，請參閱 < [IP 位址管理 (IPAM)](https://go.microsoft.com/fwlink/p/?LinkId=241533)。
