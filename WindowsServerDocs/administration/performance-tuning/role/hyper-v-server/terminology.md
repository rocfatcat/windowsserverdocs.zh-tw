# <a name="hyper-v-terminology"></a>Hyper-v 術語
本節將摘要說明整個效能微調主題所使用的虛擬機器技術特有的重要術語：

| 詞彙        | 定義           |
| ------------- |:------------|
|*子磁碟分割* | 根分割區所建立的任何虛擬機器。|
|*裝置虛擬化* | 一種機制，可讓您在多個取用者之間抽象化和共用硬體資源。|
|*模擬裝置*|模擬實際實體硬體裝置的虛擬化裝置，讓來賓可以使用該硬體裝置的一般驅動程式。|
|*啟示*|對客體作業系統的優化，讓它知道虛擬機器的環境，並調整其虛擬機器的行為。|
|*客人*|正在磁碟分割中執行的軟體。 它可以是功能完整的作業系統，也可以是小型的特殊用途核心。 虛擬程式管理與來賓無關。|
|*虛擬*|位於硬體和一或多個作業系統下的軟體層。 其主要工作是提供稱為磁碟分割的隔離執行環境。 每個分割區都有自己的一組虛擬化硬體資源 (中央處理器或 CPU、記憶體和裝置) 。 Hypervisor 可控制及仲裁對基礎硬體的存取權。|
|*邏輯處理器*| 處理一個執行執行緒 (指令資料流程) 的處理單位。 每個處理器核心可以有一或多個邏輯處理器，每個處理器通訊端可以有一或多個核心。|
| *傳遞磁片存取*|將整個實體磁片表示為來賓內的虛擬磁片。 資料和命令會透過根磁碟分割的原生儲存體堆疊) 傳遞至實體磁片 (，且虛擬堆疊不會進行任何中間處理。|
|*根磁碟分割*|第一個建立的根磁碟分割，並且擁有所有的程式管理器所沒有的資源，包括大部分的裝置和系統記憶體。 根分割區會裝載虛擬化堆疊，並建立及管理子分割。|
|*Hyper-v 專用裝置*|沒有實體硬體類比的虛擬化裝置，因此來賓可能需要驅動程式 (虛擬化服務用戶端) 到該 Hyper-v 特定的裝置。 此驅動程式可以使用虛擬機器匯流排 (VMBus) 與根磁碟分割中的虛擬化裝置軟體進行通訊。|
|*虛擬機器*|由軟體模擬所建立，且與實際電腦具有相同特性的虛擬電腦。|
| *虛擬網路交換器*| (也稱為虛擬交換器) 實體網路交換器的虛擬版本。 您可以將虛擬網路設定成可以存取一或多部虛擬機器的本機或外部網路資源。|
|*Virtual Processor - 虛擬處理器*|已排程在邏輯處理器上執行的處理器虛擬抽象概念。 虛擬機器可以有一或多個虛擬處理器。|
|*虛擬化服務用戶端 (VSC) *|來賓載入以取用資源或服務的軟體模組。 針對 i/o 裝置，虛擬化服務用戶端可以是作業系統核心載入的設備磁碟機。|
| *虛擬化服務提供者 (.VSP) *|  根分割區中的虛擬化堆疊所公開的提供者，可提供資源或服務，例如子磁碟分割的 i/o。|
| *虛擬化堆疊*|根磁碟分割中的軟體元件集合，可一起運作以支援虛擬機器。 虛擬化堆疊適用于和位於程式管理程式之上。 它也提供管理功能。|
|*VMBus*|以通道為基礎的通訊機制，用於在具有多個作用中虛擬化分割的系統上進行資料分割通訊和裝置列舉。 VMBus 會隨著 Hyper-V 整合服務一起安裝。|
