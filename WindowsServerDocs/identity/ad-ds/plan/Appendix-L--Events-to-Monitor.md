# <a name="appendix-l-events-to-monitor"></a>附錄 L：要監視的事件

>適用於：Windows Server

下表列出您應該在環境中監視的事件，並根據監視 Active Directory 中所提供的建議[來入侵徵兆](../../ad-ds/plan/security-best-practices/Monitoring-Active-Directory-for-Signs-of-Compromise.md)。 在下表中，[目前的 Windows 事件識別碼] 資料行會列出事件識別碼，因為它是在目前是主流支援的 Windows 和 Windows Server 版本中執行。

[舊版 Windows 事件識別碼] 欄會列出舊版 Windows 中的對應事件識別碼，例如執行 Windows XP 或更早版本的用戶端電腦，以及執行 Windows Server 2003 或更早版本的伺服器。 [潛在的重要性] 資料行會識別事件在偵測攻擊時是否應該被視為低、中或高重要性，而 [事件摘要] 資料行則會提供事件的簡短描述。

「高」的潛在重要性表示應該調查事件的一次。 「中」或「低」的潛在重要性表示這些事件應該只會在發生意外或在測量的一段時間內大幅超過預期基準的數位中進行調查。 所有組織應該在其環境中測試這些建議，再建立需要強制調查回應的警示。 每個環境都不同，而且某些事件的排名可能是因為其他無害的事件而導致潛在的高重要性。

如需 Windows 安全性事件識別碼及其意義的詳細資訊，請參閱 Microsoft 支援服務文章[windows 7 和 Windows Server 2008 R2 中的安全性事件說明](https://support.microsoft.com/kb/977519)。 您也可以下載[Windows 7 和 Windows server 2008 R2](https://www.microsoft.com/download/details.aspx?id=21561)和[Windows 8 和 Windows Server 2012 安全性事件詳細資料](https://www.microsoft.com/download/details.aspx?id=35753)的安全性 Audit 事件，其以試算表格式提供參考作業系統的詳細事件資訊。
