# <a name="determining-the-number-of-domains-required"></a>判斷網域所需的數目

>適用於：Windows Server 2016、Windows Server 2012 R2、Windows Server 2012

森林每開頭單一網域。 使用者可以包含單一網域樹系的上限根據必須符合您想要配置給 Active Directory Domain Services (AD DS) 頻寬網域控制站之間複製慢連結。 下表包含網域可以根據網域單一樹系的速度 [慢] 連結，以及您想要保留的複寫頻寬百分比的使用者人數，建議的最大值。 這項資訊，包含最多個 100000 使用者，有連接 28.8 以每秒 56 kbps 或更高的樹系適用於。 適用於包含超過 100000 使用者或連接小於 28.8 的樹系的建議，請洽詢經驗 Active Directory 設計師。 下表中的值為基礎複寫流量產生具有下列特性環境中：  
  
-   新的使用者加入樹系的每 20 年 %速率。  
  
-   使用者退出樹系速率為 15%每年。  
  
-   每個使用者是五個的全域群組和五個通用群組成員。  
  
-   電腦的使用者比例為 1:1。  
  
-   會使用 active Directory 整合網域名稱系統 (DNS)。  
  
-   使用 DNS 清除。  
  
> [!NOTE]  
> 下表中列出的數字會近似值。 複寫流量數量而定主要 directory 在一段指定時間的變更的數目。 確認您的網路，可容納您複寫流量藉由在實驗室測試的預估的數量和變更您的設計的之前將您的網域部署。

|28.8|10,000|25,000|40,000|  
|32|10,000|25,000|50,000|  
|56|10,000|50,000|100,000|  
|64|10,000|50,000|100,000|  
|128|25,000|100,000|100,000|  
|256|50,000|100,000|100,000|  
|512|80,000|100,000|100,000|  
|1,500|100,000|100,000|100,000|  
  
若要使用此表格：  
  
1.  在**最慢連接網域控制站的連結**欄中，找出符合的所有 AD DS 會複寫網域中的慢連結的速度。  
  
2.  在 [慢連結快速對應的列，找出代表您想要到 AD DS 配置百分比頻寬欄。 在該位置的值為使用者網域中單一網域可包含上限。  
  
如果您認為您森林中的使用者總數小於可包含您的網域中的使用者人數，您可以使用單一網域。 請務必容納規劃未來成長，當您進行此判斷。 如果您認為您森林中的使用者總數大於可包含您的網域中的使用者人數，您需要保留較高百分比的頻寬複寫，加快連結速度，或您組織區域網域進行除法運算。
