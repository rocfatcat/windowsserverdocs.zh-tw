# <a name="determining-the-number-of-domains-required"></a>決定所需的網域數

> 適用於：Windows Server 2016、Windows Server 2012 R2、Windows Server 2012

每個樹系的開頭都是單一網域。 單一網域樹系可以包含的最大使用者數目，是根據在網域控制站與您要配置給 Active Directory Domain Services (AD DS) 之間複寫所需的最慢連結。 下表列出根據單一網域樹系，網域可以包含的最大建議使用者數目、最慢連結的速度，以及您要保留複寫的頻寬百分比。 這項資訊適用于包含最多100000個使用者，且每秒有 28.8 kb 的連線 (Kbps) 或更高版本的樹系。 如需適用于包含超過100000個使用者或連接低於 28.8 Kbps 之樹系的建議，請參閱經驗豐富的 Active Directory 設計工具。 下表中的值是以具有下列特性的環境中產生的複寫流量為基礎：

- 新的使用者以每年20% 的速率加入樹系。
- 使用者以每年15% 的速率離開樹系。
- 每個使用者都是五個全域群組和五個萬用群組的成員。
- 使用者與電腦的比率為1:1。
- 使用 Active Directory 整合網域名稱系統 (DNS) 。
- 使用 DNS 清除。

> [!NOTE]
> 下表所列的圖形是近似值。 複寫流量的數量主要取決於在指定的一段時間內對目錄所做的變更數目。 在部署您的網域之前，先測試實驗室中設計的預估數量和變更率，以確認您的網路可以容納您的複寫流量。

|連接網域控制站 (Kbps 的最慢連結) |如果有1% 的頻寬可用，使用者的最大數目|有5% 頻寬可用時的使用者數目上限|如果有10% 的頻寬可用，使用者的最大數目|
| --- | --- | --- | --- |
|28.8|10,000|25,000|40,000|
|32|10,000|25,000|50,000|
|56|10,000|50,000|100,000|
|64|10,000|50,000|100,000|
|128|25,000|100,000|100,000|
|256|50,000|100,000|100,000|
|512|80,000|100,000|100,000|
|1,500|100,000|100,000|100,000|

若要使用此資料表：

1. 在 [**連接網域控制站的最慢連結**] 欄中，找出符合在網域中複寫 AD DS 的最慢連結速度的值。

2. 在對應至最慢連結速度的資料列中，找出代表您要配置給 AD DS 之百分比頻寬的資料行。 該位置的值是單一網域樹系中的網域可以包含的最大使用者數目。

如果您判斷樹系中的使用者總數小於網域可包含的最大使用者數目，您可以使用單一網域。 當您進行這項判斷時，請務必配合規劃後的未來成長。 如果您判斷樹系中的使用者總數大於網域可以包含的最大使用者數目，您必須保留較高的複寫頻寬百分比、增加連結速度，或將您的組織分成地區網域。

## <a name="dividing-the-organization-into-regional-domains"></a>將組織分成地區網域

如果您無法在單一網域中容納所有的使用者，則必須選取地區網域模型。 以適合您組織和現有網路的方式，將您的組織分成區域。 例如，您可以根據大陸界限來建立區域。

請注意，因為您必須為所建立的每個區域建立一個 Active Directory 網域，建議您將為 AD DS 定義的區域數目降至最低。 雖然樹系中可以包含不限數目的網域，但為了方便管理，我們建議樹系包含不超過10個網域。 將您的組織分成地區網域時，您必須在優化複寫頻寬和將系統管理複雜度降到最低之間建立適當的平衡。

首先，判斷您的樹系可以裝載的使用者數目上限。 以此為基礎，在網域控制站將複寫的樹系中最慢的連結，以及您要配置給 Active Directory 複寫的平均頻寬量。 下表列出樹系可以包含的最大建議使用者數目。 這是根據最慢的連結速度，以及您要為複寫保留的百分比頻寬。 這項資訊適用于包含最多100000個使用者，且其連線能力為 28.8 Kbps 或以上的樹系。 下表中的值是以下列假設為基礎：

- 所有網域控制站都是通用類別目錄伺服器。
- 新的使用者以每年20% 的速率加入樹系。
- 使用者以每年15% 的速率離開樹系。
- 使用者是五個全域群組和五個萬用群組的成員。
- 使用者與電腦的比率為1:1。
- 使用 Active Directory 整合式 DNS。
- 使用 DNS 清除。

> [!NOTE]
> 下表所列的圖形是近似值。 複寫流量的數量主要取決於在指定的一段時間內對目錄所做的變更數目。 在部署您的網域之前，先測試實驗室中設計的預估數量和變更率，以確認您的網路可以容納您的複寫流量。

|連接網域控制站 (Kbps 的最慢連結) |如果有1% 的頻寬可用，使用者的最大數目|有5% 頻寬可用時的使用者數目上限|如果有10% 的頻寬可用，使用者的最大數目|
| --- | --- | --- | --- |
|28.8|10,000|50,000|75000|
|32|10,000|50,000|75000|
|56|10,000|75000|100,000|
|64|25,000|75000|100,000|
|128|50,000|100,000|100,000|
|256|75000|100,000|100,000|
|512|100,000|100,000|100,000|
|1,500|100,000|100,000|100,000|

若要使用此資料表：

1. 在 [**連接網域控制站的最慢連結**] 資料行中，找出符合在樹系中複寫 AD DS 的最慢連結速度的值。

2. 在對應至最慢連結速度的資料列中，找出代表您要配置給 AD DS 之百分比頻寬的資料行。 該位置的值是您的樹系可以裝載的最大使用者數目。

如果您樹系可以裝載的使用者數目上限大於您需要裝載的使用者數目，則單一樹系適用于您的設計。 如果您需要裝載的使用者數目超過您識別的最大值，則需要增加最小連結速度、為 AD DS 配置較大的頻寬百分比，或部署其他樹系。

如果您判斷單一樹系將會容納您需要裝載的使用者數目，下一步就是根據位於該區域的最慢連結，判斷每個區域可支援的使用者數目上限。 將您的樹系劃分為對您有意義的區域。 請確定您是以您的決策為基礎，而不可能變更。 例如，使用大陸而不是銷售領域。 當您識別出使用者的最大數目時，這些區域就會是您的網域結構的基礎。

決定需要在每個區域中裝載的使用者數目，然後根據最慢的連結速度和配置給該區域中 AD DS 的頻寬，確認它們不會超過允許的上限。 下表列出區域網域可包含的最大建議使用者數目。 它是以最慢的連結速度，以及您想要保留以進行複寫的頻寬百分比為基礎。 這項資訊適用于包含最多100000個使用者，且其連線能力為 28.8 Kbps 或以上的樹系。 下表中的值是以下列假設為基礎：

- 所有網域控制站都是通用類別目錄伺服器。
- 新的使用者以每年20% 的速率加入樹系。
- 使用者以每年15% 的速率離開樹系。
- 使用者是五個全域群組和五個萬用群組的成員。
- 使用者與電腦的比率為1:1。
- 使用 Active Directory 整合式 DNS。
- 使用 DNS 清除。

> 下表所列的圖形是近似值。 複寫流量的數量主要取決於在指定的一段時間內對目錄所做的變更數目。 在部署您的網域之前，先測試實驗室中設計的預估數量和變更率，以確認您的網路可以容納您的複寫流量。

|連接網域控制站 (Kbps 的最慢連結) |如果有1% 的頻寬可用，使用者的最大數目|有5% 頻寬可用時的使用者數目上限|如果有10% 的頻寬可用，使用者的最大數目|

若要使用此資料表：

1. 在 [**連接網域控制站的最慢連結**] 欄中，找出符合在您區域中複寫 AD DS 的最慢連結速度的值。

2. 在對應至最慢連結速度的資料列中，找出代表您要配置給 AD DS 之百分比頻寬的資料行。 該值代表區域可以裝載的使用者數目上限。

評估每個提議的區域，並判斷每個區域中的使用者數目上限是否小於網域可包含的建議最大使用者數目。 如果您判斷該區域可以裝載您需要的使用者數目，您可以為該區域建立網域。 如果您判斷無法裝載這許多使用者，請考慮將您的設計畫分成較小的區域，然後重新計算可在每個區域中主控的最大使用者數目。 另一個替代方案是配置更多頻寬，或增加連結速度。

雖然您可以在多網域樹系的網域中放入的使用者總數，小於單一網域樹系中網域的使用者數目，多網域樹系中的使用者總數也可以更高。 多重網域樹系中每個網域的使用者數目較少，可配合在該環境中維護通用類別目錄所建立的額外複寫額外負荷。 如需適用于包含超過100000個使用者或連接低於 28.8 Kbps 之樹系的建議，請參閱經驗豐富的 Active Directory 設計工具。

## <a name="documenting-the-regions-identified"></a>記載識別的區域

將組織分成地區網域之後，請記錄您想要表示的區域，以及將存在於每個區域中的使用者數目。 此外，請記下您將用於 Active Directory 複寫之每個區域中最慢連結的速度。 此資訊可用來判斷是否需要其他網域或樹系。

如需協助您記載所識別區域的工作表，請從[Windows Server 2003 部署套件的工作輔助](https://microsoft.com/download/details.aspx?id=9608)下載 Job_Aids_Designing_and_Deploying_Directory_and_Security_Services.zip，並 ( # A1) 開啟「識別區域」。
