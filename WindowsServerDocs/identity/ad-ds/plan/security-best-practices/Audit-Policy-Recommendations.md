# <a name="audit-policy-recommendations"></a>稽核原則建議

>適用於：Windows Server 2016 中，Windows Server 2012 R2 中，Windows Server 2012、 Windows 10，Windows 8.1，Windows 7

本節說明 Windows 預設稽核原則設定，基準建議稽核原則設定，並更積極的建議從 Microsoft 取得，適用於工作站和伺服器產品。  

在這裡顯示，以及協助偵測入侵，我們建議您設定的 SCM 基準建議僅供系統管理員起始的基準指南。 每個組織必須決定所面臨的威脅; 其可接受的風險承受度中，以及哪些稽核原則類別目錄或子類別應該讓自己決定。 如需威脅的進一步資訊，請參閱[威脅與對策指南](https://technet.microsoft.com/library/hh125921(v=ws.10).aspx)。 是鼓勵開始，建議的設定而不需要仔細考量的稽核原則後的系統管理員，然後修改並測試之前在生產環境中實作。  

建議為企業級電腦，Microsoft 會定義為具有平均的安全性需求，而且需要高層級的操作功能的電腦。 實體需要較高的安全性需求應該考慮更積極的稽核原則。  

> [!NOTE]  
> Microsoft Windows 預設值，並從所進行的基準建議[Microsoft Security Compliance Manager 工具](https://technet.microsoft.com/library/cc677002.aspx)。

## <a name="set-audit-policy-on-workstations-and-servers"></a>工作站和伺服器上設定稽核原則  
工作站和伺服器，應該監視所有的事件記錄檔管理計劃。 常見的錯誤是只監視伺服器或網域控制站。 因為惡意駭客通常一開始就會發生在工作站上，將不會監視工作站忽略最佳與最早的來源資訊。  

系統管理員應該填寫完成檢閱，並在其生產環境中測試任何實作之前的稽核原則。  

## <a name="events-to-monitor"></a>要監視的事件  
若要產生安全性警示的完美的事件識別碼應該包含下列屬性：  

-   很有可能會發生表示未經授權的活動  

-   誤判數字低  

-   相符項目應導致調查/鑑識回應  

兩種類型的事件應該監視與警示：  

1.  這些事件其中甚至單一項目表示未經授權的活動  

2.  累積的事件高於預期和接受的基準  

第一個事件範例為：  

如果網域系統管理員 (DAs) 會禁止從登入不是網域控制站的電腦，登入的使用者工作站的資料成員的單一相符項目應產生警示，並調查。 這類警示是容易使用稽核特殊登入事件 4964 產生 （特殊群組已指派給新的登入）。 單一執行個體警示的其他範例包括：  

-   如果伺服器 A 應該永遠不會連線到伺服器 B，在彼此連接時的警示。  

-   如果一般使用者帳戶非預期地加入敏感的安全性群組，便會發出警示。  

-   如果在處理站位置的員工永遠不會用到了晚上，警示使用者登入時在午夜。  

-   如果網域控制站上安裝未經授權的服務，便會發出警示。  

-   如果一般使用者嘗試直接登入 SQL Server 還不清楚的理由，這項操作，請調查。  

-   如果您有任何成員資料群組中，而有人那里新增本身，請立即檢查。  

第二個事件的範例是：  

失敗的登入的 aberrant 數目可能表示密碼猜測攻擊。 提供警示的異常大量登入失敗的企業，他們必須先了解失敗的登入，在他們的環境，惡意的安全性事件之前的一般層級。  

當您監視遭到危害的徵兆時，您應包含的事件的完整清單，請參閱[附錄 l:要監視的事件](../../../ad-ds/plan/Appendix-L--Events-to-Monitor.md)。  

## <a name="active-directory-objects-and-attributes-to-monitor"></a>Active Directory 物件和屬性對監視  
以下是帳戶、 群組和可協助您偵測入侵您的 Active Directory 網域服務安裝嘗試，您應監視的屬性。  

-   系統停用或移除防毒和反惡意程式碼軟體 （會自動重新啟動保護時以手動方式停用）  

-   未經授權的變更的系統管理員帳戶  

-   所使用 （會自動移除帳戶時的可疑活動都完成或分配的時間已過期） 的特殊權限的帳戶執行的活動  

-   特殊權限和 VIP 帳戶在 AD DS 中。 監視的變更，尤其是 [帳戶] 索引標籤 （例如 cn、 名稱、 sAMAccountName、 userPrincipalName 或 userAccountControl） 屬性的變更。 除了監控帳戶，會限制誰可以修改帳戶，以較小一組系統管理使用者越好。  

請參閱[附錄 l:要監視的事件](../../../ad-ds/plan/Appendix-L--Events-to-Monitor.md)取得一份建議的事件，以監視、 其嚴重性分級和事件訊息摘要。  

-   依分類其工作負載，可讓您快速識別應該是最受監視且最嚴格設定的伺服器群組的伺服器  

-   屬性和下列 AD DS 群組的成員資格的變更：Enterprise Admins (EA)、 Domain Admins (DA)、 系統管理員 (BA) 和結構描述系統管理員 (SA)  

-   停用特殊權限的帳戶 （例如內建系統管理員帳戶在 Active Directory，並在成員的系統上） 啟用的帳戶  

-   記錄所有寫入帳戶的管理帳戶  

-   內建安全性設定精靈來設定服務、 登錄、 稽核及防火牆設定，以降低伺服器的受攻擊面。 如果您實作跳躍伺服器系統管理主機策略的一部分，請使用此精靈。

## <a name="general-list-of-security-event-id-recommendation-criticalities"></a>安全性事件識別碼建議嚴重性的一般清單  
事件識別碼的所有建議都附有重要性，評等，如下所示：  

**高：** 高嚴重性等級的事件識別碼應該一律並立即收到警示並調查。  

**中型：** 中度重要性等級事件識別碼可能表示惡意活動，但它必須伴隨著某些其他到異常 (例如，在一段特定時間、 未預期的相符項目或在電腦上的項目中發生不尋常號碼，通常會不預期記錄事件。）。 中度重要性事件也可以做為計量收集和比較經過一段時間 r。  

**低：** 與事件識別碼為低重要性事件不應在獲得注意，或會產生警示，除非使用中或高危險程度事件相互關聯。  

這些建議旨在提供系統管理員的基準指南。 在生產環境中實作之前，應該徹底檢閱所有建議。  

請參閱[附錄 l:要監視的事件](../../../ad-ds/plan/Appendix-L--Events-to-Monitor.md)取得一份建議的事件，以監視、 其嚴重性分級和事件訊息摘要。
