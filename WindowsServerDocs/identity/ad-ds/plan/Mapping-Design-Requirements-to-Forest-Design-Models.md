# <a name="mapping-design-requirements-to-forest-design-models"></a>森林設計模型對應設計需求

>適用於：Windows Server 2016、Windows Server 2012 R2、Windows Server 2012

您在組織中的大部分群組可以分享單一組織的樹系由群組單一資訊 (IT) 技術和包含帳號及分享樹系的群組的所有的資源。 此分享的樹系，稱為初始組織樹系是組織的樹系設計模式的基本知識。  
  
初始組織的樹系可以裝載組織中的多個群組，因為樹系擁有者，所有當事人了都解他們的預期必須建立服務層級合約每個群組。 這可透過建立服務同意入預期保護樹系擁有者和個人的群組。  
  
如果不是在組織中的群組的所有可以共用單一組織樹系，您必須展開樹系設計容納不同群組的需求。 這牽涉到找出的設計需求適用於群組依據他們的需求的自主隔離及它們有連接有限的網路，然後找出您可以使用容納那些需求的樹系模型。 下表列出根據自主性、 隔離，以及連接因素樹系設計模型案例。 您找出最符合您需求的樹系設計案例之後，判斷您需要做出任何其他符合您的設計規格。

## <a name="BKMK_1"></a>案例 1： 加入現有的樹系的資料自主性  
您可以只要裝載組織單位 (Ou) 在現有的樹系組織中的群組符合資料自主性的需求。 委派 Ou 控制資料的系統管理員，從達成資料自主性該群組。 如需有關如何使用 Ou 委派控制的詳細資訊，請[建立組織單位設計](../../ad-ds/plan/Creating-an-Organizational-Unit-Design.md)。  
  
## <a name="BKMK_2"></a>案例 2： 使用組織的樹系或網域服務自主  
如果您在組織中的群組辨識服務自主為需求，我們建議您第一次重新考慮此需求。 實現服務自主建立多個管理負荷和組織的額外成本。 確定服務自主需求不只是為了方便，您可以以致於無法平衡會議此需求的相關費用。  
  
您可以符合服務自主需求，執行下列其中一個動作：  
  
-   建立組織的樹系。 將使用者、 群組和需要服務自主到不同的組織樹系的群組的電腦。 指派個人的樹系擁有者該群組。 如果需要與在組織中其他樹系存取或共用資源群組，他們可以建立他們組織的樹系和其他的樹系之間信任。  
  
-   使用組織網域。 使用者、 群組和電腦放在現有的組織樹系不同的網域。 此模型網域層級服務自主性只提供，並不是完整服務自主權服務隔離或隔離的資料。  
  
如需有關如何使用網域組織的詳細資訊，請查看[使用組織網域森林模型](../../ad-ds/plan/../../ad-ds/plan/Using-the-Organizational-Domain-Forest-Model.md)。  
  
## <a name="BKMK_3"></a>案例 3： 使用服務隔離的樹系組織或是資源  
您可以符合服務隔離的需求，執行下列其中一個動作：  
  
-   使用組織的樹系。 將使用者、 群組和需要服務到不同的組織樹系的隔離群組的電腦。 指派個人的樹系擁有者該群組。 如果需要與在組織中其他樹系存取或共用資源群組，他們可以建立他們組織的樹系和其他的樹系之間信任。 不過，我們不建議這種方式因為經驗透過通用群組資源存取信任的樹系案例中限制。  
  
-   使用資源樹系。 資源與服務帳號放不同的資源樹系帳號保留現有的組織森林中。 必要時，可以建立替代帳號資源樹系存取資源森林中的資源，如果組織的樹系無法使用。 替代帳號必須登入資源樹系和維護資源的控制項組織的樹系恢復之前所需的授權。  
  
    建立信任之間的資源和樹系組織，讓使用者可以使用一般的使用者的帳號存取森林中的資源。 這項設定可讓使用者帳號的集中的管理，同時讓使用者改為使用資源樹系替代帳號如果組織的樹系無法使用。  
  
服務隔離的注意事項如下：  
  
-   建立服務隔離的樹系可以信任的其他森林網域，但不是能包含他們的服務的系統管理員群組中其他樹系的使用者。 如果隔離森林中的系統管理群組中有其他的樹系的使用者，安全性隔離的樹系可能可以受到因為服務中的系統管理員樹系不能排除控制。  
  
-   只要網域控制站在網路上的存取，是受到攻擊 （例如阻服務攻擊） 來自該網路上的惡意軟體。 您可以執行下列命令以防止攻擊：  
  
    -   只有在被認為是安全網路上的主機網域控制站。  
  
    -   存取權的網路或網路裝載的網域控制站限制。  
  
-   服務隔離需要額外的樹系建立。 評估是否維護基礎結構，以支援其他樹系的成本超過遺失資源因無法使用組織樹系的存取權的相關費用。  
  
## <a name="BKMK_4"></a>案例 4： 使用資料隔離的樹系組織或是限制的存取權  
您可以隔離的資料，執行下列其中一個動作：  
  
-   使用組織的樹系。 將使用者、 群組和需要資料到不同的組織樹系的隔離群組的電腦。 指派個人的樹系擁有者該群組。 如果需要與在組織中其他樹系存取或共用資源群組，建立信任的樹系組織和其他的樹系。 新的組織樹系存在於只需要機密資訊的存取權的使用者。 使用者必須帳號他們自己的樹系存取這兩個歸類資料和其他的樹系未分類的資料使用透過信任關係的方式。  
  
-   使用限制的存取樹系。 這是不同的樹系包含限制的資料和使用者帳號，用來存取該資料。 另外帳號會保留現有的組織樹系用於存取網路上的無限制的資源。 不信任會建立有限的存取樹系和其他企業中的樹系。 您可以在使其無法連接到其他樹系進一步限制樹系部署獨立實體網路上的樹系。 如果您部署不同的網路上的樹系，使用者必須具有兩個工作站： 一個用於限制的樹系，一個用於存取 nonrestricted 的區域網路的存取。  
  
建立的資料隔離的樹系的注意事項如下：  
  
-   組織的資料隔離建立的樹系可以信任網域其他樹系，但其他樹系使用者必須不會包含在下列其中一項：  
  
    -   負責管理服務或群組，可以管理的服務管理員群組成員資格群組  
  
    -   已儲存的電腦系統控制群組受保護的資料  
  
    -   群組，存取受保護的資料或群組負責管理物件使用者或群組物件存取受保護的資料  
  
    如果另一部樹系的使用者參與了個群組，危害的樹系可能會導致隔離的樹系危害和公開受保護資料。  
  
-   其他的樹系可以信任的樹系建立資料獨立的隔離森林中的使用者可以存取其他森林中的資源，組織設定。 不過，隔離的樹系的使用者必須從未互動方式登入信任的樹系工作站。 信任的樹系在的電腦可能會受到惡意軟體，可以用來擷取的使用者登入認證。  
  
    > [!NOTE]  
    > 若要防止信任的樹系的伺服器模擬使用者從隔離的樹系，並且再存取隔離森林中的資源，樹系擁有者可以停用委派的驗證，或使用功能有限的委派。 如需有關委派的驗證限制的委派，查看委派驗證 ([https://go.microsoft.com/fwlink/?LinkId=106614](https://go.microsoft.com/fwlink/?LinkId=106614))。  
  
-   您可能需要建立組織的樹系和其他的樹系防火牆組織限制使用者以外的樹系資訊的存取。  
  
-   雖然建立不同的樹系讓資料隔離，，只要網域控制站的隔離的樹系和電腦存取網路上的受保護的主機資訊，都受遭受網路上的電腦。 選擇攻擊的風險得太高，或攻擊或安全性違反的結果是也變得更好的組織必須限制的存取權的網路或裝載的網域控制站的網路，並裝載電腦的受保護資料。 利用技術，例如防火牆 」 和 「 網際網路通訊協定的安全性 (IPsec) 可以完成限制的存取。 萬一極端，組織可能選擇維護組織中已不實體連接到任何其他網路獨立網路上的受保護的資料。  
  
    > [!NOTE]  
    > 如果任何網路連接限制的存取樹系和其他網路，可能會有被傳輸到其他網路限制區域中的資料。  
  
## <a name="BKMK_5"></a>案例 5： 使用組織的樹系，或重新設定限制連接防火牆  
若要符合有限的連接需求，您可以執行下列其中一個動作：  
  
-   放現有組織樹系的使用者，然後打開防火牆達到允許 Active Directory 流量通過。  
  
-   使用組織的樹系。 另外組織樹系放使用者、 群組和電腦連接的有限的群組。 指派個人的樹系擁有者該群組。 組織的樹系防火牆另一邊提供不同的環境。 樹系包括帳號及管理樹系的的資源，讓使用者透過防火牆完成日常工作，不需要。 特定的使用者或應用程式可能需要的功能通過連絡其他的樹系防火牆特殊需求。 您可以在防火牆，包括所需的任何信任運作打開適當的介面，排列處理這些需求。  
  
設定使用防火牆 Active Directory Domain Services (AD DS) 的相關詳細資訊，會看到網路分段防火牆在 Active Directory ([https://go.microsoft.com/fwlink/?LinkId=37928](https://go.microsoft.com/fwlink/?LinkId=37928))。  
  
## <a name="BKMK_6"></a>案例 6： 使用網域或組織的樹系和重新設定限制連接的服務自主防火牆  
如果您在組織中的群組辨識服務自主為需求，我們建議您第一次重新考慮此需求。 實現服務自主建立多個管理負荷和組織的額外成本。 確定服務自主需求不只是為了方便，您可以以致於無法平衡會議此需求的相關費用。  
  
如果有限的連接時會發生的問題，您有適用於服務自主需求，您可以執行下列其中一個動作：  
  
-   使用組織的樹系。 將使用者、 群組和需要服務自主到不同的組織樹系的群組的電腦。 指派個人的樹系擁有者該群組。 組織的樹系防火牆另一邊提供不同的環境。 樹系包括帳號及管理樹系的的資源，讓使用者透過防火牆完成日常工作，不需要。 特定的使用者或應用程式可能需要的功能通過連絡其他的樹系防火牆特殊需求。 您可以在防火牆，包括所需的任何信任運作打開適當的介面，排列處理這些需求。  
  
-   使用者、 群組和電腦放在現有的組織樹系不同的網域。 此模型網域層級服務自主性只提供，並不是完整服務自主權服務隔離或隔離的資料。 森林中的其他群組必須信任新的網域服務系統的管理員相同程度他們信任的樹系擁有者。 基於這個原因，我們不建議這種方式。 如需有關如何使用網域組織的詳細資訊，請查看[使用組織網域森林模型](../../ad-ds/plan/../../ad-ds/plan/Using-the-Organizational-Domain-Forest-Model.md)。  
  
您也需要開放防火牆達到允許 Active Directory 流量通過。 設定使用防火牆 AD DS 的相關詳細資訊，會看到網路分段防火牆在 Active Directory ([https://go.microsoft.com/fwlink/?LinkId=37928](https://go.microsoft.com/fwlink/?LinkId=37928))。
