# <a name="mapping-design-requirements-to-forest-design-models"></a>對應至樹系設計模型的設計需求

>適用於：Windows Server 2016 中，Windows Server 2012 R2 中，Windows Server 2012

大部分組織中的群組可以共用單一組織的樹系所管理的單一資訊科技 (IT) 小組，並包含使用者帳戶和所有共用的樹系群組的資源。 此共用的樹系，稱為初始組織的樹系中，是組織的樹系設計模型的基礎。  

因為初始組織的樹系可以裝載多個組織中的群組，樹系擁有者必須建立服務等級協定，與每個群組，使所有合作對象了解什麼預期。 這會保護個別的群組和樹系擁有者，藉由建立依雙方協定服務的期望。  

如果不是所有組織中的群組可以共用單一組織的樹系，您必須展開樹系設計來容納不同群組的需求。 這項作業包括識別套用至群組根據自治與隔離和它們有連線能力有限的網路，其需求的設計需求，然後找出您可以使用來容納這些樹系模型需求。 下表列出樹系設計模型案例自主性、 隔離和連線能力的因素為基礎。 找出最符合您需求的樹系設計案例後，判斷您是否需要進行任何額外的決策，以符合您的設計規格。

## <a name="BKMK_1"></a>案例 1:加入現有的樹系的 「 資料自治 」  

您可以只要裝載現有的組織樹系中的組織單位 (Ou) 中的群組符合資料自主性需求。 委派給資料的系統管理員的控制權的 Ou，以達到 「 資料自治 」 該群組。 如需有關使用 Ou 來委派控制的詳細資訊，請參閱[建立組織單位設計](../../ad-ds/plan/Creating-an-Organizational-Unit-Design.md)。  
  
## <a name="BKMK_2"></a>案例 2:用於服務自治 」 中的組織的樹系或網域  

如果您的組織中的群組識別為需求的服務自治 」，我們建議您先考慮這項需求。 達到服務自治 」，會建立更多的管理額外負荷和組織的額外成本。 請確定服務自治 」 的需求不只是為了方便起見，您可以左右對齊滿足此需求所牽涉的成本。  
  
您可以執行下列其中一項符合服務自治 」 的需求：  

- 建立組織的樹系。 將使用者、 群組和群組到另一個組織的樹系需要服務自治 」 的電腦。 從該群組的樹系擁有者指派個人。 如果需要與組織中其他樹系的存取權或共用資源群組，他們可以建立其組織的樹系與其他樹系之間的信任。  

- 使用組織的網域。 將使用者、 群組和電腦放在現有的組織樹系中不同的網域中。 此模型只網域層級服務自治 」 提供，而且不適用於完整的服務自主性服務隔離或資料隔離。  

如需使用組織網域的詳細資訊，請參閱[使用組織的網域樹系模型](../../ad-ds/plan/../../ad-ds/plan/Using-the-Organizational-Domain-Forest-Model.md)。  

## <a name="BKMK_3"></a>案例 3:服務隔離使用組織的樹系或資源樹系  

您可以執行下列其中一項符合服務隔離的需求：  

- 使用組織的樹系。 將使用者、 群組和群組所需服務隔離到個別組織的樹系的電腦。 從該群組的樹系擁有者指派個人。 如果需要與組織中其他樹系的存取權或共用資源群組，他們可以建立其組織的樹系與其他樹系之間的信任。 不過，我們不建議這種方法因為透過通用群組的資源存取高度限制在樹系信任案例中。  

- 使用資源樹系。 將資源和服務帳戶放入個別的資源樹系中，將使用者帳戶保留在現有的組織樹系中。 如有必要，可以建立其他帳戶資源樹系，來存取資源樹系中的資源，如果無法使用組織的樹系中。 替代的帳戶必須具有登入資源樹系和維護控制的資源，直到組織的樹系已回復連線所需的授權單位。  

   讓使用者可以存取時使用其規則的使用者帳戶的樹系中的資源，請建立資源與組織的樹系之間的信任。 此設定可讓使用者帳戶的集中式的管理，同時允許使用者將改為使用替代帳戶資源樹系中如果組織的樹系會變成無法使用。  

服務隔離的考量包括下列各項：

- 建立的服務隔離的樹系可以信任來自其他樹系的網域，但必須在其服務系統管理員群組中包含來自其他樹系的使用者。 如果在隔離的樹系中的系統管理群組中包含來自其他樹系的使用者，則隔離的樹系的安全性可能被入侵因為樹系中的服務系統管理員並沒有獨佔控制。  

- 只要在網路上存取的網域控制站，會受限於攻擊 （例如阻絕服務攻擊） 從該網路上的惡意軟體。 您可以執行下列命令以防止攻擊的可能性：  

   - 只在被視為安全的網路上的主機網域控制站。  

   - 限制存取權的網路或裝載網域控制站的網路。  

- 服務隔離需要額外的樹系建立。 評估是否維護基礎結構來支援其他樹系的成本超過失去存取權，因為無法使用組織的樹系的資源相關聯的成本。  

## <a name="BKMK_4"></a>案例 4:使用組織的樹系或限制的存取樹系進行資料隔離  

您可以執行下列其中一項，以達到資料隔離：  

- 使用組織的樹系。 將使用者、 群組和群組所需的資料隔離到個別組織的樹系的電腦。 從該群組的樹系擁有者指派個人。 如果需要與組織中其他樹系的存取權或共用資源群組，請建立組織的樹系與其他樹系之間的信任。 只有需要存取機密資訊的使用者存在於新的組織樹系中。 使用者會有一個帳戶，用以存取這兩個分類在自己的樹系中的資料和其他樹系中的非保密的資料透過信任關係。  

- 使用限制的存取樹系。 這是包含受限制的資料，以及用來存取該資料的使用者帳戶的個別樹系。 個別使用者帳戶會保留現有組織樹系，用來存取網路上不受限制的資源。 限制的存取樹系與企業中的其他樹系之間，會不產生任何信任。 您可以在使它無法連線到其他樹系進一步限制樹系部署個別的實體網路上的樹系。 如果您部署個別的網路上的樹系時，使用者必須具有兩個工作站： 一個用於存取受限制的樹系，另一個用於存取的網路 nonrestricted 的區域。  

建立資料隔離的樹系考量包括下列各項：  

- 組織建立的資料隔離的樹系可以信任來自其他樹系、 網域，但來自其他樹系的使用者必須不會包含在下列其中一項：  

   - 負責服務管理或群組可以管理的服務系統管理員群組成員資格的群組  

   - 已儲存受保護的資料的電腦的系統管理控制權的群組  

   - 群組可存取受保護的資料或群組所負責的使用者物件或群組物件具有存取權管理受保護的資料  

   如果任何這些群組中包含來自其他樹系的使用者，則於隔離的樹系的洩露和受保護資料揭露，可能會導致另一個樹系的洩露。  

- 可以設定其他樹系信任組織的樹系建立的資料隔離，因此在隔離的樹系中的使用者可以存取其他樹系中的資源。 不過，從隔離的樹系的使用者必須永遠不會以互動方式登入到信任的樹系中的工作站。 信任的樹系中的電腦可能遭到惡意軟體，而且可用來擷取使用者的登入認證。  

   > [!NOTE]
   > 若要防止在信任的樹系中的伺服器模擬使用者隔離的樹系，並接著存取 隔離的樹系中的資源，樹系擁有者可以停用委派的驗證，或使用限制的委派功能。 如需有關委派的驗證和限制的委派的詳細資訊，請參閱 <<c0> [ 委派驗證](https://go.microsoft.com/fwlink/?LinkId=106614)。  

- 您可能需要建立組織的樹系和其他樹系之間有防火牆來限制對其樹系之外的資訊的使用者存取組織中。  

- 雖然建立個別的樹系可讓資料隔離，只要在隔離的樹系和電腦的網域控制站的受保護的主機資訊會在網路上存取，他們並從該網路上的電腦啟動的攻擊。 決定攻擊的風險會太高或攻擊或安全性違規的後果是太大的組織必須限制存取網路或裝載網域控制站的網路和電腦裝載之受保護的資料. 限制存取可使用的技術，例如防火牆和網際網路通訊協定安全性 (IPsec)。 在極端的情況下，組織可能會選擇維護組織中有不到任何其他網路的實體連線的獨立網路上受保護的資料。  

   > [!NOTE]  
   > 如果限制的存取樹系和其他網路之間不存在的任何網路連線，可能會有受限制區域傳輸到其他網路中的資料。  

## <a name="BKMK_5"></a>案例 5:使用組織的樹系，或重新設定連線能力有限的防火牆  

若要符合限制的連線能力需求時，您可以執行下列其中一項：  

- 將使用者放入現有的組織樹系，然後再開啟 防火牆足以允許 Active Directory 流量通過。  

- 使用組織的樹系。 將使用者、 群組和群組會受限於與連線能力的電腦放入個別組織的樹系中。 從該群組的樹系擁有者指派個人。 組織的樹系的另一端的防火牆提供不同的環境。 樹系包含以使用者帳戶和管理的樹系內的資源，因此使用者不需要通過防火牆，以完成其每日工作。 特定使用者或應用程式可能需要通過防火牆，請連絡其他樹系功能的特殊需求。 您可以個別定址這些需求，包括所需的任何函式的信任的防火牆中開啟適當的介面。  

如需與 Active Directory 網域服務 (AD DS) 設定使用的防火牆的詳細資訊，請參閱[防火牆分段之網路中的 Active Directory](https://go.microsoft.com/fwlink/?LinkId=37928)。  

## <a name="BKMK_6"></a>案例 6:使用組織的樹系或網域，並重新設定連線能力有限的服務自治 」 的防火牆  

如果您的組織中的群組識別為需求的服務自治 」，我們建議您先考慮這項需求。 達到服務自治 」，會建立更多的管理額外負荷和組織的額外成本。 請確定服務自治 」 的需求不只是為了方便起見，您可以左右對齊滿足此需求所牽涉的成本。  

如果連線能力有限的問題，而且您有服務自治 」 的需求，您可以執行下列其中一項：  

- 使用組織的樹系。 將使用者、 群組和群組到另一個組織的樹系需要服務自治 」 的電腦。 從該群組的樹系擁有者指派個人。 組織的樹系的另一端的防火牆提供不同的環境。 樹系包含以使用者帳戶和管理的樹系內的資源，因此使用者不需要通過防火牆，以完成其每日工作。 特定使用者或應用程式可能需要通過防火牆，請連絡其他樹系功能的特殊需求。 您可以個別定址這些需求，包括所需的任何函式的信任的防火牆中開啟適當的介面。  

- 將使用者、 群組和電腦放在現有的組織樹系中不同的網域中。 此模型只網域層級服務自治 」 提供，而且不適用於完整的服務自主性服務隔離或資料隔離。 樹系中的其他群組都必須信任他們信任的樹系擁有者相同程度的新網域的服務系統管理員。 基於這個理由，我們不建議這種方法。 如需使用組織網域的詳細資訊，請參閱[使用組織的網域樹系模型](../../ad-ds/plan/../../ad-ds/plan/Using-the-Organizational-Domain-Forest-Model.md)。  

您也需要開啟防火牆足以允許 Active Directory 流量通過。 如需設定防火牆供與 AD DS 的詳細資訊，請參閱[防火牆分段之網路中的 Active Directory](https://go.microsoft.com/fwlink/?LinkId=37928)。  

## <a name="BKMK_7"></a>案例 7:使用資源樹系，並重新設定連線能力有限的服務隔離的防火牆  

如果連線能力有限的問題，而且您有服務隔離的需求，您可以執行下列其中一項：  

- 使用組織的樹系。 將使用者、 群組和群組所需服務隔離到個別組織的樹系的電腦。 從該群組的樹系擁有者指派個人。 組織的樹系的另一端的防火牆提供不同的環境。 樹系包含以使用者帳戶和管理的樹系內的資源，因此使用者不需要通過防火牆，以完成其每日工作。 特定使用者或應用程式可能需要通過防火牆，請連絡其他樹系功能的特殊需求。 您可以個別定址這些需求，包括所需的任何函式的信任的防火牆中開啟適當的介面。  

- 使用資源樹系。 將資源和服務帳戶放入個別的資源樹系中，將使用者帳戶保留在現有的組織樹系中。 它可能需要建立一些替代的使用者帳戶來維護資源樹系存取權，如果組織的樹系會變成無法使用的資源樹系中。 替代的帳戶必須具有登入資源樹系和維護控制的資源，直到組織的樹系已回復連線所需的授權單位。  

   讓使用者可以存取時使用其規則的使用者帳戶的樹系中的資源，請建立資源與組織的樹系之間的信任。 此設定可讓使用者帳戶的集中式的管理，同時允許使用者將改為使用替代帳戶資源樹系中如果組織的樹系會變成無法使用。

- 建立的服務隔離的樹系可以信任來自其他樹系的網域，但必須在其服務系統管理員群組中包含來自其他樹系的使用者。 如果在隔離的樹系中的系統管理群組中包含來自其他樹系的使用者，則隔離的樹系的安全性可能被入侵因為樹系中的服務系統管理員並沒有獨佔控制。
