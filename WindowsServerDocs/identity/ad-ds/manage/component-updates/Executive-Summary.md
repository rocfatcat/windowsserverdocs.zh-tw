# <a name="executive-summary"></a>執行摘要

>適用於：Windows Server 2012

>[!IMPORTANT] 
>下列文件寫入 2013 年中，且只提供僅供歷史。  目前我們正在查看這份文件，就如有變更。  它可能不會反映目前的最佳做法。

不組織的基礎結構的資訊 (IT) 技術會受到攻擊，但如果適當的原則、處理程序，以及控制保護組織運算基礎結構金鑰區段實作時，可能會防止違約事件批發危害運算環境。  
  
這個執行摘要旨在也會大減摘要的文件，其中包含有助於組織中美化的其 Active Directory 安裝安全性建議 content 獨立文件。 利用這些建議組織將能以找出和優先順序安全性活動、保護組織的計算基礎結構金鑰區段並建立大幅降低成功攻擊 IT 環境的關鍵元件的可能性的控制項。  
  
本文討論 Active Directory 和措施，以減少攻擊最常見的攻擊，但它也包含復原完成危害發生的建議。 Active Directory 完成危害時修復只確定方式是前危害的準備。  
  
本文件的主要部分如下：  
  
-   途徑危害  
  
-   減少 Active Directory 攻擊  
  
-   Active Directory 監視危害的符號  
  
-   規劃區域的入侵  
  
## <a name="avenues-to-compromise"></a>途徑危害  
本節中提供一些最常 leveraged 弱點攻擊用於危害針對的基礎結構的資訊。 它包含一般分類的弱點，以及他們最初侵入針對的基礎結構，額外的系統上傳播危害和最後為目標，以取得完整控制權組織的樹系的 Active Directory 和網域控制站的使用方式。 不提供詳細位址各種弱點，尤其是在區域中的弱點不用來直接目標 Active Directory 中相關的建議。 不過，適用於各種弱點，我們也提供使用開發措施，並減少組織的攻擊 surface 所需的詳細資訊的連結。

-   **初始違約目標**的最資訊的安全漏洞開始組織的基礎結構通常一或兩個系統的小型危害一次。 這些初始活動或進入點網路，通常弱點可能會有修正，但無法。 常見的安全漏洞︰

    -   錯誤設定  
  
    -   缺少的安全的應用程式開發做法  
  
-   **認證竊取吸引帳號**-認證竊取攻擊是那些中攻擊最初取得授權的存取，來在網路上的電腦，然後使用免費工具解壓縮認證從其他登入帳號的工作階段。   
    包含在本區段中，如下所示：

## <a name="reducing-the-active-directory-attack-surface"></a>減少 Active Directory 攻擊  
本章節焦某技術控制項，以減少 Active Directory 安裝的攻擊。 此一節中所遇到與下列主題：  
  
-   **特殊權限帳號，並 Active Directory 中的群組**區段討論的最高的特殊權限的帳號和群組的機制特殊權限的帳號受保護的 Active Directory 中。 在 Active Directory 建的三個群組雖然也要保護的帳號，以及其他群組是 directory（企業系統管理員，網域系統管理員」或系統管理員，）中的最高的權限群組。  
  
-   **實作最低權限管理型號**區段著重於找出所使用的高特殊權限的風險帳號的日常的系統管理禮品，此外提供建議，可減少風險。  
  
太多權限不只 Active Directory 中找到危害的環境中。 當您組織所開發超過需要更多的權限授與習慣時，通常是透過基礎結構找到：  
  
-   在 Active Directory  
  
-   伺服器成員  
  
-   在 [工作站  
  
-   在應用程式  
  
-   在 [資料存放庫  
  
-   **實作安全管理主機**區段描述安全管理主機、電腦設定為支援的 Active Directory 和連接的系統管理。 這些主機致力於管理功能，而且不會執行軟體，例如電子郵件應用程式、網頁瀏覽器或生產力軟體（例如 Microsoft Office)。  
  
包含在本區段中，如下所示：  
  
-   **原則建立安全的系統管理主機的**-記住一般原則是：  
  
    -   永遠不會從低信任主機管理信任的系統。  
  
    -   不要在單一驗證規格上執行的活動特殊權限時。  
  
    -   不要忘記實體安全性時設計和實作安全管理主機。  
  
-   **保護針對攻擊網域控制站**的使用者如果惡意的使用者會取得的網域控制站存取權限，可以修改、損壞，以及破壞 Active Directory 資料庫，和延伸的所有受 Active Directory 帳號和系統。  
  
此一節中所遇到與下列主題：  
  
-   **實體安全性網域控制站的**-包含提供實體安全性網域控制站的資料中心、分公司和遠端位置的建議。  
  
-   **網域控制站作業系統**-包含保護網域控制站作業系統的建議。  
  
-   **保護設定的網域控制站**的原生和免費設定工具和設定可用於建立的安全性設定基準網域控制站的後續由群組原則物件 (Gpo)。

## <a name="planning-for-compromise"></a>規劃區域的入侵  
本節建議，可協助組織準備危害前、實作控制項，可偵測入侵事件之前發生的完整漏洞，和回應和復原指導方針，提供的 directory 完成危害透過攻擊。 此一節中所遇到與下列主題：  
  
-   **重新思考方法**-包含原則和建立的組織可以將其最重要資產安全環境指導方針。 下列指導方針操作方式如下：  
  
    -   檢測軍人分離和設定資產重要的安全性原則  
  
    -   定義有限、風險根據移轉計劃  
  
    -   必要時運用」nonmigratory「移轉  
  
    -   實作」創意破壞」  
  
    -   隔離舊版系統和應用程式  
  
    -   簡化終端使用者的安全性  
  
-   **維護更安全環境**-包含適用於對短片指導方針與用來使用開發不僅有效的安全性，而且生效週期管理高階建議。 此一節中所遇到與下列主題：  
  
    -   **Active Directory 建立商務中心安全性做法**-要有效管理的使用者，資料、應用程式和系統由 Active Directory 週期，請遵循這些原則。  
  
        -   **Active Directory 資料指派公司擁有**-指派基礎結構元件的擁有權 IT;資料，以支援商務，例如，新員工、新應用程式和新資訊存放庫中的 Active Directory Domain Services (AD DS) 中新增了指定的營業或使用者應該相關聯的資料。  
  
        -   **實作 Business-Driven 週期管理**-週期管理應 Active Directory 中的資料。  
  
        -   **可所有 Active Directory 資料**-企業擁有者應該提供分類的 Active Directory 中的資料。 資料分類模式，當中應該包含分類的 Active Directory 下列資料：  
  
            -   **系統**-分類伺服器擴展，其作業系統自己的角色，以及其上的應用程式和記錄的公司擁有者。  
  
            -   **應用程式**-分類應用程式的功能、使用者基底，以及他們作業系統。  
  
            -   **使用者**-中最常攻擊者會對應的 Active Directory 安裝帳號應該標記及監視。
