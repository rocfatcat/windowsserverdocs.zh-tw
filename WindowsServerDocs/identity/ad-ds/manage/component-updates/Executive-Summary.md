# <a name="executive-summary"></a>執行摘要

>適用於：Windows Server 2012

>[!IMPORTANT] 
>下列檔是以2013撰寫，而且僅供歷程記錄之用。  我們目前正在審查此檔，但仍有可能變更。  它可能不會反映目前的最佳作法。

不具資訊技術（IT）基礎結構的組織不會遭受攻擊，但如果適當的原則、程式和控制項是為了保護組織運算基礎結構的重要區段而實行，則可能會防止缺口事件成長到計算環境的大量入侵。  
  
本「執行摘要」的目的是要做為檔內容摘要的獨立檔，其中包含的建議可協助組織加強其 Active Directory 安裝的安全性。 藉由實施這些建議，組織將能夠識別安全性活動並設定其優先順序、保護其組織運算基礎結構的重要區段，並建立可大幅降低其可能性的控制項對 IT 環境的重要元件進行成功的攻擊。  
  
雖然這份檔討論的是對 Active Directory 的最常見攻擊，以及減少受攻擊面的對策，但它也包含在完全洩露的情況下，復原的建議。 唯一能在 Active Directory 危害的情況下復原的方法，是在發生問題之前，先準備好進行入侵。  
  
本檔的主要章節包括：  
  
-   危及系統安全的途徑  
  
-   減少 Active Directory 的攻擊面  
  
-   監視 Active Directory 遭到危害的徵兆  
  
-   規劃危害因應措施  
  
## <a name="avenues-to-compromise"></a>危及系統安全的途徑  
本節提供攻擊者用來危害客戶基礎結構的一些最常見弱點的相關資訊。 其中包含弱點的一般類別，以及它們如何用來一開始滲透客戶的基礎結構、在其他系統之間傳播危害，最後以 Active Directory 和網域控制站為目標，以取得完整的控制組織的樹系。 它不會提供有關解決每種弱點類型的詳細建議，特別是在未使用弱點直接以 Active Directory 為目標的區域中。 不過，針對每種類型的弱點，我們提供了其他資訊的連結，可用來開發對策和減少組織的受攻擊面。  
  
其中包含下列主題：  
  
-   **初始漏洞目標**-大部分資訊安全性缺口會從組織基礎結構的一小部分（通常是一或兩個系統）洩露。 這些初始事件或進入網路的進入點，通常會利用可能已修正但卻無法解決的弱點。 常見的弱點包括：  
  
    -   防毒軟體和反惡意程式碼部署的間距  
  
    -   不完整的修補  
  
    -   過期的應用程式和作業系統  
  
    -   設定錯誤  
  
    -   缺乏安全的應用程式開發實務  
  
-   **認證竊取的吸引**人-認證竊取攻擊是攻擊者一開始取得網路上電腦的特殊許可權存取權，然後使用自由可用的工具，從其他的會話解壓縮認證已登入的帳戶。   
    本節包含下列內容：  
  
    -   **增加危害**機率的活動-因為認證竊取的目標通常是具有高許可權的網域帳戶和「非常重要的人員」（VIP）帳戶，因此系統管理員很重視活動這會增加認證竊取攻擊成功的可能性。 這些活動包括：  
  
        -   使用具有特殊許可權的帳戶登入不安全的電腦  
  
        -   使用高許可權帳戶流覽網際網路  
  
        -   以相同的認證在系統間設定本機特殊許可權帳戶  
  
        -   Overpopulation 和濫用特殊許可權網域群組  
  
        -   管理網域控制站的安全性不足。  
  
    -   **許可權提升和傳播**特有的帳戶、伺服器和基礎結構元件通常是針對 Active Directory 攻擊的主要目標。 這些帳戶包括：  
  
        -   永久的特殊許可權帳戶  
  
        -   VIP 帳戶  
  
        -   「附加許可權」 Active Directory 帳戶  
  
        -   網域控制站  
  
        -   其他會影響身分識別、存取和設定管理的基礎結構服務，例如公開金鑰基礎結構（PKI）伺服器和 systems management server  
  
## <a name="reducing-the-active-directory-attack-surface"></a>減少 Active Directory 的攻擊面  
本節著重于技術控制項，以減少 Active Directory 安裝的受攻擊面。 本章節包含下列主題：  
  
-   Active Directory 一節中的特殊**許可權帳戶和群組**會討論 Active Directory 中最高許可權的帳戶和群組，以及用來保護特殊許可權帳戶的機制。 在 Active Directory 中，有三個內建組是目錄中的最高許可權群組（Enterprise Admins、Domain Admins 和 Administrators），不過也應該保護一些額外的群組和帳戶。  
  
-   「**執行最小許可權的系統管理模型**」一節著重于找出使用高許可權帳戶進行日常管理的風險，以及提供建議來降低風險。  
  
在遭入侵的環境中，不能在 Active Directory 中找到過多的許可權。 當組織已開發出授與所需更多許可權的習慣時，通常會在整個基礎結構中找到：  
  
-   在 Active Directory  
  
-   在成員伺服器上  
  
-   在工作站上  
  
-   在應用程式中  
  
-   在資料存放庫中  
  
-   「正在**執行安全的系統管理主機**」一節說明安全的系統管理主機，也就是設定為支援 Active Directory 和連線系統管理的電腦。 這些主機專門用於系統管理功能，而且不會執行電子郵件應用程式、網頁瀏覽器或生產力軟體（例如 Microsoft Office）等軟體。  
  
本節包含下列內容：  
  
-   **建立安全的系統管理主機的原則**-要牢記在心的一般原則如下：  
  
    -   永遠不要從較不受信任的主機管理受信任的系統。  
  
    -   執行特殊許可權活動時，請勿依賴單一驗證因素。  
  
    -   設計和執行安全的系統管理主機時，請不要忘記實體安全性。  
  
-   **保護網域控制站免于遭受攻擊**-如果惡意使用者取得網域控制站的特殊許可權存取權，該使用者可以修改、損毀和終結 Active Directory 資料庫，以及延伸模組，管理的所有系統和帳戶依 Active Directory。  
  
本章節包含下列主題：  
  
-   **網域控制站的實體安全性**-包含為資料中心、分公司和遠端位置的網域控制站提供實體安全性的建議。  
  
-   **網域控制站作業系統**-包含保護網域控制站作業系統安全的建議。  
  
-   **安全設定網域控制站**：原生和自由可用的設定工具和設定，可用來建立網域控制站的安全性設定基準，之後可以由群組原則物件（gpo）來強制執行。  
  
## <a name="monitoring-active-directory-for-signs-of-compromise"></a>監視 Active Directory 遭到危害的徵兆  
本節提供舊版審查分類和稽核原則子類別的相關資訊（在 Windows Vista 和 Windows Server 2008 中引進），以及 Advanced Audit Policy （這是在 Windows Server 2008 R2 中引進）。 此外，也提供要監視之事件和物件的相關資訊，這可能表示嘗試危害環境，以及一些其他參考，可用來為 Active Directory 建立完整的稽核原則。  
  
本章節包含下列主題：  
  
-   **Windows 稽核原則**-windows 安全性事件記錄檔具有分類和子類別，可決定要追蹤和記錄哪些安全性事件。  
  
-   **稽核原則建議**-本節說明 Windows 預設稽核原則設定、Microsoft 建議的稽核原則設定，以及更積極的建議，讓組織用來審核重要的伺服器和台.  
  
## <a name="planning-for-compromise"></a>規劃危害因應措施  
本節包含的建議可協助組織在發生損毀之前進行入侵，並在發生完全缺口之前，先執行可偵測入侵事件的控制項，並針對下列情況提供回應和復原指導方針：攻擊者會完成完整的目錄入侵。 本章節包含下列主題：  
  
-   **重新思考方法**-包含原則和指導方針，以建立組織可在其中放置最重要資產的安全環境。 這些指導方針如下：  
  
    -   識別用來隔離和保護重要資產的原則  
  
    -   定義以風險為基礎的有限遷移計畫  
  
    -   視需要運用「nonmigratory」遷移  
  
    -   實行「創意損毀」  
  
    -   隔離舊版系統和應用程式  
  
    -   簡化終端使用者的安全性  
  
-   **維護更安全的環境**-包含高階建議，目的是要用來做為開發原則，而不只是有效的安全性，而是有效的生命週期管理。 本章節包含下列主題：  
  
    -   **建立 Active Directory 的商業中心安全性實務**，以有效管理由 Active Directory 管理的使用者、資料、應用程式和系統的生命週期，請遵循這些原則。  
  
        -   **將商務擁有權指派給 Active Directory 資料**-將基礎結構元件的擁有權指派給它;針對新增至 Active Directory Domain Services （AD DS）以支援企業的資料（例如，新的員工、新的應用程式和新的資訊儲存機制），指定的業務單位或使用者應該與資料相關聯。  
  
        -   **實行商業導向的生命週期管理**-應針對 Active Directory 中的資料來實行生命週期管理。  
  
        -   **分類所有 Active Directory 資料**-商務擁有者應該為 Active Directory 中的資料提供分類。 在資料分類模型中，應包含下列 Active Directory 資料的分類：  
  
            -   **系統**-分類伺服器群體、其作業系統的角色、在其上執行的應用程式，以及記錄的 IT 和商務擁有者。  
  
            -   **應用程式**-依功能、使用者基底和其作業系統來分類應用程式。  
  
            -   **使用者**-Active Directory 安裝中最可能被攻擊者設為目標的帳戶，應加以標記和監視。  
  
## <a name="summary-of-best-practices-for-securing-active-directory-domain-services"></a>保護 Active Directory Domain Services 的最佳做法摘要  
下表提供本檔中提供的建議摘要，以確保 AD DS 安裝的安全。 某些最佳作法本質上是策略性的，而且需要完整的規劃和執行專案;其他則是策略性，著重于 Active Directory 和相關基礎結構的特定元件。  
  
做法會依照優先順序的大致順序列出，亦即，較低的數位表示較高的優先順序。 在適用的情況下，最佳作法會識別為預防性或偵探的本質。 所有這些建議都應該根據貴組織的特性和需求，進行完整的測試和修改。  
  
  
||**最佳做法**|**戰術或策略性**|**預防性或偵探**|  
|-|-|-|-|  
|1|修補應用程式。|戰術|預防性|  
|2|修補作業系統。|戰術|預防性|  
|3|部署並立即更新所有系統上的防毒軟體和反惡意程式碼軟體，並監視是否嘗試移除或停用它。|戰術|兩者|  
|4|監視敏感的 Active Directory 物件以進行修改嘗試，並針對可能表示嘗試洩露的事件進行 Windows。|戰術|偵探|  
|5|針對可存取敏感性資料的使用者，保護和監視帳戶|戰術|兩者|  
|6|防止未經授權的系統使用強大的帳戶。|戰術|預防性|  
|7|排除高許可權群組中的永久成員資格。|戰術|預防性|  
|8|在需要時，執行控制項以授與特殊許可權群組中的暫時成員資格。|戰術|預防性|  
|9|執行安全的系統管理主機。|戰術|預防性|  
|10|在網域控制站、管理主機和其他機密系統上使用應用程式允許清單。|戰術|預防性|  
|11|識別重要資產，並設定其安全性和監視的優先順序。|戰術|兩者|  
|12|執行最低許可權、以角色為基礎的存取控制來管理目錄、其支援的基礎結構，以及已加入網域的系統。|策略|預防性|  
|13|找出舊版的系統和應用程式。|戰術|預防性|  
|14|解除委任舊版系統和應用程式。|策略|預防性|  
|15|為自訂應用程式實施安全的開發生命週期程式。|策略|預防性|  
|16|執行設定管理、定期審查合規性，並使用每個新的硬體或軟體版本來評估設定。|策略|預防性|  
|17|使用嚴格的安全性和監視需求，將重要資產遷移至初始化樹系。|策略|兩者|  
|18|簡化終端使用者的安全性。|策略|預防性|  
|19|使用主機型防火牆來控制和保護通訊的安全。|戰術|預防性|  
|20|修補裝置。|戰術|預防性|  
|21|為 IT 資產實行以商務為中心的生命週期管理。|策略|N/A|  
|22|建立或更新事件復原計畫。|策略|N/A|
