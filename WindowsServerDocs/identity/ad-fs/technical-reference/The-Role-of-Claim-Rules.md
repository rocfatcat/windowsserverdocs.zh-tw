# <a name="the-role-of-claim-rules"></a>宣告規則的角色
在 Active Directory 同盟服務 \(AD FS\) 同盟服務的整體功能是發行包含一組宣告預付碼。 AD FS 接受並再問題所宣告相關受到理賠要求規則。  
  
## <a name="what-are-claim-rules"></a>有哪些規則理賠要求？  
宣告規則表示商務邏輯操作，將需要一或多個傳入宣告套用條件的執行個體 \ （如果 x 然後 y\） 和產生依據條件的參數的一或多個出的宣告。 如需傳入的和傳出宣告，請查看[宣告角色](The-Role-of-Claims.md)。  
  
當您需要執行企業邏輯，可以將控制宣告管線索賠項目時，您可以使用理賠要求規則。 更多的邏輯概念宣告管線時 end\ to\ 高階處理程序適用於這一條宣告，理賠要求規則的實際管理項目，您可以使用來自訂流程宣告透過宣告發行處理程序。  
  
如需宣告管線的詳細資訊，請查看[的角色宣告引擎的](The-Role-of-the-Claims-Engine.md)。  
  
宣告規則提供下列優點：  
  
-   提供機制套用企業 run\ 階段邏輯信任宣告宣告提供者的系統管理員  
  
-   提供機制來定義所宣告推出給信賴的系統管理員  
  
-   想要允許或拒絕存取特定使用者的系統管理員提供豐富且詳細 claims\ 為基礎的授權的功能  
  
### <a name="how-claim-rules-are-processed"></a>宣告規則的處理方式  
宣告規則處理宣告管線使用*宣告引擎*。 宣告引擎是連入宣告出示使用者的設定會檢查同盟服務的元件邏輯，而且會然後，根據每個規則邏輯操作，產生輸出集索賠項目。  
  
在一起，宣告規則引擎和設定的相關聯的特定聯盟信任規則判斷是否連入宣告應通過他們、 符合特定的條件篩選掉或之前同盟服務發行為傳出宣告轉換成索賠項目全新的一組理賠要求。  
  
如需有關這個程序，請查看[的角色宣告引擎的](The-Role-of-the-Claims-Engine.md)。

### <a name="how-to-create-a-claim-rule"></a>如何建立理賠要求規則  
理賠要求規則同盟服務中每個聯盟的信任關係分開建立並不會在多個信任共用。 您可以從理賠要求規則範本建立規則、 從頭開始製作宣告規則語言的使用規則或使用 Windows PowerShell 來自訂規則。

#### <a name="using-claim-rule-templates"></a>使用理賠要求規則範本  
取得範本只能理賠要求規則的建立程序期間規則。 您可以使用下列範本任一建立理賠要求規則：  
  
-   通過或篩選連入宣告  
  
-   轉換輸入宣告
