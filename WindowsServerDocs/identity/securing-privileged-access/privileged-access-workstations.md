    1.  **確保所有系統管理員使用不同、個人帳號管理和使用者活動的**（包括電子郵件、瀏覽網際網路、業務的應用程式，與其他非系統管理員活動）。  每個授權的人員不同指派管理帳號，從他們標準使用者 account 是基本爪模型，登入本身爪允許只有特定帳號。

        > [!NOTE]
        > 每個管理員應該使用自己 account 進行管理。  不要共用管理 account。

    2.  **最小化層 0 特殊權限管理員數量**。  每個系統管理員必須使用爪，因為減少的系統管理員可以減少的腳印來支援這些及相關的成本。 計數較低的系統管理員也會導致低公開這些權限，以及相關的風險。 雖然您可以在同一個位置分享爪的系統管理員，在不同的所在位置的系統管理員會需要腳印不同。

    3.  **取得的受信任的供應商符合所有的技術需求硬體**。 Microsoft 建議取得硬體符合技術文件中的[保護網域認證 Credential Guard 的](https://technet.microsoft.com/en-us/library/mt483740%28v=vs.85%29.aspx)。

        > [!NOTE]
        > 爪安裝，而這些功能的硬體上可以提供重大保護，但無法使用進階的安全性功能，例如 Credential Guard 和 Device Guard。  Credential Guard 及 Device Guard 並不需要階段 1 部署，但建議階段 3（進階強化）的一部分。

        請確定用於爪硬體源自製造商及供應商信任組織的安全性做法的規範。 這是提供鏈結安全性應用程式的全新來源原則。

        > [!NOTE]
        > 更多的供應鏈結安全性重要性背景，請造訪[這個網站](https://www.microsoft.com/security/cybersecurity/)。

#### <a name="phase-3-extend-and-enhance-protection"></a>階段 3：擴充和美化保護
範圍：這些保護美化階段 1 進階功能，包括多因素驗證和網路存取規則 bolstering 基本保護中之後建置套件。

> [!NOTE]
> 1 階段完成之後隨時可執行這個階段。  它不受完成第 2 階段的因此可以再執行，同時，或之後第 2 階段。

請依照下列步驟來設定此階段：

1.  讓特殊權限帳號多因素驗證。  多因素驗證強化 account 安全性要求的使用者提供的認證除了實體預付碼。  多因素驗證補充驗證原則非常運作，但它不需要驗證原則部署（，驗證原則同樣地，不需要多因素驗證）。  Microsoft 建議使用這些多因素驗證的格式之一：

    -   **智慧卡**：智慧卡是竄改及可移植實體裝置提供 Windows 登入程序期間的第二個驗證。  藉由要求個人擁有卡登入，您可以減少遭竊認證遠端重複使用的風險。  適用於在 Windows 中的智慧卡登入的詳細資訊，請指向文章[智慧卡概觀](https://technet.microsoft.com/en-us/library/hh831433.aspx)。

    -   **Virtual 智慧卡**: virtual 智慧卡提供相同的安全性優點為實體智慧卡，與好處，就的連結到特定的硬體。  如需部署和硬體需求的詳細資訊，請參考文章，[Virtual 智慧卡概觀](https://technet.microsoft.com/en-us/library/dn593708.aspx)和[開始使用 Virtual 智慧卡：逐步解說指南](https://technet.microsoft.com/en-us/library/dn579260.aspx)。

    -   **Microsoft Passport**: Microsoft Passport 讓使用者驗證 Microsoft account、Active Directory 帳號，Microsoft Azure Active Directory (Azure AD) 帳號，或非 Microsoft 的支援（fast ring) ID Online (FIDO) 驗證服務。 Microsoft Passport 註冊期間初始雙步驟驗證] 之後, Microsoft Passport 設定使用者的裝置上，並使用者設定手勢，可為 Windows Hello 或 pin 碼。 Microsoft Passport 認證的非對稱式的金鑰，可以產生隔離的受信任平台模組 (Tpm) 的環境中。
        Microsoft Passport 的詳細資訊朗讀的[Microsoft Passport 概觀](https://technet.microsoft.com/en-us/library/dn985839%28v=vs.85%29.aspx)文章。

    -   **Azure 多因素驗證**: Azure 多因素驗證 (MFA) 提供的第二個驗證比例，以及透過監視和機器學習根據分析美化的保護安全性。  Azure MFA 可以安全 Azure 系統管理員不僅許多其他方案，包括 web 應用程式、Azure Active Directory 及像遠端存取和遠端桌面先方案。  如需有關 Azure 多因素驗證的資訊，請參考文章[多因素驗證](https://azure.microsoft.com/en-us/services/multi-factor-authentication)。

2.  **份受信任的應用程式使用 Device Guard 和/或 AppLocker**。  藉由限制爪執行未受信任或簽署的程式碼的能力，進一步降低惡意活動與危害的可能性。  Windows 會包含應用程式控制項主要有兩個選項：

    -   **AppLocker**: AppLocker 幫助系統管理員控制哪些應用程式可以在特定的系統上執行。  AppLocker 集中群組原則、透過控制，適用於特定的使用者或群組（適用於特定對象的應用程式的使用者腳印）。  如需有關 AppLocker 的詳細資訊，請參考 TechNet 文章[AppLocker 概觀](https://technet.microsoft.com/library/hh831440.aspx)。

    -   **Device Guard**：新的 Device Guard 功能影響的裝置上提供的與 AppLocker，不會覆寫美化的硬體應用程式控制項。  AppLocker，例如 Device Guard 可以控制透過群組原則和特定使用者的目標。  如需有關限制使用 Device Guard 的應用程式使用方式的詳細資訊，請參考 TechNet 文章中，[裝置 Guard 部署指南](https://technet.microsoft.com/en-us/library/mt463091(v=vs.85).aspx)。

3.  **使用保護使用者、驗證原則和驗證筒倉進一步保護特殊權限的帳號**。  受保護的憑證額外的安全性原則儲存在本機安全性代理程式所以無法受保護的使用者的成員，並大幅認證竊取及重複使用風險降到最低。  驗證原則和筒倉控制如何特殊權限的使用者可以存取網域中的資源。  共同，這些保護大幅加強 account 的安全性這些權限的使用者。  這些功能的其他詳細資料，請參考網站文章[設定保護帳號如何](https://technet.microsoft.com/en-us/library/dn518179.aspx)。

    > [!NOTE]
    > 這些保護是適用於對短片補充、不取代現有安全性措施階段 1。  系統管理員仍然應該使用不同帳號管理與一般使用。
