### <a name="bkmk_channel"></a>指定無線 AP 的通道頻率

當您部署在單一地理站台的多個無線 Ap 時，您必須設定有重疊的訊號，來使用唯一的通道頻率減少干擾無線 Ap 的無線 Ap。

您可以使用下列指導方針可協助您選擇與您的無線網路的地理位置的其他無線網路沒有衝突的通道頻率。

- 如果沒有其他組織中有辦公室，十分接近，或在您的組織為同一個建築物，找出是否有任何這些組織所擁有的無線網路。 找出位置和指定的通道頻率其無線 AP 的因為您需要將不同的通道頻率指派給您的 AP，您需要決定最佳的位置，以安裝您的 AP

- 識別重疊在貴組織內的相鄰樓層的無線訊號。 識別重疊的涵蓋範圍地區以外，且您組織內為您的無線 Ap 指派通道頻率之後，如此可確保任何兩個重疊的涵蓋範圍的無線 Ap 會指派不同的通道頻率。

### <a name="bkmk_wirelessaps"></a>設定無線 Ap

您可以使用以及無線 AP 製造商所提供的產品文件的下列資訊來設定您的無線 Ap。

此程序會列舉常在無線 AP 上設定的項目。 項目名稱的廠牌和型號而有所不同，而且可能會不同於下面清單中。 特定的詳細資訊，請參閱您的無線 AP 文件。

#### <a name="to-configure-your-wireless-aps"></a>若要設定您的無線 Ap  

- **SSID**。 指定無線網路的名稱\(s\) \(比方說，ExampleWLAN\)。 這是通知給無線用戶端的名稱。

- **加密**。 指定 WPA2\-企業\(慣用\)或 WPA\-Enterprise 和其中一個 AES\(慣用\)或 TKIP 加密編碼器，依據版本會受到您無線用戶端電腦網路介面卡。

- **無線 AP 的 IP 位址\(靜態\)** 。 在每個 AP，設定唯一的靜態 IP 位址子網路的 DHCP 領域排除範圍。 使用排除由 DHCP 指派的位址，可防止 DHCP 伺服器將相同的 IP 位址指派給電腦或其他裝置。

- **子網路遮罩**。 設定這個值以符合您已連線的無線 AP 的 LAN 子網路遮罩設定。  

- **DNS 名稱**。 可以設定一些無線 Ap，使用 DNS 名稱。 網路上的 DNS 服務可以解析為 IP 位址的 DNS 名稱。 在每個支援此功能的無線 AP，輸入 DNS 解析的唯一名稱。  

- **DHCP 服務**。 如果您的無線 AP 有內建\-在 DHCP 服務，可以停用。  

- **RADIUS 共用的密碼**。 除非您打算將 Ap 設定為 RADIUS 用戶端，在 NPS 中的群組，則您可以用於每個無線 AP 唯一的 RADIUS 共用的密碼。 如果您打算在 NPS 中設定群組的 APs，共用的密碼必須是相同群組的每個成員。 此外，您使用的每一個共用的密碼應該至少 22 混用大寫的字元和小寫字母、 數字以及標點符號隨機序列。 若要確保隨機性，您可以使用隨機字元產生器中，例如在 NPS 中找到的隨機字元產生器**設定 802.1x**精靈，以建立共用的密碼。

>[!TIP]
>記錄每一個無線 AP 的共用的密碼，並將它儲存在安全的位置，例如 office 安全。 當您在 NPS 中設定 RADIUS 用戶端時，您便必須知道每一個無線 AP 的共用的密碼。  

- **RADIUS 伺服器 IP 位址**。 輸入執行 NPS 的伺服器的 IP 位址。

- **UDP 連接埠\(s\)** 。 根據預設，NPS 會使用 UDP 連接埠 1812年與 1645年驗證訊息和 UDP 連接埠 1813年與 1646 用於帳戶處理訊息。 建議您在您的 Ap，使用這些相同的 UDP 連接埠，但如果您有合理的原因，若要使用不同的連接埠，確保您不只使用新的連接埠號碼設定 Ap，但是也重新設定所有您要使用相同的連接埠號碼作為 APs NPSs。 如果 APs 和 NPSs，具有相同的 UDP 連接埠未設定，NPS 無法接收或處理連線要求從 Ap，並在網路上的所有無線連線嘗試會失敗。

- **Vsa**。 一些無線 Ap 需要廠商\-特定屬性\(Vsa\)來提供完整的無線 AP 的功能。 在 NPS 網路原則中新增 Vsa。

- **DHCP 篩選**。 設定無線 Ap，以封鎖 UDP 連接埠 68 傳送 IP 封包到網路，無線用戶端，由無線 AP 廠商所述。

- **DNS 篩選**。 設定無線 Ap，以封鎖從 TCP 或 UDP 連接埠 53 傳送 IP 封包到網路，無線用戶端，由無線 AP 廠商所述。

### <a name="bkmk_groups"></a>建立無線使用者的安全性群組

您可以使用此程序來建立無線網路安全性的群組中的 Active Directory 使用者和電腦 Microsoft Management Console \(MMC\)貼齊\-中。  

若要執行此程序，至少需要 **Domain Admins** 的成員資格或同等權限。

#### <a name="to-create-a-wireless-users-security-group"></a>若要建立無線使用者的安全性群組

1. 依序按一下 [開始]  、[系統管理工具]  及 [Active Directory 使用者和電腦]  。 Active Directory 使用者和電腦嵌入式管理單元\-中隨即開啟。 如果尚未選取，請按一下您的網域節點。 例如，如果您的網域 example.com，請按一下**example.com**。

2. 在 詳細資料 窗格中，以滑鼠右鍵\-按一下您要在其中加入新群組的資料夾\(比方說，以滑鼠右鍵\-按一下 **使用者**\)，指向**新增**，然後按一下**群組**。

3. 在 [新增物件 - 群組]  的 [群組名稱]  ，輸入新群組的名稱。 例如，輸入**無線群組**。

4. 在 **群組領域**，選取下列選項之一：

    - **網域本機**

    - **全域**

    - **標準**

5. 在 [群組類型]  中，選取 [安全性]  。

6. 按一下 [確定]  。

如果您為無線使用者需要一個以上的安全性群組，重複上述步驟來建立其他的無線使用者群組。 稍後您可以建立個別的網路原則，在 NPS 中將不同的狀況和條件約束套用至每個群組，並提供不同的存取權限和連線規則。

### <a name="bkmk_addusers"></a> 將使用者新增至無線使用者的安全性群組

您可以使用此程序，將使用者、 電腦或群組新增至您無線網路安全性的群組中的 Active Directory 使用者和電腦 Microsoft Management Console \(MMC\)貼齊\-中。

若要執行此程序，至少需要 **Domain Admins** 的成員資格或同等權限。

#### <a name="to-add-users-to-the-wireless-security-group"></a>若要將使用者新增至無線網路安全性的群組

1. 依序按一下 [開始]  、[系統管理工具]  及 [Active Directory 使用者和電腦]  。 這樣會開啟 [Active Directory 使用者和電腦] MMC。 如果尚未選取，請按一下您的網域節點。 例如，如果您的網域 example.com，請按一下**example.com**。

2. 在 [詳細資料] 窗格中，按兩下\-按一下包含您無線網路安全性的群組的資料夾。

3. 在 [詳細資料] 窗格中，以滑鼠右鍵\-按一下無線的安全性群組，然後再按一下**屬性**。 **屬性**安全性群組 對話方塊隨即開啟。

4. 在 **成員**索引標籤上，按一下**新增**，然後完成下列程序，請將電腦新增，或新增使用者或群組的其中一個。

##### <a name="to-add-a-user-or-group"></a>若要新增使用者或群組

1. 在 **輸入要選取的物件名稱**，輸入使用者名稱或群組，您想来新增，然後再按一下**確定**。

2. 若要將群組成員資格指派給其他使用者或群組，請重複此程序步驟 1。  

##### <a name="to-add-a-computer"></a>若要新增電腦

2. 在 **物件類型**，選取**電腦**，然後按一下**確定**。

3. 在 **輸入要選取的物件名稱**，輸入您要新增，然後按一下 電腦名稱**確定**。

4. 若要將群組成員資格指派給其他電腦，請重複步驟 1\-3 這個程序。

##### <a name="to-configure-a-wireless-connection-profile-for-peap-ms-chap-v2"></a>若要設定的無線連線設定檔，對 PEAP\-MS\-MS-CHAP v2

1. 在 GPME，在您剛剛建立，在原則的無線網路內容 對話方塊**一般** 索引標籤並在**描述**，輸入原則的簡短描述。

3. 在 **連線到可用的網路，按照下面所列的設定檔順序**，按一下**新增**，然後選取**基礎結構**。 **新的設定檔屬性**對話方塊隨即開啟。

4. 在 **新的設定檔屬性**對話方塊的 **連線**索引標籤**設定檔名稱**欄位中，輸入設定檔的新名稱。 例如，輸入**Example.com WLAN 設定檔適用於 Windows 10**。

5. 在 **網路名稱\(s\) \(SSID\)** ，輸入您的無線 Ap 上設定之 SSID 相對應的 SSID，然後按一下**新增**。

    如果您的部署使用多個 SSID，且每個無線 AP 都使用相同的無線安全性設定，請重複此步驟來為您要套用此設定檔的每個無線 AP 新增 SSID。

    如果您的部署使用多個 SSID，但每個 SSID 的安全性設定不相符，請為使用相同安全性設定的每個 SSID 群組設定個別的設定檔。 例如，如果您有一組無線 Ap 設定為使用 WPA2\-Enterprise 與 AES，而另一組無線 Ap，以使用 WPA\-企業與 TKIP，則在每一組無線 Ap 設定設定檔。

6. 如果預設的文字**NEWSSID**是呈現、 選取它，然後按一下**移除**。

7. 如果您部署設定為抑制廣播指標的無線存取點，請選取 [即使網路未廣播，還是進行連線]  。

    > [!NOTE]
    > 啟用此選項可建立有安全性風險，因為無線用戶端將探測並嘗試連線到任何無線網路。 預設不會啟用此設定。  

8. 按一下 [安全性]  索引標籤，按一下 [進階]  ，然後設定下列設定：

    1. 若要設定進階 802.1X 設定，請在 [IEEE 802.1X]  中選取 [強制執行進階 802.1X 設定]  。

        當進階的 802.1x 設定會強制執行的預設值**Max Eapol\-啟動 Msgs**，**保留週期**，**開始週期**，和**驗證期間**都足以用於一般的無線部署。 基於這個原因，您不需要變更預設值，除非您有這種方式的特定原因。

    2. 若要啟用「單一登入」，請選取 [對此網路啟用單一登入]  。

    3. [單一登入]  中其餘的預設值足以用於一般的無線部署。

    4. 在 **快速漫遊**如果無線 AP 設定為預先\-驗證，請選取**此網路使用預先\-驗證**。

    > - EAP 識別隱私權由特定的 EAP 方法，其中是空白或匿名身分識別\(不同於實際的身分識別\)在回應 EAP 識別要求時傳送。 PEAP 兩次驗證期間傳送身分識別。 在第一個階段中，身分識別以純文字傳送，以及此身分識別用於路由的使用，不適用於用戶端驗證。 真實的身分識別，用於驗證 — 傳送的驗證，在第一個階段中建立安全通道中的第二個階段。 如果**啟用識別隱私權**選取核取方塊，在文字方塊中指定的項目會取代使用者名稱。 例如，假設**啟用識別隱私權**已選取和身分識別隱私權別名**匿名**文字方塊中指定。 使用者的真實身分識別別名<strong>jdoe@example.com</strong>、 傳送的驗證的第一個階段中的身分識別就會變更為<strong>anonymous@example.com</strong>。 因為使用它來進行路由，就不會修改第 1 階段身分識別的領域部分。  

20. 按一下 [ **[確定]** 以關閉**保護的 EAP 內容**] 對話方塊。

## <a name="bkmk_domain"></a>將新的無線電腦加入網域
將新的無線電腦加入網域的最簡單方法是實際將電腦連接到有線 LAN 的區段\(區段，不受控制的 802.1 X 交換機\)之前先將電腦加入網域。 這是最簡單的因為無線群組原則設定會自動並立即套用，而且，如果您已部署您自己的 PKI，電腦會接收的 CA 憑證，並將它放在受信任的根憑證授權單位憑證存放區中，允許使用您的 CA 所發出的伺服器憑證信任 NPSs 無線用戶端。

同樣地，在新的無線電腦已加入網域之後，使用者登入加入網域的慣用的方法是使用網路的有線的連線，在執行記錄檔。

### <a name="other-domain-join-methods"></a>其他網域加入方法
在情況下，很不實際將電腦加入網域，使用有線的乙太網路連線，或在情況下，使用者無法登入網域的第一次使用有線的連線，您必須使用替代方法。

- **IT 人員電腦設定**。 IT 人員成員的無線電腦加入網域，並設定單一登入啟動程序的無線設定檔。 使用此方法，IT 系統管理員會連接到有線的乙太網路的無線電腦，並將電腦加入網域。 然後系統管理員會將散發給使用者的電腦。 當使用者啟動時沒有使用有線的連線，以手動方式進行使用者登入指定的網域認證的電腦是用來同時建立連線到無線網路並登入網域。

如需詳細資訊，請參閱[使用 IT 人員的電腦設定方法來加入網域並登入](#bkmk_itstaff)

-   **啟動使用者的無線設定檔設定**。 使用者手動設定的無線電腦與啟動程序的無線設定檔，並加入網域，以取得從 IT 系統管理員的指示。 啟動程序的無線設定檔可讓使用者建立無線連線，然後加入網域。 將電腦加入網域並重新啟動電腦之後, 使用者可以登入網域使用無線連線和其網域帳戶認證。

如需詳細資訊，請參閱[藉由使用 Bootstrap 無線設定檔設定，使用者所加入的網域和登入](#bkmk_userbootstrap)。

### <a name="bkmk_itstaff"></a>使用 IT 人員的電腦設定方法來加入網域並登入
與網域的網域成員使用者\-聯結的無線用戶端電腦可以使用暫存的無線設定檔，連線至 802.1x\-驗證而不需要先連接到有線區域網路的無線網路。 此暫存的無線設定檔叫做*無線設定檔啟動程序*。

啟動程序的無線設定檔會要求使用者以手動方式指定其網域使用者帳戶認證，並不會驗證憑證的遠端驗證撥\-入使用者服務\(RADIUS\)伺服器執行網路原則伺服器\(NPS\)。

建立無線連線之後，會套用群組原則的無線用戶端電腦上，並自動發出新的無線設定檔。 新的原則會使用電腦和使用者帳戶認證進行用戶端驗證。 

此外，PEAP 的一部分\-MS\-CHAP v2 相互驗證使用新的設定檔，而不啟動程序的設定檔，用戶端會驗證 RADIUS 伺服器的認證。

您將電腦加入網域之後，使用此程序來設定單一登入啟動程序無線設定檔，散發到網域的無線電腦之前\-成員使用者。

#### <a name="to-configure-a-single-sign-on-bootstrap-wireless-profile"></a>若要設定單一登入啟動程序的無線設定檔

1. 建立名為本指南中使用的程序的啟動程序的設定檔[PEAP 設定無線連線設定檔\-MS\-MS-CHAP v2](#bkmk_configureprofile)，並使用下列設定：

    - PEAP\-MS\-CHAP v2 驗證

    - 驗證 RADIUS 伺服器憑證已停用

    - 單一的登入已啟用

2. 您可以在其中建立新的啟動程序設定檔，來在無線網路原則內容中**一般**索引標籤上，選取的啟動程序的設定檔，然後按一下**匯出**匯出至設定檔網路共用 USB 快閃磁碟機或其他容易存取的位置。 設定檔會儲存為 *.xml 檔案到您指定的位置。

3. 將新的無線電腦加入網域\(例如，透過乙太網路連線，而無須 IEEE 802.1x 驗證\)並新增至電腦的啟動程序的無線設定檔，使用**netsh wlan新增設定檔**命令。

    >[!NOTE]
    >如需詳細資訊，請參閱的 Netsh 命令的無線區域網路\(WLAN\)在[http:\/\/technet.microsoft.com\/文件庫\/dd744890.aspx](https://technet.microsoft.com/library/dd744890).

4. 散發給使用者 「 登入網域使用執行 Windows 10 的電腦。 」 的程序與新無線電腦

當使用者啟動電腦時，Windows 會提示使用者輸入其網域使用者帳戶名稱和密碼。 因為單一登入已啟用，電腦就會使用網域使用者帳戶認證，第一次建立與此無線網路，然後登入網域的連線。
