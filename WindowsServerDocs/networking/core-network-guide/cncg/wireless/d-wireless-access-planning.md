# <a name="wireless-access-deployment-planning"></a>無線存取部署規劃

>適用於：Windows Server (半年度管道)、Windows Server 2016

部署無線存取之前，您必須先規劃下列專案：

- 在您的網路上\(安裝\)無線存取點的 ap

- 無線用戶端設定和存取

下列各節提供有關這些規劃步驟的詳細資料。

## <a name="planning-wireless-ap-installations"></a>規劃無線 AP 安裝
當您設計無線網路存取解決方案時，您必須執行下列動作：

1. 判斷您的無線 Ap 必須支援哪些標準
2. 判斷您想要提供無線服務的涵蓋範圍區域
3. 判斷您想要找出無線 Ap 的位置

此外，您必須規劃無線 AP 和無線用戶端的 IP 位址配置。 如需相關資訊，請參閱下面的**在 NPS 中規劃無線 AP**的設定一節。

### <a name="verify-wireless-ap-support-for-standards"></a>確認無線 AP 對標準的支援
基於一致性和輕鬆部署和 AP 管理的目的，建議您部署相同品牌和型號的無線 Ap。

您部署的無線 Ap 必須支援下列各項：

- **IEEE 802.1 X**

- **RADIUS 驗證**

- **無線驗證和加密。** 列出順序最高到最低的優先順序：

    1.  具有\-AES 的 WPA2 Enterprise

    2.  WPA2\-Enterprise 與 TKIP

    3.  使用\-AES 的 WPA Enterprise

    4.  具有\-TKIP 的 WPA Enterprise

>[!NOTE]
>若要部署 WPA2，您必須使用也支援 WPA2 的無線網路介面卡和無線 Ap。 否則，請使用\-WPA Enterprise。

此外，若要為網路提供增強的安全性，無線 Ap 必須支援下列安全性選項：

- **DHCP 篩選。** 無線 AP 必須篩選 IP 埠，以防止在將無線用戶端設定為 DHCP 伺服器的情況下，進行 DHCP 廣播訊息的傳輸。 無線 AP 必須封鎖用戶端，將 IP 封包從 UDP 埠68傳送到網路。

- **DNS 篩選。** 無線 AP 必須篩選 IP 埠，以防止用戶端以 DNS 伺服器的身分執行。 無線 AP 必須封鎖用戶端，將 IP 封包從 TCP 或 UDP 埠53傳送到網路。

- **用戶端隔離**如果您的無線存取點提供用戶端隔離功能，您應該啟用此功能，以防止可能\(的\)位址解析通訊協定 ARP 詐騙入侵。

### <a name="identify-areas-of-coverage-for-wireless-users"></a>識別無線使用者的涵蓋範圍區域
針對每個建築物使用每個樓層的架構繪圖，以識別您想要提供無線涵蓋範圍的區域。 例如，識別適當的辦公室、會議室、大廳、cafeterias 或 courtyards。

在繪圖上，指出干擾無線信號的任何裝置，例如醫療設備、無線攝影機、在2.4 到 2.5 GHz 工業、科學和醫療\(ISM\)中操作的無繩電話範圍，以及已\-啟用 Bluetooth 的裝置。

在繪圖上，標示建築物中可能幹擾無線信號的各個層面;建築物中使用的金屬物件可能會影響無線信號。 例如，下列常見物件可能會干擾信號傳播：電梯、加熱和空調\-管道，以及具體支援 girders。

如需可能導致無線 AP 無線電頻率衰減的來源相關資訊，請參閱您的 AP 製造商。 大部分的 Ap 都會提供測試軟體，供您用來檢查信號強度、錯誤率和資料輸送量。

### <a name="determine-where-to-install-wireless-aps"></a>判斷無線 Ap 的安裝位置
在架構繪圖上，找出您的無線 Ap，以提供足夠的無線涵蓋範圍，但距離不會干擾彼此。

Ap 之間所需的距離取決於 AP 和 AP 天線的類型、建築物中封鎖無線信號的層面，以及其他干擾來源。 您可以標示無線 AP 放置，讓每個無線 AP 不會超過任何鄰近無線 AP 的300英尺。 如需放置的 AP 規格和指導方針，請參閱無線 AP 製造商的檔。

在架構繪圖上指定的位置中，暫時安裝無線 Ap。 然後，使用配備了802.11 無線介面卡的膝上型電腦，以及常用於無線介面卡的網站問卷軟體，來判斷每個涵蓋範圍區域內的信號強度。

在信號強度較低的涵蓋範圍區域中，定位 AP 以改善涵蓋範圍區域的信號強度，安裝額外的無線 Ap 以提供所需的涵蓋範圍、重新放置或移除信號干擾的來源。  

更新您的架構繪圖，以指出所有無線 Ap 的最終位置。 具有精確的 AP 位置對應將會在疑難排解作業期間，或當您想要升級或取代 Ap 時提供協助。

### <a name="plan-wireless-ap-and-nps-radius-client-configuration"></a>規劃無線 AP 和 NPS RADIUS 用戶端設定
您可以使用 NPS 個別或群組來設定無線 Ap。

如果您要部署包含許多 Ap 的大型無線網路，在群組中設定 Ap 會變得更容易。 若要將 Ap 新增為 NPS 中的 RADIUS 用戶端群組，您必須使用這些屬性來設定 Ap。

- 無線 Ap 會使用來自相同 IP 位址範圍的 IP 位址進行設定。

- 無線 Ap 全都設定了相同的共用密碼。

### <a name="plan-the-use-of-peap-fast-reconnect"></a>規劃使用 PEAP 快速重新連線
在 802.1 X 基礎結構中，會將無線存取點設定為 RADIUS 伺服器的 RADIUS 用戶端。 部署 PEAP 快速重新連線時，在兩個或多個存取點之間漫遊的無線用戶端，不需要使用每個新的關聯來進行驗證。

PEAP 快速重新連線可減少用戶端與驗證器之間驗證的回應時間，因為驗證要求會從新的存取點轉送至原先執行用戶端驗證和授權的 NPS連接要求。

由於 PEAP 用戶端和 NPS 兩者都使用先前快取的傳輸\(層\)安全性 TLS \(連接屬性，其名為 TLS 控制碼\)，因此 NPS 可以快速判斷用戶端已獲得重新連線的授權。

>[!IMPORTANT]
>若要讓快速重新連線正常運作，必須將 Ap 設定為相同 NPS 的 RADIUS 用戶端。

如果原始 NPS 無法使用，或用戶端移至設定為不同 RADIUS 伺服器之 RADIUS 用戶端的存取點，則用戶端與新的驗證器之間必須進行完整驗證。

### <a name="wireless-ap-configuration"></a>無線 AP 設定
下列清單摘要說明在 802.1 x\-支援的無線 ap 上一般設定的專案：

> [!NOTE]
> 專案名稱可能因品牌和型號而異，而且可能與下列清單中的不同。 如需設定的\-特定詳細資料，請參閱您的無線 AP 檔。

- **服務組識別\(元\)SSID**。 這是無線網路\(的名稱，例如 ExampleWlan\)，以及向無線用戶端通告的名稱。 為了減少混淆，您選擇要通告的 SSID 不應符合無線網路接收範圍內的任何無線網路所廣播的 SSID。

    在多個無線 Ap 部署為相同無線網路一部分的情況下，請使用相同的 SSID 來設定每個無線 AP。 在多個無線 Ap 部署為相同無線網路一部分的情況下，請使用相同的 SSID 來設定每個無線 AP。  

    如果您需要部署不同的無線網路以符合特定的商務需求，則您的無線 AP 在一個網路上應廣播與其他網路\( \)的 ssid 不同的 ssid。 例如，如果您的員工和來賓需要個別的無線網路，您可以設定商業網路的無線 Ap，並將 SSID 設為廣播**ExampleWLAN**。 針對您的來賓網路，您可以將每個無線 AP 的 SSID 設定為廣播**GuestWLAN**。 如此一來，您的員工和來賓就可以連接到預期的網路，而不會產生不必要的混淆。  

    > [!TIP]  
    > 有些無線 AP 能夠廣播多個 SSID 來容納多\-個網路部署。 可以廣播多個 SSID 的無線 AP 可以降低部署和營運維護成本。  

- **無線驗證和加密**。

    無線驗證是在無線用戶端與無線存取點建立關聯時使用的安全性驗證。  

    無線加密是與無線驗證搭配使用的安全性加密密碼，用來保護無線 AP 和無線用戶端之間傳送的通訊。  

- **無線 AP IP 位址\(靜態\)** 。 在每個無線 AP 上，設定唯一的靜態 IP 位址。 如果子網是由 DHCP 伺服器提供服務，請確定所有的 AP IP 位址都落在 DHCP 排除範圍內，讓 DHCP 伺服器不會嘗試對另一部電腦或裝置發出相同的 IP 位址。 排除範圍記載于[核心網路指南](https://technet.microsoft.com/windows-server-docs/networking/core-network-guide/core-network-guide)中的「建立和啟用新的 DHCP 領域」程式。 如果您打算依照 NPS 中的群組將 Ap 設定為 RADIUS 用戶端，則群組中的每個 AP 都必須具有來自相同 IP 位址範圍的 IP 位址。

- **DNS 名稱**。 某些無線 Ap 可以設定 DNS 名稱。 使用唯一的名稱來設定每個無線 AP。 例如，\-如果您在多個案例建立中有已部署的無線 ap，您可以將第三個 AP3\-01、AP3\-02 和 AP3\-03 上部署的無線 ap 命名為。

- **無線 AP 子網路遮罩**。 設定遮罩來指定 IP 位址的哪個部分是網路識別碼，而 IP 位址的哪個部分是主機。

- **AP DHCP 服務**。 如果您的無線 AP 有內\-建的 DHCP 服務，請將它停用。

- **RADIUS 共用密碼**。 除非您打算設定群組中的 NPS RADIUS 用戶端，否則請為每個無線 AP 使用唯一的 RADIUS 共用密碼-在這種情況下，您必須使用相同的共用密碼來設定群組中的所有 Ap。 共用密碼應為長度至少為22個字元的隨機序列，其中包含大小寫字母、數位和標點符號。 為確保隨機性，您可以使用隨機字元產生程式來建立共用密碼。 建議您記錄每個無線 AP 的共用密碼，並將其儲存在安全的位置，例如 office 安全。 當您在 NPS 主控台中設定 RADIUS 用戶端時，您會建立每個 AP 的虛擬版本。 您在 NPS 中的每個虛擬 AP 上設定的共用密碼，必須符合實際實體 AP 上的共用密碼。

- **RADIUS 伺服器 IP 位址**。 輸入您要用來對此存取點進行驗證和授權連線要求的 NPS IP 位址。

- **UDP 埠\(s\)** 。 根據預設，NPS 會針對 RADIUS 驗證訊息使用 UDP 埠1812和1645，並針對 RADIUS 帳戶處理訊息使用 UDP 埠1813和1646。 建議您不要變更預設的 RADIUS UDP 埠設定。

- **Vsa**。 某些無線 ap 需要廠商\-特有的\(屬性\) vsa，才能提供完整的無線 AP 功能。

- **DHCP 篩選**。 設定無線 Ap 以封鎖無線用戶端，將 IP 封包從 UDP 埠68傳送到網路。 請參閱無線 AP 的檔以設定 DHCP 篩選。

- **DNS 篩選**。 設定無線 Ap 以封鎖無線用戶端，將 IP 封包從 TCP 或 UDP 埠53傳送到網路。 請參閱無線 AP 的檔以設定 DNS 篩選。

## <a name="planning-wireless-client-configuration-and-access"></a>規劃無線用戶端設定和存取

規劃 802.1 x\-驗證無線存取的部署時，您必須考慮幾個客戶\-端的特定因素：

- **規劃多項標準的支援**。

    判斷您的無線電腦是否全都使用相同版本的 Windows，或者它們是否混合成執行不同作業系統的電腦。 如果不同，請確定您瞭解作業系統支援的任何標準差異。

    判斷所有無線用戶端電腦上的所有無線網路介面卡是否支援相同的無線標準，或您是否需要支援不同的標準。 例如，判斷某些網路介面卡硬體驅動程式是否支援 WPA2\-enterprise 和 AES，而有些則只支援\-WPA enterprise 和 TKIP。

- **規劃用戶端驗證模式**。 驗證模式定義 Windows 用戶端處理網域認證的方式。 您可以從無線網路原則中的下列三種網路驗證模式中選取。  

    1. **使用者重新\-驗證**。 此模式指定一律根據電腦的目前狀態，使用安全性認證來執行驗證。 當沒有任何使用者登入電腦時，會使用電腦認證來執行驗證。 當使用者登入電腦時，一律會使用使用者認證來執行驗證。  

    2. **僅有電腦**。 [僅限電腦] 模式指定一律只使用電腦認證執行驗證。  

    3.  **使用者驗證**。 使用者驗證模式指定只有在使用者登入電腦時，才會執行驗證。 當沒有任何使用者登入電腦時，就不會執行驗證嘗試。  

- **規劃無線限制**。 決定您是否要為所有無線使用者提供相同的無線網路存取層級，或者您是否想要限制某些無線使用者的存取權。 您可以針對特定的無線使用者群組套用 NPS 中的限制。  例如，您可以定義特定的日期和時數，允許特定群組存取無線網路。  

- **用於新增無線電腦的規劃方法**。 針對已\-加入網域的無線電腦，在部署無線網路之前，如果電腦連線到未受 802.1 x 保護的有線網路區段，則無線設定為在您于網域控制站上設定\(無線網路\) IEEE 802.11 原則之後，以及在無線用戶端上重新整理群組原則之後，會自動套用。  

    不過，對於尚未加入網域的電腦，您必須規劃方法來套用 802.1 x\-驗證存取所需的設定。 例如，您可以使用下列其中一種方法來判斷是否要將電腦加入網域。

    1.  將電腦連線到未受 802.1 X 保護的有線網路區段，然後將電腦加入網域。

    2.  提供您的無線使用者新增自己的無線啟動程式設定檔所需的步驟和設定，讓他們能夠將電腦加入網域。

    3.  指派 IT 人員以將無線用戶端加入網域。

### <a name="planning-support-for-multiple-standards"></a>規劃多項標準的支援

群組原則的無線\(網路 IEEE\) 802.11 原則延伸提供各種不同的設定選項，以支援各種部署選項。

您可以部署以您想要支援的標準設定的無線 ap，然後在無線網路\(IEEE 802.11\)原則中設定多個無線設定檔，其中每個設定檔都指定一組標準您需要的。

例如，如果您的網路有支援 WPA2\-enterprise 和 aes 的無線電腦、其他支援 wpa\-enterprise 和 aes 的電腦，以及其他支援 wpa\-enterprise 和 TKIP 的電腦，您必須判斷您是否想要：

- 使用所有電腦都支援的最弱加密方法，設定單一設定檔以支援所有的無線電腦-在此情況下為 WPA\-Enterprise 和 TKIP。  
- 設定兩個設定檔，以提供每部無線電腦支援的最佳安全性。 在此例中，您會設定一個設定檔，指定\(最\-強的加密\)WPA2 enterprise 和 AES，以及一個使用較\-弱的 WPA enterprise 和 TKIP 加密的設定檔。 在此範例中，您必須將使用 WPA2\-Enterprise 和 AES 最高的設定檔放在喜好設定順序中。 無法使用 WPA2\-enterprise 和 AES 的電腦將會自動跳到下一個設定檔的喜好設定順序，並處理指定 WPA\-Enterprise 和 TKIP 的設定檔。

> [!IMPORTANT]
> 您必須在設定檔的排序清單中，將具有最安全標準的設定檔放在較高的位置，因為連接的電腦會使用他們能夠使用的第一個設定檔。

### <a name="planning-restricted-access-to-the-wireless-network"></a>規劃限制存取無線網路

在許多情況下，您可能會想要為無線使用者提供不同的無線網路存取層級。 例如，您可能會想要允許某些使用者不受限制的存取權，這是一天當中每一周的任何小時。 對於其他使用者，您可能只想要允許在核心時間（星期一到星期五）進行存取，以及在星期六和星期日拒絕存取。

本指南提供的指示可讓您建立存取環境，將您的所有無線使用者都放在具有無線資源通用存取權的群組中。 您會在 [Active Directory 使用者和電腦] 嵌入式管理單元\-中建立一個無線使用者安全性群組，然後讓您想要授與無線存取權的每位使用者都是該群組的成員。

當您設定 NPS 網路原則時，您會將無線使用者安全性群組指定為 NPS 在決定授權時所處理的物件。

不過，如果您的部署需要支援不同層級的存取，您只需要執行下列動作：  

1. 建立一個以上的無線使用者安全性群組，在 Active Directory 使用者和電腦中建立額外的無線安全性群組。 例如，您可以建立一個群組，其中包含具有完整存取權的使用者、只在一般工作時間擁有存取權的群組，以及符合符合您需求之其他條件的其他群組。

2. 將使用者新增至您所建立的適當安全性群組。

3. 為每個額外的無線安全性群組設定其他 NPS 網路原則，並設定原則以套用每個群組所需的條件和限制。

### <a name="planning-methods-for-adding-new-wireless-computers"></a>用於新增無線電腦的規劃方法

將新的無線電腦加入網域，然後登入網域的慣用方法，是使用可存取網域控制站的 LAN 區段的有線連線，而且不受 802.1 X 驗證 Ethernet 交換器的保護。

不過，在某些情況下，使用有線連線來將電腦加入網域，或使用已加入網域的電腦進行第一次登入嘗試時，可能不會很實用。

若要使用無線連線將電腦加入網域，或讓使用者在第一次使用已加入網域\-的電腦和無線連線登入網域，則無線用戶端必須先在上建立與無線網路的連線具有網路網域控制站存取權的區段，其方式是使用下列其中一種方法。

1. **IT 人員的成員將無線電腦加入網域，然後設定單一登入啟動程式的無線設定檔。** 使用此方法時，IT 系統管理員會將無線電腦連接到有線 Ethernet 網路，然後將電腦加入網域。 然後，系統管理員會將電腦散發給使用者。 當使用者啟動電腦時，他們針對使用者登入程式手動指定的網域認證會用來建立連到無線網路的連線，並登入網域。  

2. **使用者以手動方式設定具有啟動程式無線設定檔的無線電腦，然後加入網域。** 使用此方法，使用者可根據 IT 系統管理員的指示，使用啟動程式無線設定檔手動設定其無線電腦。 啟動程式無線設定檔可讓使用者建立無線連線，然後將電腦加入網域。 將電腦加入網域並重新啟動電腦之後，使用者可以使用無線連線和其網域帳號憑證來登入網域。

若要部署無線存取，請參閱[無線存取部署](e-wireless-access-deployment.md)。
