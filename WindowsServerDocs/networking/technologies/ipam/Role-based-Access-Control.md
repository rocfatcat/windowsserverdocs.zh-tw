# <a name="role-based-access-control"></a>以角色為基礎存取控制

>適用於：Windows Server（以每年次管道）、Windows Server 2016

本主題提供使用以角色為基礎存取控制 IPAM 中相關資訊。  
  
> [!NOTE]  
> 本主題中，除了下列 IPAM 存取控制文件已在本區段中。  
>   
> -   [管理角色為基礎存取控制與伺服器管理員](../../technologies/ipam/Manage-Role-Based-Access-Control-with-Server-Manager.md)  
> -   [管理角色為基礎存取控制使用 Windows PowerShell](../../technologies/ipam/Manage-Role-Based-Access-Control-with-Windows-PowerShell.md)  
  
以角色為基礎存取控制可讓您指定在各種不同層級，包括 DNS 伺服器、DNS 區域，以及 DNS 資源使用碼表進行層級的存取權限。  
使用角色為基礎存取控制，您可以指定誰控制細微來建立、編輯、及 delete 不同類型的 DNS 資源記錄作業。  
  
您可以存取控制設定使用者僅限於下列權限。  
  
-   使用者可以編輯只特定 DNS 資源記錄  
  
-   使用者可以編輯 DNS 資源記錄特定類型，例如 PTR 或 MX  
  
-   使用者可以編輯 DNS 資源記錄特定區域
