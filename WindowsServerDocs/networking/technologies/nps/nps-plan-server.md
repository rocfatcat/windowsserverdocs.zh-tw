# <a name="plan-nps-as-a-radius-server"></a>規劃 NPS RADIUS 伺服器

>適用於：Windows Server（以每年次管道）、Windows Server 2016

當您要部署的網路原則伺服器 \(NPS\) 為遠端驗證 Dial 使用者服務 (RADIUS) 伺服器時、 NPS 執行驗證、 授權及計量本機網域和信任本機網域的連接要求。 您可以使用下列計劃的指導方針操作來簡化 RADIUS 部署。

這些計劃的指導方針不包含您要部署 NPS RADIUS proxy 為的環境。 當您部署 NPS RADIUS proxy 為時，NPS 轉送執行 NPS 或其他 RADIUS 伺服器遠端網域，不受信任的網域，或兩者伺服器連接要求。 

您部署 NPS RADIUS 伺服器以您網路上之前，請使用下列指導方針計劃部署。

- 規劃伺服器 NPS 的設定。

- 規劃 RADIUS 戶端。

- 計畫使用的驗證方法。

- 規劃網路原則。

- 規劃 NPS 計量。

## <a name="plan-the-use-of-authentication-methods"></a>使用的驗證方法計劃

NPS 支援兩種密碼與憑證基礎驗證方法。 不過，並非所有的網路存取伺服器支援相同的驗證方法。 有時候，您可能想要部署不同的驗證方法根據網路存取權的類型。

例如，您可能想要部署無線和 VPN 存取您的組織，但的存取權的每一種使用不同的驗證方法： EAP-TLS 的 VPN 連接穩固安全性 EAP 使用 Tls (EAP-TLS) 提供，因為和 PEAP MS-CHAP v2 802.1 X wireless 連接。

因為 「 積極型 」 重新連接，提供的安全性 PEAP MS-CHAP v2，PEAP MS-CHAP v2 是邏輯選擇 wireless 連接的驗證方法。

### <a name="certificate-based-authentication-methods"></a>憑證式的驗證方法

憑證式的驗證方法有提供穩固安全性; 的優點而且他們會更難以部署比密碼為基礎的驗證方法的缺點。

PEAP MS-CHAP v2 和 EAP-TLS 憑證架構的驗證方法，但有許多不同之處他們，他們部署的方式。

### <a name="eap-tls"></a>EAP-TLS

EAP-TLS 使用 client 和伺服器驗證憑證，並要求您在組織中部署公用基礎結構 (PKI)。 部署 PKI 複雜，並需要規劃階段的獨立規劃 NPS RADIUS 伺服器為使用。

EAP-TLS 具有 NPS 伺服器註冊伺服器的憑證憑證授權單位 \(CA\)，以及憑證會儲存在本機電腦的憑證存放區。 驗證程序期間當 NPS 伺服器將其身份存取 client 存取 client 來傳送它伺服器的憑證，就會發生伺服器的驗證。 存取 client 檢查以判斷是否憑證無效的而且伺服器的驗證期間是適用於各種憑證屬性。 如果伺服器的憑證符合最低伺服器的憑證，來存取 client 信任的憑證授權單位發行 NPS 伺服器成功驗證 client 的。

同樣地，client 進行驗證驗證程序期間當 client 將其 client 憑證傳送至其身份 NPS 伺服器 NPS 伺服器。 NPS 伺服器檢查憑證，並如果 client 憑證符合最低 client 憑證，並由 NPS 伺服器信任的憑證授權單位發行，NPS 伺服器成功驗證存取 client。

雖然它所需的伺服器的憑證，會儲存在 NPS 伺服器的憑證存放區，client 或使用者憑證可以儲存任一憑證存放區 client 或智慧卡上。

成功此驗證程序，如有需要的所有電腦目前的使用者的本機電腦的受信任的根憑證授權單位憑證存放區都有您組織的憑證。

### <a name="peap-ms-chap-v2"></a>PEAP MS-CHAP v2

PEAP MS-CHAP v2 伺服器的驗證並密碼型使用者驗證憑證的使用的憑證。 由於伺服器驗證只會使用的憑證，您就不需要部署 PKI，才能使用了 v2 PEAP MS-CHAP。 當部署 PEAP MS-CHAP v2 時，您可以在下列兩種方式取得伺服器 NPS 伺服器的憑證：

- 您可以安裝 Active Directory 憑證 Services (AD CS)，然後自動註冊憑證 NPS 伺服器。 如果您使用此方法，您還必須註冊 CA 憑證 client 電腦連接到您的網路，讓它們在信任發行 NPS 伺服器的憑證。

- 您可以從 VeriSign 例如公用 CA 購買伺服器的憑證。 如果您使用此方法，請確定已選取已經信任的樹系 client 電腦，CA。 若要判斷是否 client 電腦信任 CA，請打開憑證 Microsoft Management Console (MMC) 嵌入式管理單元 client 的電腦上，然後檢視 [本機電腦，以及目前的使用者的受信任的根憑證授權單位存放區。 如果從這些憑證存放區 CA 憑證，client 電腦信任 CA，進而信任任何 CA 所發行的憑證。

PEAP MS-CHAP v2 驗證程序期間當 NPS 伺服器 client 電腦來傳送它伺服器的憑證，就會發生伺服器的驗證。 存取 client 檢查以判斷是否憑證無效的而且伺服器的驗證期間是適用於各種憑證屬性。 如果伺服器的憑證符合最低伺服器的憑證，來存取 client 信任的憑證授權單位發行 NPS 伺服器成功驗證 client 的。

使用者驗證發生於使用者想要的網路類型密碼基礎認證來連接，並嘗試登入。 NPS 接收憑證，並執行驗證及授權。 如果驗證使用者並獲得授權成功，且驗證 NPS 伺服器成功 client 電腦，會授與連接要求。

### <a name="plan-network-policies"></a>規劃網路原則

NPS 使用網路原則來判斷是否有權從 RADIUS 收到連接要求。 NPS 也會讓授權判斷使用使用者 account 撥號的屬性。

NPS 嵌入式管理單元出現的順序處理的網路原則，因為計畫原則的清單中第一次放置最低原則。 針對每個連接要求 NPS 嘗試符合的條件的連接要求屬性的原則。 NPS 會檢查每個的網路原則順序直到找到符合。 如果找不到相符項目，會拒絕連接要求。

### <a name="key-steps"></a>步驟鍵

規劃網路原則，您可以使用下列步驟。

- 判斷慣用的網路原則，從最低到最高限制 NPS 處理訂單。

- 判斷原則狀態。 功能或停用狀態原則可能的值。 如果支援原則，NPS 會同時執行授權評估原則。 如果不支援原則，則不會評估。

- 判斷原則類型。 您必須判斷原則的設計目的是要權限授與時連接要求或是否原則的設計目的是即可時連接要求，以符合的條件原則的授權符合的條件的原則。 例如，如果您想要即可明確授權 wireless Windows 群組成員，您可以建立指定群組、 wireless 連接方法，以及的已輸入的拒絕存取設定原則的網路原則。

- 判斷您是否想要 NPS 略過的群組原則所依據的帳號撥號屬性。 當不支援此設定時的帳號撥號屬性覆寫的網路原則設定的設定。 例如，如果您的網路原則設定所授與的使用者存取權，但撥號屬性使用者的使用者帳號設定即可授權，使用者無法存取。 但如果您可以原則類型略過使用者 account 撥號中設定屬性的相同使用者授與網路的存取權。

- 判斷原則是否使用這項原則來源設定。 這項設定可讓您輕鬆地指定的所有存取要求來源。 來源可能是車票服務閘道器 （TS 閘道）、 遠端存取伺服器 （VPN 或撥號）、 DHCP 伺服器、 wireless 存取點和健康登記授權單位伺服器。 或者，您可以指定廠商特定來源。

- 判斷是否連接要求，以符合的條件的網路原則已經套用的設定。

- 判斷您是否要使用、 修改或 delete 預設的網路原則。

- 若要使用的所有的計量要求傳送給課程屬性網路存取伺服器計劃。 課程屬性傳送到 RADIUS client 接受存取的訊息，且適合用來與驗證的工作階段相關聯計量要求訊息。 如果計量要求訊息中的網路存取伺服器來傳送課程屬性，它可以用於隸屬會計和驗證。 屬性獨特的序號服務重新開機時，並伺服器位址的組合必須為每個接受伺服器的驗證的唯一驗證。

- 若要使用的網路存取伺服器支援暫時計量計劃。

- 計畫使用網路存取伺服器傳送計量上及計量關閉訊息。

- 規劃網路存取伺服器支援儲存和轉接計量資料的使用。 網路存取伺服器支援此功能可以儲存計量資料時網路存取伺服器無法具有 NPS 伺服器通訊。 NPS 伺服器可用時，網路存取伺服器轉送儲存的記錄 NPS 伺服器，透過不提供這項功能的網路存取伺服器提供計量的可靠性。

- 規劃網路原則設定帳號過渡間隔屬性。 帳號過渡間隔屬性設定網路的存取伺服器傳送每個暫時更新 （以秒計） 長的時間間隔。 根據 RFC 2869 帳號過渡間隔屬性的值不得小於 60 秒，或是 1 分鐘，且不得小於 600 秒或 10 分鐘。 如需詳細資訊，請查看 RFC 2869，「 RADIUS 擴充功能]。
