# <a name="configure-network-policies"></a>設定網路原則

>適用於：Windows Server （半年通道），Windows Server 2016

您可以使用本主題在 NPS 中設定網路原則。

## <a name="add-a-network-policy"></a>新增網路原則

網路原則伺服器\(NPS\)使用網路原則和以判斷是否授權連線要求連線到網路的使用者帳戶的撥入內容。

您可以使用此程序在 NPS 主控台或遠端存取主控台中設定新的網路原則。

### <a name="performing-authorization"></a>執行授權

當 NPS 執行連線要求的授權時，它會比較每個原則，從第一個原則，然後再設定的原則清單的排序清單中的網路原則的要求。 如果 NPS 發現條件符合連線要求原則，NPS 就會使用來執行授權的比對原則和使用者帳戶撥入屬性。 如果授與存取權或控制的存取權，透過網路原則設定的使用者帳戶撥入內容，並授權連線要求，NPS 會套用至連線的網路原則中所設定的設定。

如果 NPS 找不到符合連線要求的網路原則，除非使用者帳戶的撥入屬性會設定為授與存取權，就是會拒絕連線要求。

如果的使用者帳戶撥入內容設定為拒絕存取，NPS 會拒絕連線要求。

### <a name="key-settings"></a>機碼設定

當您使用新增網路原則精靈來建立網路原則，您在中指定的值**網路連線方法**用來自動進行**原則類型**條件： 

- 如果您未指定的預設值，您所建立的網路原則評估 nps 上使用任何一種網路存取伺服器 (NAS) 的所有網路連線類型。
- 如果您指定的網路連線方法時，NPS 會評估網路原則，只有當連接要求是來自您所指定的網路存取伺服器類型。

在 **存取權限**頁面上，您必須選取**授與存取權**如果您想要允許使用者連線到您的網路原則。 如果您想要防止使用者連線到網路時，選取的原則**拒絕存取**。 

如果您想要取決於使用者帳戶撥入內容，在 Active Directory 中的存取權限&reg;網域服務\(AD DS\)，您可以選取**存取由使用者撥入內容**  核取方塊。

若要完成此程序，至少需要 **Domain Admins** 的成員資格或同等權限。

### <a name="to-add-a-network-policy"></a>若要新增網路原則 

1. 開啟 NPS 主控台，然後按兩下**原則**。

2. 在主控台樹狀目錄中，以滑鼠右鍵按一下**網路原則**，然後按一下**新增**。 新的網路原則精靈 隨即開啟。

3. 您可以使用新增網路原則精靈來建立原則。

## <a name="create-network-policies-for-dial-up-or-vpn-with-a-wizard"></a>撥號或 VPN 使用精靈建立網路原則

您可以使用此程序來建立連線要求原則和網路部署撥號伺服器或虛擬私人網路所需的原則\(VPN\)伺服器作為遠端驗證撥入使用者服務\(RADIUS\) NPS RADIUS 伺服器的用戶端。

>[!NOTE]
>用戶端電腦，例如膝上型電腦及其他執行用戶端作業系統的電腦不是 RADIUS 用戶端。 RADIUS 用戶端網路存取伺服器--例如無線存取點、 802.1 X 驗證交換器、 虛擬私人網路\(VPN\)伺服器和撥號伺服器，因為這些裝置會使用 RADIUS 通訊協定進行通訊NPSs 這類的 RADIUS 伺服器。

此程序說明如何在 NPS 中，開啟新增撥號或虛擬私人網路連線精靈。

執行精靈之後，會建立下列原則：

- 一個連線要求原則
- 一個網路原則

每當您需要為撥號伺服器和 VPN 伺服器建立新原則，您可以執行新增撥號或虛擬私人網路連線精靈。

執行新增撥號或虛擬私人網路連線精靈並非部署撥號或 VPN 伺服器為 NPS RADIUS 用戶端所需的唯一步驟。 這兩個網路存取方法都需要您部署額外的硬體和軟體元件。

若要完成此程序，至少需要 **Domain Admins** 的成員資格或同等權限。

### <a name="to-create-policies-for-dial-up-or-vpn-with-a-wizard"></a>若要使用精靈建立撥號或 VPN 原則

1. 開啟 NPS 主控台。 如果未選取，按一下**NPS\(本機\)**。 如果您想要在遠端 NPS 上建立原則，選取的伺服器。

2. 在**快速入門**並**標準組態**，選取**撥號或 VPN 連線的 RADIUS 伺服器**。 文字和文字變更以反映您的選取項目底下的連結。

3. 按一下 **設定 VPN 或使用精靈的 撥號**。 新的撥號或虛擬私人網路連線精靈 隨即開啟。

4. 遵循精靈中的指示，完成建立新的原則。

## <a name="create-network-policies-for-8021x-wired-or-wireless-with-a-wizard"></a>建立網路原則的 802.1x 有線或無線的精靈

您可以使用此程序建立的連線要求原則和部署 802.1x 驗證交換器或 802.1x 無線存取點做為 NPS 的遠端驗證撥號使用者服務 (RADIUS) 用戶端所需的網路原則RADIUS 伺服器。

此程序說明如何在 NPS 中啟動 [新增 IEEE 802.1x 的安全有線及無線連線] 精靈。

執行精靈之後，會建立下列原則：

- 一個連線要求原則
- 一個網路原則

每當您需要建立 802.1x 存取的新原則，您可以執行 [新增 IEEE 802.1x 的安全有線及無線連線] 精靈。

執行 [新增 IEEE 802.1x 的安全有線及無線連線] 精靈並非部署 802.1x 驗證交換器及無線存取點做為 NPS 的 RADIUS 用戶端所需的唯一步驟。 這兩個網路存取方法都需要您部署額外的硬體和軟體元件。

若要完成此程序，至少需要 **Domain Admins** 的成員資格或同等權限。

### <a name="to-create-policies-for-8021x-wired-or-wireless-with-a-wizard"></a>若要使用精靈建立原則的 802.1x 有線或無線

1. 在 NPS 中，在 伺服器管理員 中，按一下 **工具**，然後按一下**網路原則伺服器**。 NPS 主控台隨即開啟。 

2. 如果未選取，按一下**NPS\(本機\)**。 如果您想要在遠端 NPS 上建立原則，選取的伺服器。

3. 在**快速入門**並**標準組態**，選取**802.1x 無線或有線連線的 RADIUS 伺服器**。 文字和文字變更以反映您的選取項目底下的連結。

4. 按一下 [**設定 802.1x] 使用精靈**。 [新增 IEEE 802.1x 的安全有線及無線連線] 精靈隨即開啟。

5. 遵循精靈中的指示，完成建立新的原則。

## <a name="configure-nps-to-ignore-user-account-dial-in-properties"></a>設定 NPS 略過使用者帳戶撥入內容

若要設定 NPS 網路原則，若要在授權程序期間略過 Active Directory 中的使用者帳戶撥入內容中使用此程序。 在 Active Directory 使用者和電腦的使用者帳戶具有撥入內容，NPS 會評估授權程序期間，除非**網路存取權限**的使用者帳戶的屬性設定為**控制項透過 NPS 網路原則存取**。 

有兩種情況下您可能要設定 NPS 略過 Active Directory 中的使用者帳戶撥入內容：

- 當您想要使用的網路原則，來簡化 NPS 授權，但不是所有的使用者帳戶都有**網路存取權限**屬性設定為**透過 NPS 網路原則控制存取**。 比方說，有部分使用者帳戶可能**網路存取權限**的使用者帳戶的屬性設定為**拒絕存取**或是**允許存取**。

- 當使用者帳戶的其他撥入內容並不適用於已在網路原則中設定的連接類型。 比方說，以外的屬性**網路存取權限**設定僅適用於撥號或 VPN 連線，但您要建立的網路原則是無線或驗證交換器連線。

若要設定 NPS 略過使用者帳戶撥入內容，您可以使用此程序。 如果連線要求符合網路原則，其中已選取此核取方塊，NPS 不會使用使用者帳戶撥入內容來判斷是否授權使用者或電腦存取網路。網路原則中的設定用來決定授權。

中的成員資格**系統管理員**，或同等權限，才能完成此程序的最小值。

1. 在 NPS 中，在 伺服器管理員 中，按一下 **工具**，然後按一下**網路原則伺服器**。 NPS 主控台隨即開啟。

2. 按兩下**原則**，按一下**網路原則**，然後在詳細資料 窗格中按兩下您想要設定的原則。

3. 在原則中**屬性**對話方塊的 **概觀**索引標籤**存取權限**，選取**略過使用者帳戶撥入內容**核取方塊，然後按一下**確定**。

## <a name="configure-nps-for-vlans"></a>針對 Vlan 設定 NPS

Windows Server 2016 中使用感知 VLAN 的網路存取伺服器與 NPS 中,，您可以提供使用者的群組只能存取網路資源，適用於其安全性權限。 比方說，您可以提供訪客與無線存取網際網路而不允許他們存取您的組織網路。 

此外，虛擬區域網路可讓您以邏輯方式群組網路資源存在於不同的實體位置或不同的實體子網路上。 比方說，您的業務部門和其網路資源，例如用戶端電腦、 伺服器和印表機的成員可能位於數個不同的大樓，在您的組織，但您可以將所有這類資源放在一個使用相同的 IP 的 VLAN 上位址範圍。 VLAN 和 函式，從使用者的觀點而言，為單一子網路。

當您想要區隔為各種不同的使用者群組之間的網路時，您也可以使用 Vlan。 您已經決定要如何定義您的群組之後，可以在 Active Directory 使用者和電腦 嵌入式管理單元中建立安全性群組，並再將成員新增至群組。

### <a name="configure-a-network-policy-for-vlans"></a>設定 vlan 的網路原則

若要設定將使用者指派給 VLAN 的網路原則，您可以使用此程序。 當您使用 VLAN 感知的網路硬體，例如路由器、 交換器及存取控制站，您可以設定網路原則，指示存取伺服器，將特定的 Active Directory 群組的成員放置在特定 Vlan 上。 此群組以邏輯方式與 Vlan 的網路資源的能力，提供設計與實作網路解決方案的彈性。

當您設定的 NPS 網路原則，以使用 vlan 時，您必須設定屬性**通道媒體類型**， **Tunnel Pvt-群組 ID**， **Tunnel-type**，並**Tunnel-tag**。 

此程序會提供做為指導方針;您的網路設定可能需要不同於如下所述的設定。

中的成員資格**系統管理員**，或同等權限，才能完成此程序的最小值。

### <a name="to-configure-a-network-policy-for-vlans"></a>若要設定 vlan 的網路原則

1. 在 NPS 中，在 伺服器管理員 中，按一下 **工具**，然後按一下**網路原則伺服器**。 NPS 主控台隨即開啟。

2. 按兩下**原則**，按一下**網路原則**，然後在詳細資料 窗格中按兩下您想要設定的原則。

3. 在原則中**屬性** 對話方塊中，按一下**設定** 索引標籤。

4. 在原則中**屬性**，請在**設定**，請在**RADIUS 屬性**，確定**標準**已選取。

5. 在 [詳細資料] 窗格中，在**屬性**，則**服務類型**屬性已使用預設值是**框架**。 根據預設，使用的 VPN 和撥號、 存取方法的原則**框架通訊協定**屬性設定值是**PPP**。 若要指定其他連接屬性所需的 Vlan，請按一下**新增**。 **加入標準的 RADIUS 屬性**對話方塊隨即開啟。

6. 在 **加入標準的 RADIUS 屬性**、 在 屬性 向下捲動並新增下列屬性：

    - **通道媒體類型**。 選取適用於您所做的原則中的上一個選取項目值。 例如，如果您要設定網路原則是無線的原則，選取**值：802 （包含所有 802 媒體加上乙太網路標準格式）**。

    - **通道 Pvt-群組識別碼**。 輸入代表將指派群組成員的 VLAN 編號的整數。 

    - **通道類型**。 選取 **虛擬 Lan (VLAN)**。


7. 在 **加入標準的 RADIUS 屬性**，按一下**關閉**。 

8. 如果您的網路存取伺服器 (NAS) 需要使用**Tunnel-tag**屬性，請使用下列步驟來新增**Tunnel-tag**至網路原則屬性。 如果您的 NAS 文件則不涉及此屬性，不加入至原則。 如有需要，新增屬性，如下所示：

    - 在原則中**屬性**，請在**設定**，請在**RADIUS 屬性**，按一下 **特定廠商**。 

    - 在 [詳細資料] 窗格中，按一下**新增**。 **新增特定廠商屬性**對話方塊隨即開啟。

    - 在 **屬性**，向下捲動並選取**Tunnel-tag**，然後按一下 **新增**。 **屬性資訊**對話方塊隨即開啟。 

    - 在 **屬性值**，輸入您從您的硬體文件取得的值。

## <a name="configure-the-eap-payload-size"></a>設定 EAP 承載大小

在某些情況下，路由器或防火牆卸除封包因為它們已捨棄需要分散的封包。

當您使用可延伸驗證通訊協定的網路原則與部署 NPS \(EAP\)具有傳輸層安全性\(TLS\)，或 EAP-TLS，做為驗證方法，預設最大值傳輸單位\(MTU\) NPS 使用 EAP 承載的是 1500 個位元組。 

此大小上限為 EAP 裝載可以建立需要被路由器或防火牆 NPS 及 RADIUS 用戶端之間的分散程度的 RADIUS 訊息。 如果發生這種情況，路由器或防火牆位於之間的 RADIUS 用戶端與 NPS 伺服器以無訊息方式可能會捨棄部分片段，導致驗證失敗，且存取用戶端無法連線到網路。

您可以使用下列程序來降低 NPS 藉由調整框架 MTU 中屬性的值不大於 1344年網路原則用於 EAP 承載的大小上限。

中的成員資格**系統管理員**，或同等權限，才能完成此程序的最小值。

### <a name="to-configure-the-framed-mtu-attribute"></a>若要設定框架 MTU 屬性

1. 在 NPS 中，在 伺服器管理員 中，按一下 **工具**，然後按一下**網路原則伺服器**。 NPS 主控台隨即開啟。
 
2. 按兩下**原則**，按一下**網路原則**，然後在詳細資料 窗格中按兩下您想要設定的原則。

3. 在原則中**屬性** 對話方塊中，按一下**設定** 索引標籤。

4. 在 **設定**，請在**RADIUS 屬性**，按一下 **標準**。 在 [詳細資料] 窗格中，按一下**新增**。 **加入標準的 RADIUS 屬性**對話方塊隨即開啟。

5. 在 **屬性**，向下捲動並按一下 **框架 MTU**，然後按一下**新增**。 **屬性資訊**對話方塊隨即開啟。

6. 在 **屬性值**，輸入的值等於或小於比**1344年**。 按一下  **確定**，按一下**關閉**，然後按一下**確定**。
