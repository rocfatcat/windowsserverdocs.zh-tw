# <a name="deploying-a-two-node-clustered-file-server"></a>部署兩個節點的叢集的檔案伺服器

> 適用於：Windows Server 2019，Windows Server 2016

容錯移轉叢集是一組一起運作以提高可用性的應用程式和服務的獨立電腦。 各個叢集伺服器 (稱為節點) 是透過實體纜線與軟體連接。 如果其中一個叢集節點失敗，另一個節點即會開始提供服務 (稱為容錯移轉的處理程序)。 使用者會感覺的服務中斷。

本指南說明安裝和設定有兩個節點的一般用途檔案伺服器容錯移轉叢集的步驟。 藉由建立本指南中的組態，您可以深入了解容錯移轉叢集，並熟悉的容錯移轉叢集管理嵌入式管理單元介面在 Windows Server 2019 或 Windows Server 2016。

## <a name="overview-for-a-two-node-file-server-cluster"></a>雙節點檔案伺服器叢集概觀

在容錯移轉叢集中的伺服器可在各種不同的角色，包括檔案伺服器、 HYPER-V 伺服器或資料庫伺服器的角色，並可以各種不同的其他服務和應用程式提供高可用性。 本指南說明如何設定雙節點檔案伺服器叢集。

雖然任何磁碟區的儲存體中只有一部伺服器一次存取容錯移轉叢集通常會包括在實際連線到叢集中的所有伺服器的儲存體單位。 下圖顯示連接至儲存體單位的雙節點容錯移轉叢集。

![兩個節點叢集](media\Cluster-File-Server\Cluster-FS-Overview.png)

存放磁碟區或邏輯單元編號 (Lun) 公開給叢集中的節點必須不會公開至其他伺服器，包括其他叢集中的伺服器。 下圖說明這點。

![在儲存體 Lun](media\Cluster-File-Server\Cluster-FS-LUNs.png)

請注意，針對任何伺服器的最大的可用性，請務必遵循最佳做法來管理伺服器 — 比方說，小心管理伺服器、 測試軟體再套用之前，實體環境，並且仔細追蹤的軟體更新和所有叢集伺服器上的進行組態變更。

以下案例說明如何設定檔案伺服器容錯移轉叢集。 要共用的檔案位於叢集存放區，並可能是叢集的伺服器可做為檔案伺服器共用。

## <a name="shared-folders-in-a-failover-cluster"></a>共用資料夾容錯移轉叢集

下列清單說明整合到容錯移轉叢集的共用的資料夾設定功能：

- 顯示的範圍為叢集的共用資料夾只 （不再混合非叢集的共用資料夾）：當使用者檢視共用資料夾指定的叢集的檔案伺服器的路徑時，顯示將包含僅屬於特定的檔案伺服器角色的共用的資料夾。 它會排除非叢集的共用的資料夾和共用都剛好在叢集節點的個別檔案伺服器角色的一部分。

- 存取型列舉：若要隱藏指定的資料夾，使用者的檢視中，您可以使用存取型列舉。 而不是讓使用者看到資料夾但不是會存取在其上的任何項目，您可以選擇防止它們根本不到資料夾。 您可以在非叢集的共用資料夾的相同方式設定叢集的共用資料夾的存取為基礎的列舉。

- 離線存取：您可以設定離線快取叢集的共用資料夾中的非叢集的共用資料夾的相同方式。

- 叢集的磁碟永遠會被視為叢集的一部分：不論您是使用容錯移轉叢集介面、 Windows 檔案總管中，或 [共用與存放管理] 嵌入式管理單元，Windows 會識別磁碟是否被指定為叢集存放區。 如果這類磁碟已設定在容錯移轉叢集管理的叢集的檔案伺服器的一部分，您可以再使用任何先前所述的介面來建立共用磁碟上。 如果這類磁碟尚未設定為叢集的檔案伺服器的一部分，則您不小心無法在其上建立共用。 相反地，錯誤會指出，磁碟必須先設定為叢集的檔案伺服器的一部分才能共用。

- Services for Network File System 整合：Windows Server 中的檔案伺服器角色包括選用的角色服務，稱為服務的網路檔案系統 (NFS)。 藉由安裝角色服務，並使用 Services for NFS 設定共用的資料夾，您可以建立叢集的檔案伺服器支援以 UNIX 為基礎的用戶端。

## <a name="requirements-for-a-two-node-failover-cluster"></a>雙節點容錯移轉叢集的需求

在 Windows Server 2016 或 Windows Server 2019 視為 Microsoft 正式支援的方案中容錯移轉叢集，解決方案必須符合下列準則。

- 所有硬體與軟體元件需符合適當標誌的資格。 適用於 Windows Server 2016，這是 「 適用於 Windows Server 2016 已認證 」 標誌。 Windows Server 2019，這是 「 適用於 Windows Server 2019 認證 」 標誌。 如需有關哪些硬體和軟體的系統已經過認證的詳細資訊，請瀏覽 Microsoft [Windows Server Catalog](https://www.windowsservercatalog.com/default.aspx)站台。

- 完整設定的解決方案 （伺服器、 網路和儲存體） 必須通過所有測試，在驗證精靈中，這是容錯移轉叢集 嵌入式管理單元的一部分。

下列被必要的雙節點容錯移轉叢集。

- **伺服器：** 我們建議使用相同或類似元件的比對的電腦。  雙節點容錯移轉叢集的伺服器必須執行相同版本的 Windows Server。 它們也應該擁有相同的軟體更新 （修補程式）。

- **網路介面卡和纜線：** 網路硬體，例如在容錯移轉叢集解決方案中，其他元件必須與 Windows Server 2016 或 Windows Server 2019 相容。 如果您使用 iSCSI，網路介面卡必須專門用於網路通訊或 iSCSI，不可同時為兩者。 在連接叢集節點的網路結構中，避免發生單一失敗點。 有多種方式來完成這項作業。 您可以連接您的叢集節點的多個不同的網路。 或者，您可以使用藉由協同運作網路介面卡、備援交換器、備援路由器，或移除單一失敗點的類似硬體所建構的一種網路來連接叢集節點。

   > [!NOTE]
   > 如果使用單一網路來連接叢集節點，網路將會傳入 [驗證設定精靈] 的備援需求。  不過，報表將包含警告，在網路不應該是單一失敗點。

- **裝置控制器或適當的介面卡的儲存體：**
    - **序列連接 SCSI 或光纖通道：** 如果您使用序列連結 SCSI 或光纖通道，在所有叢集伺服器，則應該相同的儲存堆疊的所有元件。 它是必要的多重路徑 I/O (MPIO) 軟體及相同的裝置特定模組 (DSM) 軟體元件。  建議連結到叢集存放裝置的大型存放裝置控制器 (也就是主機匯流排介面卡 (HBA)、HBA 驅動程式及 HBA 韌體) 是相同的。 如果您使用不相同的 HBA，則應向存放裝置廠商確認，以符合其支援或建議的設定。
    - **iSCSI:** 如果您使用 iSCSI，每部叢集的伺服器都必須有一或多個網路介面卡或主機匯流排介面卡專用於 ISCSI 存放裝置。 您用於 iSCSI 的網路無法用於網路通訊。 在所有叢集伺服器中，用來連接 iSCSI 存放目標的網路介面卡都應該相同，建議您使用 Gigabit Ethernet 或速度更快的介面卡。  

- **儲存體：** 您必須使用 Windows Server 2016 或 Windows Server 2019 已通過認證的共用存放裝置。
  
    對於雙節點容錯移轉叢集，如果使用仲裁見證磁碟的儲存體時，應該包含至少兩個不同的磁碟區 (Lun)。 見證磁碟是設計來保存叢集設定資料庫複本的叢集儲存體中的磁碟。 對於這個雙節點叢集範例中，仲裁設定會節點與磁碟多數。 節點與磁碟多數表示節點和見證磁碟每個包含的叢集組態中，複本的叢集有仲裁，只要這些複本的多數 （三分之二以上） 都可以使用。 磁碟區 (LUN) 會包含使用者所共用的檔案。

    存放裝置的需求包含下列各項：

    - 若要使用容錯移轉叢集的原生磁碟支援，請使用基本磁碟而非動態磁碟。
    - 我們建議您格式化為 NTFS 磁碟分割 （如為見證磁碟，磁碟分割必須是 NTFS）。
    - 至於磁碟的磁碟分割樣式，您可以使用主開機記錄 (MBR) 或 GUID 磁碟分割表格 (GPT)。
    - 存放裝置必須正確回應特定的 SCSI 命令，存放裝置必須遵循稱為 SCSI Primary commands-3 (spc-3) 標準。 特別的是，存放裝置必須支援 spc-3 標準中所指定的持續保留。 
    -  存放裝置使用的 miniport 驅動程式必須搭配 Microsoft Storport 存放裝置驅動程式。

## <a name="deploying-storage-area-networks-with-failover-clusters"></a>部署容錯移轉叢集的存放區域網路

在部署時的容錯移轉叢集與存放區域網路 (SAN)，應該要觀察下列指導方針。

- **確認儲存體的認證：** 使用[Windows Server Catalog](https://www.windowsservercatalog.com/default.aspx)站台上，確認廠商的存放裝置，包括驅動程式、 韌體及軟體，已通過認證的 Windows Server 2016 或 Windows Server 2019。

- **隔離存放裝置，每個裝置一個叢集：** 不同叢集的伺服器不得存取同一個存放裝置。 在大部分情況下，用於某一組叢集伺服器的 LUN 應透過 LUN 遮罩或分區的所有其他伺服器隔離。

- **請考慮使用多重路徑 I/O 軟體：** 在高可用性存放裝置網狀架構中，您可以部署容錯移轉叢集具有多個主機匯流排介面卡使用多重路徑 I/O 軟體。 這樣可提供最高層級的備援和可用性。 多重路徑解決方案必須以 Microsoft 多重路徑 I/O (MPIO)。 儲存體硬體供應商可能提供 MPIO 裝置特定模組 (DSM) 為您的硬體，雖然 Windows Server 2016 和 Windows Server 2019 包含一個或多個 Dsm，做為作業系統的一部分。

## <a name="network-infrastructure-and-domain-account-requirements"></a>網路基礎架構和網域帳戶需求

您將需要下列網路基礎結構的雙節點容錯移轉叢集和下列網域權限的系統管理帳戶：

- **網路設定和 IP 位址：** 當您使用相同的網路介面卡的網路時，也使用相同的通訊設定 （例如，速度、 雙工模式、 流量控制和媒體類型），這些介面卡上。 此外，比較網路介面卡與其所連線之交換器之間的設定，並確保設定之間沒有衝突。

    如果您擁有不會路由到網路基礎結構其餘部分的私人網路，請確定這些私人網路都是使用唯一的子網路。 即使您為每個網路介面卡指定一個唯一的 IP 位址，也必須這樣做。 例如，如果您在使用一個實體網路的總公司中有一個叢集節點，而在使用不同實體網路的分公司中有另一個節點，那麼即使您為每個介面卡指定了一個唯一的 IP 位址，也不要同時為這兩個網路指定 10.0.0.0/24。

    如需詳細的網路介面卡的詳細資訊，請參閱雙節點容錯移轉叢集，稍早在本指南中的硬體需求。

- **DNS:** 叢集中的伺服器必須使用網域名稱系統 (DNS) 進行名稱解析。 可以使用 DNS 動態更新通訊協定。

- **網域角色：** 在叢集中的所有伺服器必須都位於相同的 Active Directory 網域。 最佳作法是所有叢集伺服器都應具有相同的網域角色 (成員伺服器或網域控制站)。 建議的角色是成員伺服器。

- **網域控制站：** 我們建議您叢集的伺服器是成員伺服器。 如有需要，您會需要額外的伺服器做為包含您的容錯移轉叢集的網域的網域控制站。

- **用戶端：** 因為必須測試，您可以將一或多個網路的用戶端連接到您建立時，容錯移轉叢集，並觀察當您移動或容錯移轉到另一個叢集節點的叢集的檔案伺服器的用戶端上的生效。

- **用於管理叢集的帳戶：** 當您第一次建立叢集，或將伺服器新增到它時，您必須登入，使用該叢集中的所有伺服器具有系統管理員權限和權限的帳戶向網域。 帳戶不需要是網域系統管理員帳戶，但可以是位於每個叢集伺服器之 Administrators 群組中的網域使用者帳戶。 此外，如果帳戶不是網域系統管理員帳戶，帳戶 （或群組帳戶的成員） 必須具備**建立電腦物件**並**讀取全部內容**中的權限網域組織單位 (OU)，將位於。

## <a name="steps-for-installing-a-two-node-file-server-cluster"></a>安裝雙節點檔案伺服器叢集的步驟

您必須完成下列步驟來安裝雙節點檔案伺服器容錯移轉叢集。

步驟 1：叢集伺服器連接到網路和儲存體

步驟 2：安裝容錯移轉叢集功能

步驟 3：驗證叢集設定

步驟 4：建立叢集

如果您已安裝在叢集節點，而且想要設定檔案伺服器容錯移轉叢集，請參閱設定雙節點檔案伺服器叢集上，本指南稍後的步驟。

### <a name="step-1-connect-the-cluster-servers-to-the-networks-and-storage"></a>步驟 1：叢集伺服器連接到網路和儲存體

針對容錯移轉叢集網路，避免單一失敗點。 有多種方式來完成這項作業。 您可以連接您的叢集節點的多個不同的網路。 或者，連接您的叢集節點的網路，建構使用小組的網路介面卡、 備援交換器、 備援路由器或可移除單點失敗之類似硬體 （如果您使用 iSCSI 網路，您必須建立此網路在其他網路之外）。

對於雙節點檔案伺服器叢集，當您將伺服器連接至叢集儲存體中，您就必須公開至少兩個磁碟區 (Lun)。 視需要針對要完整測試您的組態，您可以公開其他磁碟區。 不會公開叢集磁碟區無法在叢集中的伺服器。

#### <a name="to-connect-the-cluster-servers-to-the-networks-and-storage"></a>將叢集伺服器連線到網路和儲存體

1. 檢閱網路中的雙節點容錯移轉叢集硬體需求和雙節點容錯移轉叢集，稍早在本指南中的網路基礎結構和網域帳戶需求的相關詳細資料。

2. 連線並設定叢集中的伺服器將使用的網路。

3. 如果您的測試設定中包含用戶端或非叢集的網域控制站，請確定這些電腦可以連線到叢集的伺服器，透過至少一個網路。

4. 請依照製造商的指示，將伺服器實際連線到存放裝置。

5. 請確定您想要在叢集中使用的磁碟 (LUN) 有向您將叢集的伺服器 (而且只有這些伺服器) 公開。 您可以使用任何下列的介面來公開磁碟或 Lun:

    - 存放裝置製造商所提供的介面。

    - 如果您使用 iSCSI，適用的 iSCSI 介面。

6. 如果您已購買可控制格式或功能之磁碟的軟體，請遵循廠商有關如何搭配 Windows Server 中使用該軟體指示。

7. 其中一個您想要叢集，請按一下 開始 的伺服器上 系統管理工具，按一下 電腦管理，然後按一下 磁碟管理。 （如果出現 使用者帳戶控制 對話方塊中，確認顯示的動作是想什麼方法，而且然後按一下 繼續）。在 磁碟管理，請確認叢集磁碟會顯示。

8. 如果您想要擁有大於 2 TB 的存放磁碟區，且要使用 Windows 介面來控制磁碟格式，請將該磁碟轉換成稱為 GUID 磁碟分割表格 (GPT) 的磁碟分割樣式。 若要這樣做，請備份磁碟上的任何資料刪除磁碟上的所有磁碟區，然後，在 磁碟管理，以滑鼠右鍵按一下磁碟 （不是磁碟分割） 並按一下 轉換成 GPT 磁碟。  對於小於 2 TB 的磁碟區，請不要使用 GPT，您可以改為使用稱為主開機記錄 (MBR) 的磁碟分割樣式。

9. 檢查任何已公開磁碟區或 LUN 的格式。 建議您使用 NTFS 格式 （如為見證磁碟，您必須使用 NTFS）。

### <a name="step-2-install-the-file-server-role-and-failover-cluster-feature"></a>步驟 2：安裝檔案伺服器角色和容錯移轉叢集功能

在此步驟中，將安裝檔案伺服器角色和容錯移轉叢集功能。 這兩個伺服器必須執行 Windows Server 2016 或 Windows Server 2019。

#### <a name="using-server-manager"></a>使用伺服器管理員

1. 開啟**伺服器管理員**下方，並在**管理**下拉式清單中，選取**新增角色及功能**。

   ![新增功能](media\Cluster-File-Server\Cluster-FS-Add-Feature.png)

2. 如果**在您開始之前** 視窗隨即開啟，選擇**下一步**。

3. 針對**安裝類型**，選取**角色型或功能型安裝**並**下一步**。

4. 請確定**從伺服器集區選取伺服器**是選取的機器名稱會反白顯示，並**下一步**。

5. 伺服器角色，從清單中的角色，開啟**檔案服務**，選取**檔案伺服器**，並**下一步**。

   ![新增角色](media\Cluster-File-Server\Cluster-FS-Add-FS-Role-1.png)

6. 對於功能，從清單中的功能，請選取**容錯移轉叢集**。  快顯對話方塊會顯示，列出一併安裝的系統管理工具。  保留所有所選選擇**將功能加入**並**下一步**。

   ![新增功能](media\Cluster-File-Server\Cluster-FS-Add-WSFC-1.png)

7. 在 確認 頁面中，選取 安裝。

#### <a name="using-powershell"></a>使用 PowerShell

### <a name="step-3-validate-the-cluster-configuration"></a>步驟 3：驗證叢集設定

建立叢集之前，我們強烈建議您驗證您的設定。 驗證可協助您確認您的伺服器、 網路和儲存體組態符合一組容錯移轉叢集的特定需求。

#### <a name="using-failover-cluster-manager"></a>使用容錯移轉叢集管理員

1. 從**伺服器管理員**，選擇**工具**下拉式清單，然後選取**容錯移轉叢集管理員**。

2. 在 **容錯移轉叢集管理員**，請移至中間的資料行底下**管理**，然後選擇 **驗證設定**。

3. 如果**在您開始之前** 視窗隨即開啟，選擇**下一步**。

4. 在 [**選取伺服器或叢集**] 視窗中，增益集的兩部電腦將成為叢集的節點名稱。  例如，如果名稱是 NODE1 和 NODE2，輸入名稱並選取**新增**。  您也可以選擇**瀏覽**按鈕搜尋 Active Directory 的名稱。  一旦兩者都列在**選取伺服器**，選擇**下一步**。

5. 在 [**測試選項**視窗中，選取**執行所有測試 （建議）**，並**下一步]**。

6. 在 [**確認**] 頁面上，它會提供您的清單中就會檢查的所有測試。  選擇**下一步**測試將會開始。

7. 一旦完成之後，**摘要**頁面隨即出現在測試執行之後。 若要檢視幫助您解譯結果的 [說明] 主題，請按一下**深入了解叢集驗證測試**。

8. 雖然仍在 摘要 頁面中，按一下 檢視報告以了解測試結果。 在組態中進行任何必要變更，然後重新執行測試。 <br>若要關閉精靈之後，請檢視測試結果，請參閱*SystemRoot\Cluster\Reports\Validation Report date and time.html*。

9. 若要檢視叢集驗證的說明主題，您關閉精靈之後，在容錯移轉叢集管理，按一下 說明、 按一下 說明 主題、 按一下 內容 索引標籤、 展開的內容為容錯移轉叢集的說明，請然後按一下 驗證容錯移轉叢集設定.

#### <a name="using-powershell"></a>使用 PowerShell

1. 開啟系統管理的 PowerShell 工作階段，以滑鼠右鍵按一下 [開始] 按鈕，然後選取**Windows PowerShell (Admin)**。

2. 若要驗證機器 （例如，正在 NODE1 和 NODE2 機器名稱） 的容錯移轉叢集，執行命令：

    ```PowerShell
    Test-Cluster -Node "NODE1","NODE2"
    ```
4. 若要關閉精靈之後，請檢視測試結果，請參閱指定的檔案 (在 SystemRoot\Cluster\Reports\)，然後在組態中進行任何必要變更，並重新執行測試。

如需詳細資訊，請參閱 <<c0> [ 驗證容錯移轉叢集設定](https://docs.microsoft.com/previous-versions/windows/it-pro/windows-server-2012-r2-and-2012/jj134244(v=ws.11))。

### <a name="step-4-create-the-cluster"></a>步驟 4：建立叢集

下列將之外建立叢集的機器和您擁有的設定。

#### <a name="using-failover-cluster-manager"></a>使用容錯移轉叢集管理員

1. 從**伺服器管理員**，選擇**工具**下拉式清單，然後選取**容錯移轉叢集管理員**。

2. 在 **容錯移轉叢集管理員**，請移至中間的資料行底下**管理**，然後選擇 **建立叢集**。

3. 如果**在您開始之前** 視窗隨即開啟，選擇**下一步**。

4. 在 [**選取伺服器**] 視窗中，增益集的兩部電腦將成為叢集的節點名稱。  例如，如果名稱是 NODE1 和 NODE2，輸入名稱並選取**新增**。  您也可以選擇**瀏覽**按鈕搜尋 Active Directory 的名稱。  一旦兩者都列在**選取伺服器**，選擇**下一步**。

5. 在 **用於管理叢集的存取點**視窗中，輸入您要將叢集的名稱。  請注意這不是您用來連接到您使用的檔案共用名稱。  這是只管理叢集。

   > [!NOTE]
   > 如果您使用靜態 IP 位址，您必須選取要使用，並輸入叢集名稱，它會使用的 IP 位址的網路。  如果您使用 DHCP 的 IP 位址，IP 位址將會為您自動設定。

6. 選擇**下一步**。

7. 在 [**確認**頁面上，確認內容已設定並選取**下一步]** 來建立叢集。

8. 在 [**摘要**] 頁面上，它會提供您它已建立的組態。  您可以選取 檢視報表，以查看建立的報表。

#### <a name="using-powershell"></a>使用 PowerShell

1. 開啟系統管理的 PowerShell 工作階段，以滑鼠右鍵按一下 [開始] 按鈕，然後選取**Windows PowerShell (Admin)**。

2. 執行下列命令來建立叢集，如果您使用靜態 IP 位址。  例如，電腦名稱為 NODE1 和 NODE2、 叢集的名稱會是叢集中，和 IP 位址會是 1.1.1.1。

   ```PowerShell
    New-Cluster -Name CLUSTER -Node "NODE1","NODE2" -StaticAddress 1.1.1.1
   ```

3. 執行下列命令來建立叢集，如果您使用 DHCP 的 IP 位址。  例如，電腦名稱為 NODE1 和 NODE2，而叢集的名稱會是叢集。

   ```PowerShell    
   New-Cluster -Name CLUSTER -Node "NODE1","NODE2"
   ```

### <a name="steps-for-configuring-a-file-server-failover-cluster"></a>設定檔案伺服器容錯移轉叢集的步驟

若要設定檔案伺服器容錯移轉叢集，請遵循下列步驟。

1. 從**伺服器管理員**，選擇**工具**下拉式清單，然後選取**容錯移轉叢集管理員**。

2. 當容錯移轉叢集管理員 開啟時，它應該會自動將您所建立的叢集名稱。  如果不存在，請移至中間的資料行底下**管理**，然後選擇**連線到叢集**。  輸入您所建立的叢集名稱和**確定**。

3. 在主控台樹狀目錄中，按一下">"的叢集，您建立用來展開其下的項目旁邊。

4. 以滑鼠右鍵按一下**角色**，然後選取**設定角色**。

5. 如果**在您開始之前** 視窗隨即開啟，選擇**下一步**。

6. 在角色的清單中，選擇**檔案伺服器**並**下一步**。

7. 針對檔案伺服器類型中，選取**一般用途檔案伺服器**並**下一步**。<br>向外延展檔案伺服器的相關資訊，請參閱[向外延展檔案伺服器概觀](sofs-overview.md)。

   ![檔案伺服器類型](media\Cluster-File-Server\Cluster-FS-File-Server-Type.png)

8. 在 **用戶端存取點**視窗中，輸入您要使用的檔案伺服器的名稱。  請注意這不是叢集的名稱。  這是檔案共用連線。  例如，如果我想要連線到\\伺服器，則輸入的名稱會是伺服器。

   > [!NOTE]
   > 如果您使用靜態 IP 位址，您必須選取要使用，並輸入叢集名稱，它會使用的 IP 位址的網路。  如果您使用 DHCP 的 IP 位址，IP 位址將會為您自動設定。

6. 選擇**下一步**。

7. 在 **選取儲存體** 視窗中，選取其他磁碟機 （不是見證） 會保留您的共用和**下一步**。

8. 在 [**確認**頁面上，確認您的設定，然後選取**下一步]**。

9. 在 [**摘要**] 頁面上，它會提供您它已建立的組態。  您可以選取 [檢視報表]，請參閱檔案伺服器角色建立的報表。

10. 底下**角色**的主控台樹狀目錄中，您會看到列出為您建立的名稱建立新角色。  使用它反白顯示，底下**動作**右邊的窗格選擇**新增共用**。

11. 透過輸入下列共用精靈 來執行：

    - 它將會是的共用的類型
    - 位置/路徑會是共用的資料夾
    - 共用的使用者名稱將會連接到
    - 其他設定，例如存取型列舉，快取、 加密等等
    - 如果它們將會是預設值以外的檔案層級的權限

12. 在 **確認**頁面上，確認內容已設定並選取**建立**建立檔案伺服器共用。

13. 在 **結果**頁面上，選取 關閉 如果建立共用。  如果它無法建立共用，它會提供您造成的錯誤訊息。

14. 選擇**關閉**。

15. 重複此程序的任何其他的共用。
