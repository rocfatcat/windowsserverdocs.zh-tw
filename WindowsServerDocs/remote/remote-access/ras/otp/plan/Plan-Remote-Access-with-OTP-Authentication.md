# <a name="plan-remote-access-with-otp-authentication"></a>使用 OTP 驗證規劃遠端存取

>適用於：Windows Server (半年度管道)、Windows Server 2016

 Windows Server 2016 和 Windows Server 2012 將 DirectAccess 與路由及遠端存取服務（RRAS） VPN 合併成一個遠端存取角色。 本總覽提供部署單一 Windows Server 2016 或 Windows Server 2012 遠端存取多網站部署所需之設定步驟的簡介。  
  
  
-  步驟 1:[使用 [Advanced] 設定部署單一 DirectAccess 伺服器](https://technet.microsoft.com/windows-server-docs/networking/remote-access/directaccess/single-server-advanced/deploy-a-single-directaccess-server-with-advanced-settings)。 此步驟包含規劃部署單一伺服器所需的基礎結構。 其中包含規劃網路和伺服器設定、憑證需求、DNS 設定、網路位置伺服器部署、DirectAccess 管理伺服器、Active Directory 設定，以及群組原則物件（Gpo）。  
  
-   [步驟 2：規劃 RADIUS 伺服器部署 @ no__t-0  
  
-   [步驟 3：規劃 OTP 憑證部署 @ no__t-0  
  
-   [步驟 4：在遠端存取服務器上規劃 OTP @ no__t-0  
  
完成這些規劃步驟之後，請參閱[使用 OTP 驗證設定遠端存取](https://technet.microsoft.com/windows-server-docs/networking/remote-access/ras/otp/configure/configure-ra-with-otp-authentication)。 如需在實驗室環境中將多網站部署設定為概念證明的詳細資訊，請參閱 @no__t 0Test 實驗室指南：示範使用 OTP 驗證和 RSA SecurID @ no__t-0 的 DirectAccess。
