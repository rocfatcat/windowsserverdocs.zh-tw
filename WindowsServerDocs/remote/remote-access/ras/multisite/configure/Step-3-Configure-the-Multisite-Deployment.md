# <a name="step-3-configure-the-multisite-deployment"></a>步驟 3 設定多站台部署

>適用於：Windows Server （半年通道），Windows Server 2016

設定多站台的基礎結構之後遵循下列步驟來設定遠端存取多站台部署。  
  
|工作|描述|  
|----|--------|  
|3.1. 設定遠端存取伺服器|設定額外的遠端存取伺服器設定的 IP 位址、 將它們加入網域，以及安裝 「 遠端存取 」 角色。|  
|3.2. 授與系統管理員存取權|DirectAccess 系統管理員在其他遠端存取伺服器上授與權限。|  
|3.3. 設定 IP-HTTPS 站台部署|設定多站台部署中使用的 IP-HTTPS 憑證。|  
|3.4. 設定站台部署的網路位置伺服器|設定多站台部署中使用的網路位置伺服器憑證。|  
|3.5. 設定 DirectAccess 用戶端的站台部署|Windows 7 用戶端電腦移除 Windows 8 的安全性群組。|  
|3.6. 啟用多站台部署|啟用多站台部署第一個遠端存取伺服器上。|  
|3.7. 新增多站台部署的進入點|多站台部署中加入其他的進入點。|  
  
> [!NOTE]  
> 本主題包含可讓您用來將部分所述的程序自動化的 Windows PowerShell Cmdlet 範例。 如需詳細資訊，請參閱[使用 Cmdlet](https://go.microsoft.com/fwlink/p/?linkid=230693).  
  
## <a name="BKMK_ConfigServer"></a>3.1. 設定遠端存取伺服器  

  
### <a name="to-install-the-remote-access-role"></a>安裝「遠端存取」角色  
  
1.  請確定每個遠端存取伺服器，設定適當的部署拓撲 (Edge，NAT 背後，單一網路介面)，以及對應的路由。  
  
2.  根據站台拓撲和您的組織的 IP 位址配置每個遠端存取伺服器上設定的 IP 位址。  
  
3.  加入 Active Directory 網域中的每個遠端存取伺服器。  
  
4.  在 [伺服器管理員] 主控台中，在**儀表板**，按一下**新增角色及功能**。  
  
5.   按 [下一步]  3 次，進入伺服器角色選擇畫面。  
  
6.  在 [**選取伺服器角色**對話方塊中，選取**遠端存取**，然後按一下**下一步]** 。  
  
7.  按一下 [**下一步]** 三次。  
  
8.  在 **選取角色服務**對話方塊中，選取**DirectAccess 和 VPN (RAS)** ，然後按一下 **新增功能**。  
  
9.  選取 **路由**，選取**Web 應用程式 Proxy**，按一下 **新增功能**，然後按一下**下一步**。  
  
10. 按 [下一步]  ，然後按一下 [安裝]  。  
  
11.  在 [安裝進度]  對話方塊中，確認安裝成功，然後按一下 [關閉]  。  
  
  
![Windows PowerShell](../../../../media/Step-3-Configure-the-Multisite-Deployment/PowerShellLogoSmall.gif)***<em>Windows PowerShell 對等的命令</em>***  

  
步驟 1-3 必須手動執行，而且不會使用這個 Windows PowerShell cmdlet 完成。  
  
下列 Windows PowerShell Cmdlet 執行與前述程序相同的功能。 在單一行中，輸入各個 Cmdlet (即使因為格式限制，它們可能會在這裡出現自動換行成數行)。  
  
```  
Install-WindowsFeature RemoteAccess -IncludeManagementTools  
```  
  
## <a name="BKMK_Admin"></a>3.2. 授與系統管理員存取權  
  
#### <a name="to-grant-administrator-permissions"></a>若要授與系統管理員權限  
  
1.  遠端存取在伺服器上的其他進入點：在 **開始**畫面上，輸入**電腦管理**，然後按 ENTER 鍵。  
  
2.  在左窗格中，按一下**本機使用者和群組**。  
  
3.  按兩下**群組**，然後按兩下**系統管理員**。  
  
4.  在上**系統管理員屬性** 對話方塊中，按一下**新增**，然後在**選取使用者、 電腦、 服務帳戶或群組** 對話方塊中，按一下  **位置**。  
  
5.  在上**位置**對話方塊中，於**位置**樹狀結構中，按一下包含 DirectAccess 系統管理員的使用者帳戶的位置，然後按一下**確定**。  
  
6.  在 **輸入要選取的物件名稱**，輸入 DirectAccess 系統管理員的使用者名稱，然後按一下**確定**兩次。  
  
7.  在上**系統管理員屬性** 對話方塊中，按一下**確定**。  
  
8.  關閉 [電腦管理] 視窗。  
  
9. 重複此程序，將會屬於多站台部署的所有遠端存取伺服器上。  
  
## <a name="BKMK_IPHTTPS"></a>3.3. 設定 IP-HTTPS 站台部署  
多站台的部署將會加入每個遠端存取伺服器上的 SSL 憑證，才能驗證 IP-HTTPS 網頁伺服器的 HTTPS 連線的。 您至少必須有本機 **Administrators** 群組的成員資格或同等權限，才能完成此程序。  
  
#### <a name="to-obtain-an-ip-https-certificate"></a>若要取得 IP-HTTPS 憑證  
  
1.  每個遠端存取伺服器上：在 **開始**畫面上，輸入**mmc**，然後按 ENTER 鍵。 如果出現 [**使用者帳戶控制**] 對話方塊，請確認它所顯示的動作就是您所需的動作，然後按一下 [**是**]。  
  
2.  按一下 **檔案**，然後按一下**新增/移除嵌入式管理單元**。  
  
3.  按一下 **憑證**，按一下**新增**，按一下 **電腦帳戶**，按一下 **下一步**，選取**本機電腦**，按一下**完成**，然後按一下**確定**。  
  
4.  在 [憑證] 嵌入式管理單元的主控台樹狀目錄中，開啟 [憑證 (本機電腦)\個人\憑證]  。  
  
5.  在 [憑證]  上按一下滑鼠右鍵，指向 [所有工作]  ，然後按一下 [要求新憑證]  。  
  
6.  按兩次 [下一步]  。  
  
7.  在 **要求憑證**頁面上，按一下 Web 伺服器憑證範本，然後按一下**的詳細資訊，才能註冊此憑證**。  
  
    如果 Web 伺服器憑證範本未出現，請確定遠端存取伺服器電腦帳戶已註冊的 Web 伺服器憑證範本的權限。 如需詳細資訊，請參閱 <<c0> [ 設定 Web 伺服器憑證範本的權限](https://technet.microsoft.com/library/ee649249(v=ws.10).aspx)。  
  
8.  上**主旨** 索引標籤**憑證內容**對話方塊中，於**主體名稱**，如**類型**，選取**常見名稱**。  
  
9. 在 **值**，輸入遠端存取伺服器 (例如，「 Europe.contoso.com 」) 的網際網路名稱的完整的網域名稱 (FQDN)，然後按一下**新增**。  
  
10. 依序按一下 [確定]  、[註冊]  ，然後按一下 [完成]  。  
  
11. 在 [憑證] 嵌入式管理單元的詳細資料窗格中，確認已向新的憑證使用 FQDN**預定目的**的**伺服器驗證**。  
  
12. 以滑鼠右鍵按一下憑證，然後按一下**屬性**。  
  
13. 在**易記名稱**，型別**IP-HTTPS 憑證**，然後按一下 **確定**。  
  
    > [!TIP]  
    > 步驟 12 和 13 是選擇性的但讓您更輕鬆地選取 ip-https 憑證，設定遠端存取時。  
  
14. 重複此程序，在您的部署中的所有遠端存取伺服器上。  
  
## <a name="BKMK_NLS"></a>3.4. 設定站台部署的網路位置伺服器  
如果您選取要設定遠端存取伺服器上的網路位置伺服器網站，當您設定您的第一部伺服器時，每個新的遠端存取伺服器，您將必須設定與 Web 伺服器憑證具有相同的主體名稱所選取的 t他的網路位置伺服器的第一部伺服器。 每一部伺服器需要憑證來驗證連線到網路位置伺服器，並位於內部網路的用戶端電腦必須能夠解析在 DNS 中網站的名稱。  
  
#### <a name="to-install-a-certificate-for-network-location"></a>若要安裝的網路位置的憑證  
  
1.  在遠端存取伺服器上：在 **開始**畫面上，輸入**mmc**，然後按 ENTER 鍵。 如果出現 [**使用者帳戶控制**] 對話方塊，請確認它所顯示的動作就是您所需的動作，然後按一下 [**是**]。  
  
2.  按一下 **檔案**，然後按一下**新增/移除嵌入式管理單元**。  
  
3.  按一下 **憑證**，按一下**新增**，按一下 **電腦帳戶**，按一下 **下一步**，選取**本機電腦**，按一下**完成**，然後按一下**確定**。  
  
4.  在 [憑證] 嵌入式管理單元的主控台樹狀目錄中，開啟 [憑證 (本機電腦)\個人\憑證]  。  
  
5.  在 [憑證]  上按一下滑鼠右鍵，指向 [所有工作]  ，然後按一下 [要求新憑證]  。  
  
    > [!NOTE]  
    > 您也可以匯入相同的憑證所用的第一部遠端存取伺服器的網路位置伺服器。  
  
6.  按兩次 [下一步]  。  
  
7.  在 **要求憑證**頁面上，按一下 Web 伺服器憑證範本，然後按一下**的詳細資訊，才能註冊此憑證**。  
  
    如果 Web 伺服器憑證範本未出現，請確定遠端存取伺服器電腦帳戶已註冊的 Web 伺服器憑證範本的權限。 如需詳細資訊，請參閱 <<c0> [ 設定 Web 伺服器憑證範本的權限](https://technet.microsoft.com/library/ee649249(v=ws.10).aspx)。  
  
8.  上**主旨** 索引標籤**憑證內容**對話方塊中，於**主體名稱**，如**類型**，選取**常見名稱**。  
  
9. 在 **值**，輸入網路位置伺服器憑證的第一部遠端存取伺服器 (例如 nls.corp.contoso.com) 已設定的完整的網域名稱 (FQDN)，然後按一下**新增**.  
  
10. 依序按一下 [確定]  、[註冊]  ，然後按一下 [完成]  。  
  
11. 在 [憑證] 嵌入式管理單元的詳細資料窗格中，確認已向新的憑證使用 FQDN**預定目的**的**伺服器驗證**。  
  
12. 以滑鼠右鍵按一下憑證，然後按一下**屬性**。  
  
13. 在 **易記名稱**，型別**網路位置憑證**，然後按一下  **確定**。  
  
    > [!TIP]  
    > 步驟 12 和 13 是選擇性的但讓您更輕鬆地設定遠端存取時，選取 網路位置的憑證。  
  
14. 重複此程序，在您的部署中的所有遠端存取伺服器上。  
  
### <a name="NLS"></a>若要建立網路位置伺服器的 DNS 記錄  
  
1.  在 DNS 伺服器：在 **開始**畫面上，輸入**dnsmgmt.msc**，然後按 ENTER 鍵。  
  
2.  在左窗格中**DNS 管理員**主控台中，開啟內部網路的正向對應區域。 以滑鼠右鍵按一下相關的區域，然後按一下**新的主機 （A 或 AAAA）** 。  
  
3.  在上**新的主機**對話方塊中，於**名稱 （使用父系網域名稱如果空白）** 方塊中，輸入用於第一部遠端存取伺服器的網路位置伺服器的名稱。 在  **IP 位址**方塊中，輸入遠端存取伺服器的內部網路對向 IPv4 位址，然後按一下**新增主機**。 在 [DNS]  對話方塊中，按一下 [確定]  。  
  
4.  在上**新的主機**對話方塊中，於**名稱 （使用父系網域名稱如果空白）** 方塊中，輸入用於第一部遠端存取伺服器的網路位置伺服器的名稱。 在  **IP 位址**方塊中，輸入遠端存取伺服器的內部網路對向 IPv6 位址，然後按一下**新增主機**。 在 [DNS]  對話方塊中，按一下 [確定]  。  
  
5.  針對您的部署中每個遠端存取伺服器重複步驟 3 和 4。  
  
6.  按一下 [完成]  。  
  
7.  重複此程序，再將伺服器新增為部署中的其他進入點。  
  
## <a name="BKMK_Client"></a>3.5. 設定 DirectAccess 用戶端的站台部署  
DirectAccess 的 Windows 用戶端電腦必須是定義其 DirectAccess 關聯安全性群組的成員。 在啟用多站台之前，這些安全性群組可以同時包含 Windows 8 用戶端和 Windows 7 用戶端 （如果已選取適當的 「 舊版 」 模式）。 一旦啟用多站台時，現有用戶端安全性群組，在單一伺服器模式中，會轉換成安全性群組適用於 Windows 8 只。 在啟用多站台之後，DirectAccess 的 Windows 7 用戶端電腦必須移至對應專用 Windows 7 用戶端的安全性群組 （也就是與特定進入點相關聯），或無法透過 DirectAccess 連線。 Windows 7 用戶端必須先移除現有的安全性群組現在是 Windows 8 的安全性群組。 注意：Windows 7 用戶端電腦的 Windows 7 和 Windows 8 的用戶端安全性群組的成員將會遺失遠端連線能力，且未安裝 SP1 的 Windows 7 用戶端將失去公司的連線能力。 因此，所有的 Windows 7 用戶端電腦必須從 Windows 8 安全性群組移除。  
  
#### <a name="remove--windows-7--clients-from-windows-8-security-groups"></a>從 Windows 8 安全性群組中移除 Windows 7 用戶端  
  
1.  在主要網域控制站中，按一下**開始**，然後按一下**Active Directory 使用者和電腦**。  
  
2.  若要移除的安全性群組中的電腦，請按兩下 安全性群組，然後在 **< 群組名稱 > 屬性** 對話方塊中，按一下**成員** 索引標籤。  
  
3.  選取 Windows 7 用戶端電腦，然後按一下**移除**。  
  
4.  重複此程序來移除 Windows 7 用戶端電腦從 Windows 8 的安全性群組。  
  
> [!IMPORTANT]  
> 啟用遠端存取多站台的設定時 （Windows 7 和 Windows 8） 的所有用戶端電腦將會遺失遠端連線，直到能夠連線到公司網路直接或透過 VPN，以更新其群組原則。 這是，則為 true，當第一次，啟用多站台的功能，以及停用多站台時。  
  
## <a name="BKMK_Enable"></a>3.6. 啟用多站台部署  
若要設定站台部署，讓您現有的遠端存取伺服器上的多站台的功能。 之前啟用多站台部署中，請確定您具有下列資訊：  
  
1.  全域負載平衡器的設定和 IP 位址，如果您想要負載平衡 DirectAccess 用戶端連線到您的部署中的所有進入點。  
  
2.  如果您想要啟用遠端存取適用於 Windows 7 的用戶端電腦，安全性會群組包含 Windows 7 用戶端電腦，在您部署中，於第一個進入點。  
  
3.  群組原則物件名稱，如果您需要使用非預設群組原則物件，如果您需要支援 Windows 7 用戶端電腦，您的部署中的第一個進入點的 Windows 7 用戶端電腦上套用。  
  
### <a name="EnabledMultisite"></a>若要啟用多站台設定  
  
1.  在您現有的遠端存取伺服器：在 **開始**畫面上，輸入**RAMgmtUI.exe**，然後按 ENTER 鍵。 如果出現 [**使用者帳戶控制**] 對話方塊，請確認它所顯示的動作就是您所需的動作，然後按一下 [**是**]。  
  
2.  在 遠端存取管理主控台中，按一下**組態**，然後在**工作**窗格中，按一下 **啟用多站台**。  
  
3.  在 [**啟用多站台部署**精靈**在您開始前**頁面上，按一下**下一步]** 。  
  
4.  在 **部署名稱**頁面上，於**多站台的部署名稱**，輸入您的部署的名稱。 在 [**第一個進入點名稱**，輸入名稱來識別這是目前的遠端存取伺服器，第一個進入點，然後按一下**下一步]** 。  
  
5.  在 **進入點選取**頁面上，執行下列其中之一：  
  
    -   按一下 **自動指派的進入點，並允許用戶端手動選取**來自動路由傳送用戶端電腦最適合的進入點，同時也讓用戶端電腦手動選取的進入點。 手動輸入的點選取範圍是僅適用於 Windows 8 電腦。 按一下 [下一步]  。  
  
    -   按一下 [**自動指派的進入點**來進行自動路由傳送用戶端電腦至最適合的進入點，然後按一下**下一步]** 。  
  
6.  在 **全域負載平衡**頁面上，執行下列其中之一：  
  
    -   按一下 [**否，不要使用全域負載平衡**如果您不想要使用全域負載平衡，然後按一下**下一步]** 。  
  
        > [!NOTE]  
        > 當選取此選項的用戶端電腦自動連線到其最接近的進入點。  
  
    -   按一下 **可以，請使用全域負載平衡**如果您想要載入平衡所有進入點的全域之間的流量。 在 [**輸入全域負載平衡可供所有進入點的 FQDN**，輸入全域負載平衡 FQDN，然後在**輸入全域負載平衡 IP 位址，此進入點的**，其中包含第一個遠端存取伺服器，輸入全域負載平衡 IP 位址，對於此進入點，然後再按一下**下一步]** 。  
  
7.  在 **用戶端支援**頁面上，執行下列其中之一：  
  
    -   若要限制存取執行 Windows 8 或更新版本的作業系統的用戶端電腦，請按一下**限制存取執行 Windows 8 或更新版本作業系統的用戶端電腦**，然後按一下**下一步**。  
  
    -   若要讓用戶端執行 Windows 7，以存取此進入點的電腦，按一下**讓用戶端電腦執行 Windows 7，以存取此進入點**，然後按一下**新增**。 在上**選取群組** 對話方塊中，選取的安全性群組，其中包含 Windows 7 用戶端電腦中，按一下**確定**，然後按一下**下一步**。  
  
8.  在 **用戶端 GPO 設定**頁面上，接受預設 GPO 適用於 Windows 7 用戶端電腦，此進入點，輸入想要自動建立，或按一下 遠端存取 GPO 名稱**瀏覽**至尋找 GPO 適用於 Windows 7 的用戶端電腦，並按一下**下一步**。  
  
    > [!NOTE]  
    > -   **用戶端 GPO 設定**會出現頁面只有當您設定讓 Windows 7 用戶端電腦存取的進入點的進入點。  
    > -   您可以選擇性地按一下**驗證的 Gpo**以確保您擁有此進入點的所選的 GPO 或 Gpo 的適當權限。 如果 GPO 不存在，就會自動建立，然後建立並連結不需要權限。 Gpo 已手動建立的情況下，然後編輯、 修改安全性，並不需要刪除權限。  
  
9. 在 **摘要**頁面上，按一下**認可**。  
  
10. 在 **啟用多站台部署** 對話方塊中，按一下**關閉**然後在 啟用多站台部署精靈 中，按一下 **關閉**。  
  
![Windows PowerShell](../../../../media/Step-3-Configure-the-Multisite-Deployment/PowerShellLogoSmall.gif)***<em>Windows PowerShell 對等的命令</em>***  
  
下列 Windows PowerShell Cmdlet 執行與前述程序相同的功能。 在單一行中，輸入各個 Cmdlet (即使因為格式限制，它們可能會在這裡出現自動換行成數行)。  
  
若要啟用多站台部署，名為 'Contoso' 的第一個進入點上名為 ' Edge1-US '。 部署可讓用戶端，以手動方式選取 進入點，並不會使用全域負載平衡器。  
  
```  
Enable-DAMultiSite -Name 'Contoso' -EntryPointName 'Edge1-US' -ManualEntryPointSelectionAllowed 'Enabled'  
```  
  
若要允許透過安全性群組 DA_Clients_US 的第一個進入點的 Windows 7 用戶端電腦存取和使用 GPO DA_W7_Clients_GPO_US。  
  
```  
Add-DAClient -EntrypointName 'Edge1-US' -DownlevelSecurityGroupNameList @('corp.contoso.com\DA_Clients_US') -DownlevelGpoName @('corp.contoso.com\DA_W7_Clients_GPO_US)  
```  
  
## <a name="BKMK_EntryPoint"></a>3.7. 新增多站台部署的進入點  
啟用多站台部署中之後, 您可以新增其他的進入點使用 [新增進入點] 精靈。 之前新增進入點，請確定您具有下列資訊：  
  
-   全域負載平衡器 IP 位址，每個新項目點，如果您使用全域負載平衡。  
  
-   包含您想要啟用遠端存取適用於 Windows 7 的用戶端電腦才會加入每個進入點的 Windows 7 用戶端電腦的安全性群組。  
  
-   群組原則物件名稱，如果您需要使用非預設群組原則物件，如果您需要支援 Windows 7 用戶端電腦將會新增，每個進入點的 Windows 7 用戶端電腦上套用。  
  
-   在組織的網路中部署 IPv6 時的情況下，您必須準備新的進入點的 IP-HTTPS 首碼。  
  
### <a name="AddEP"></a>若要新增至您的多站台部署的進入點  
  
1.  在您現有的遠端存取伺服器：在 **開始**畫面上，輸入**RAMgmtUI.exe**，然後按 ENTER 鍵。 如果出現 [**使用者帳戶控制**] 對話方塊，請確認它所顯示的動作就是您所需的動作，然後按一下 [**是**]。  
  
2.  在 遠端存取管理主控台中，按一下**組態**，然後在**工作**窗格中，按一下 **新增進入點**。  
  
3.  在 新增項目，請點上的精靈中，**項目點的詳細資料**頁面上，於**遠端存取伺服器**，輸入要新增之伺服器的完整的網域名稱 (FQDN)。 在 [**進入點名稱**，輸入進入點的名稱，然後按一下**下一步]** 。  
  
4.  在 [**全域負載平衡設定**頁面上，輸入全域負載平衡的這個進入點的 IP 位址，然後按一下**下一步]** 。  
  
    > [!NOTE]  
    > **全域負載平衡設定**只有當多站台的設定使用全域負載平衡器時，才會出現的頁面。  
  
5.  在 **網路拓樸**頁面上，按一下對應的遠端存取伺服器，您要新增，然後按一下 網路拓撲的拓撲**下一步**。  
  
6.  在上**網路名稱或 IP 位址**頁面上，於**中的公用名稱或用戶端用來連線到遠端存取伺服器的 IP 位址類型**，輸入公用名稱或用戶端用來連接到的 IP 位址遠端存取伺服器。 與 IP-HTTPS 憑證的主體名稱，對應的公用名稱。 在邊緣網路拓撲實作的位置的情況下，IP 位址是遠端存取伺服器的外部介面卡。 按一下 [下一步]  。  
  
7.  在 **網路介面卡**頁面上，執行下列動作之一：  
  
    -   如果您要在兩個網路介面卡，部署拓樸**外部介面卡**，選取 連線到外部網路介面卡。 在 **內部介面卡**，選取 連線到內部網路介面卡。  
  
    -   如果您要在具有一個網路介面卡，部署拓樸**網路介面卡**，選取 連線到內部網路介面卡。  
  
8.  在 **網路介面卡**頁面上，於**選取用來驗證 IP-HTTPS 連線的憑證**，按一下 **瀏覽**找出並選取 IP-HTTPS 憑證。 按一下 [下一步]  。  
  
9. 如果 IPv6 已在公司網路上，在**前置詞組態**頁面上，於**指派給用戶端電腦的 IPv6 首碼**，輸入將 IPv6 位址指派給 DirectAccess 用戶端的 IP-HTTPS 首碼電腦，然後按一下 [**下一步]** 。  
  
10. 在 **用戶端支援**頁面上，執行下列其中之一：  
  
    -   若要限制存取執行 Windows 8 或更新版本的作業系統的用戶端電腦，請按一下**限制存取執行 Windows 8 或更新版本作業系統的用戶端電腦**，然後按一下**下一步**。  
  
    -   若要讓用戶端執行 Windows 7，以存取此進入點的電腦，按一下**讓用戶端電腦執行 Windows 7，以存取此進入點**，然後按一下**新增**。 在上**選取群組**對話方塊方塊中，選取包含 Windows 7 用戶端電腦會連線到此進入點，請按一下 [安全性群組 **[確定]** ，然後按一下**下一步]** .  
  
11. 在 **用戶端 GPO 設定**頁面上，接受預設 GPO 適用於 Windows 7 用戶端電腦，此進入點，輸入您想要自動建立，或按一下 遠端存取 GPO 名稱**瀏覽**至找出 GPO 適用於 Windows 7 的用戶端電腦，然後按一下**下一步**。  
  
    > [!NOTE]  
    > -   **用戶端 GPO 設定**會出現頁面只有當您設定讓 Windows 7 用戶端電腦存取的進入點的進入點。  
    > -   您可以選擇性地按一下**驗證的 Gpo**以確保您擁有此進入點的所選的 GPO 或 Gpo 的適當權限。 如果 GPO 不存在，就會自動建立，然後建立並連結不需要權限。 Gpo 已手動建立的情況下，然後編輯、 修改安全性，並不需要刪除權限。  
  
12. 在 **伺服器 GPO 設定**頁面上，接受預設的 GPO，此遠端存取伺服器，輸入您想要自動建立，或按一下 遠端存取 GPO 名稱**瀏覽**找出的 GPO此伺服器，然後再按一下**下一步**。  
  
13. 在上**網路位置伺服器**頁面上，按一下**瀏覽**若要選取遠端存取伺服器上所執行的網路位置伺服器網站憑證，然後按一下**下一步**.  
  
    > [!NOTE]  
    > **網路位置伺服器**只有網路位置伺服器網站在執行時在遠端存取伺服器上會出現的頁面。  
  
14. 在 **摘要**頁面上，檢閱項目點設定，然後按一下 **認可**。  
  
15. 在 **新增進入點** 對話方塊中，按一下**關閉**然後在 新增項目點精靈 中，按一下 **關閉**。  
  
    > [!NOTE]  
    > 如果已新增的進入點是在不同的樹系中，比現有的進入點或用戶端電腦，則必須按一下 **重新整理管理伺服器**中**工作**窗格，即可探索網域控制站和 System Center Configuration Manager 在新樹系中。  
  
16. 重複此程序步驟 2 中的每個您想要新增至您的多站台部署的進入點。  
  
![Windows PowerShell](../../../../media/Step-3-Configure-the-Multisite-Deployment/PowerShellLogoSmall.gif)***<em>Windows PowerShell 對等的命令</em>***  
  
下列 Windows PowerShell Cmdlet 執行與前述程序相同的功能。 在單一行中，輸入各個 Cmdlet (即使因為格式限制，它們可能會在這裡出現自動換行成數行)。  
  
若要從 corp2 網域做為第二個進入點新增電腦 edge2 名為 Edge2 歐洲。 進入點設定是： 用戶端 IPv6 首碼 ' 2001:db8:2:2000:: / 64'，連接至地址 （edge2 電腦上的 IP-HTTPS 憑證） 'edge2.contoso.com'，伺服器 GPO，名為 「 DirectAccess 伺服器設定-Edge2-歐洲 」，以及內部和外部介面名稱為網際網路和 Corpnet2 分別所示：  
  
```  
Add-DAEntryPoint -RemoteAccessServer 'edge2.corp2.corp.contoso.com' -Name 'Edge2-Europe' -ClientIPv6Prefix '2001:db8:2:2000::/64' -ConnectToAddress 'Europe.contoso.com' -ServerGpoName 'corp2.corp.contoso.com\DirectAccess Server Settings - Edge2-Europe' -InternetInterface 'Internet' -InternalInterface 'Corpnet2'  
```  
  
若要允許透過安全性群組 DA_Clients_Europe 的第二個進入點的 Windows 7 用戶端電腦存取和使用 GPO DA_W7_Clients_GPO_Europe。  
  
```  
Add-DAClient -EntrypointName 'Edge2-Europe' -DownlevelGpoName @('corp.contoso.com\ DA_W7_Clients_GPO_Europe') -DownlevelSecurityGroupNameList @('corp.contoso.com\DA_Clients_Europe')  
```
