# 步驟 4. 安裝和設定網路原則伺服器 (NPS)

>   適用於： Windows Server 2019、 Windows Server （半年度管道）、 Windows Server 2016、 Windows Server 2012 R2、 Windows 10


& #171; [**下一步：** 步驟 3。設定 Always On VPN 的遠端存取伺服器](vpn-deploy-ras.md)<br>
& #187; [**下一步：** 步驟 5。設定 DNS 和防火牆設定](vpn-deploy-dns-firewall.md)


在此步驟中，您可以安裝網路原則伺服器 (NPS) 的 VPN 伺服器所傳送的連線要求處理：

- 執行授權，以確認使用者有權限連線。
- 執行以驗證使用者的身分識別驗證。
- 執行計量，以記錄您選擇在 NPS 中設定 RADIUS 帳戶處理時的連線要求的層面。

在本節中的步驟讓您完成下列項目：

1.  在電腦或已計劃 NPS 伺服器，且安裝在您的組織或公司網路上的 VM，您可以安裝 NPS。

   >[!TIP] 
   >如果您已經在您網路上有一或多個 NPS 伺服器，您不需要執行 NPS 伺服器安裝-相反地，您可以使用本主題來更新現有的 NPS 伺服器的設定。

>[!NOTE]  
您可以在 Windows Server Core 安裝網路原則伺服器服務。

2.  在公司的組織/NPS 伺服器上，您可以設定 NPS 作為 RADIUS 伺服器處理從 VPN 伺服器收到連線要求執行。

## 安裝網路原則伺服器

在此程序，您可以安裝 NPS 藉由使用 Windows PowerShell 或伺服器管理員新增角色與功能精靈。 NPS 是網路原則與存取服務伺服器角色的角色服務。

>[!TIP] 
>根據預設，NPS 接聽連接埠 1812年、 1813年、 1645年和 1646年上所有已安裝的網路介面卡上的 RADIUS 流量。 當您安裝 NPS，並啟用具有進階安全性的 Windows 防火牆時，防火牆例外，為這些連接埠取得自動建立 IPv4 和 IPv6 的流量。 如果您的網路存取伺服器設定為將 RADIUS 流量傳送這些預設值以外的連接埠，移除 NPS 在安裝期間，建立具有進階安全性的 Windows 防火牆中的例外狀況，並建立適用於您使用的連接埠的例外狀況RADIUS 流量。

**適用於 Windows PowerShell 的程序：**

若要使用 Windows PowerShell 中，以系統管理員，執行 Windows PowerShell 來執行此程序中輸入下列命令，，然後按 enter 鍵。

`Install-WindowsFeature NPAS -IncludeManagementTools`

    >[!NOTE]

2.  在 [選取目的地伺服器，請確定已選取 [**選取伺服器從伺服器集區**。

## 設定 NPS

安裝 NPS，設定 NPS 以處理所有驗證，授權，並計量執行任務適用於連線要求它之後會收到來自 VPN 伺服器。

### 在 Active Directory 中登錄 NPS 伺服器

在此程序，您註冊 Active Directory 中的伺服器，使其在處理連線要求的使用者帳戶資訊的存取權限。

**程序：**

1.  在伺服器管理員中，按一下 [**工具**]，，然後按一下 [**網路原則伺服器**。 NPS 主控台會開啟。

2.  在 NPS 主控台中，以滑鼠右鍵按一下**NPS （本機）**，然後按一下來選取它的**註冊 Active Directory 中的伺服器**。<p>網路原則伺服器] 對話方塊隨即開啟。

3.  在網路原則伺服器] 對話方塊中，按兩次 **[確定]** 。

如需登錄 NPS 的替代方法，請參閱[登錄 NPS 伺服器在 Active Directory 網域中](../../../../../networking/technologies/nps/nps-manage-register.md)。

### 設定網路原則伺服器計量

在此程序，設定網路原則伺服器計量使用其中一個下列記錄類型：

-   **事件記錄**。 主要用於稽核和疑難排解連線的嘗試次數。 您可以設定 NPS 事件記錄取得 NPS 伺服器內容在 NPS 主控台。

-   **記錄使用者驗證及帳戶處理要求至本機檔案**。 主要用於連線分析及計費目的。 因為它提供您追蹤的惡意使用者活動，攻擊後的一種方法也用做為安全性調查工具。 您可以設定使用計量設定精靈的本機檔案記錄。

-   **使用者驗證及帳戶處理要求至 Microsoft SQL Server XML 相容的資料庫記錄**。 用來讓多個伺服器執行 NPS 有一個資料來源。 也會提供使用關聯式資料庫的優點。 您可以藉由使用計量設定精靈設定 SQL Server 記錄。

若要設定網路原則伺服器計量，請參閱[設定網路原則伺服器計量](../../../../../networking/technologies/nps/nps-accounting-configure.md)。

### 新增 VPN 伺服器作為 RADIUS 用戶端

在[設定遠端存取伺服器適用於 Always On VPN]](vpn-deploy-ras.md)區段中，您可以安裝並設定您的 VPN 伺服器。 VPN 伺服器在設定期間，您可以新增半徑共用的密碼 VPN 伺服器上。 

在此程序，您可以使用相同的共用的祕密文字字串為 RADIUS 用戶端在 NPS 中設定 VPN 伺服器。 使用相同的文字字串，您使用的 VPN 伺服器上，或 NPS 伺服器和 VPN 伺服器之間的通訊失敗。

>[!IMPORTANT] 
>當您新增至您網路的新網路存取伺服器 （VPN 伺服器、 無線存取點，驗證參數或撥號伺服器） 時，您必須將新增伺服器為 RADIUS 用戶端在 NPS 中以便 NPS 感知的並可以與 「 網路存取 」 伺服器通訊。

**程序：**

1.  在 NPS 伺服器上，在 NPS 主控台中，按兩下**RADIUS 用戶端和伺服器**。

2.  **RADIUS 用戶端**上按一下滑鼠右鍵，然後按一下 [**新增]**。 新的 RADIUS 用戶端] 對話方塊隨即開啟。

3.  確認已選取 [**啟用此 RADIUS 用戶端**] 核取方塊。

4.  **易記名稱**，在輸入 VPN 伺服器的顯示名稱。

5.  在**位址 （IP 或 DNS）** 中，輸入 NAS IP 位址或 FQDN。<p>如果您輸入的 FQDN，按一下 [**驗證**，如果您想要確認名稱正確且對應至有效的 IP 位址。

6.  在**共用密碼**，請執行：

    1.  確定已選取 [**手動**。

    2.  輸入也輸入 VPN 伺服器的強式的文字字串。

    3.  重新輸入中確認共用密碼的共用的密碼。

7.  按一下 **\[確定\]**。 VPN 伺服器會出現在 NPS 伺服器上設定 RADIUS 用戶端的清單。

## 設定 NPS 做為半徑，適用於 VPN 連線

在此程序，您 NPS 作為 RADIUS 伺服器上設定您組織的網路。 在 NPS 中，您必須定義只允許使用者在透過 VPN 伺服器-存取組織/公司網路特定群組原則，然後使用有效的使用者憑證中 PEAP 驗證要求時，才。

**程序：**

1.  在 NPS 主控台中，在標準設定中，請確定已選取 [**適用於撥號或 VPN 連線的 RADIUS 伺服器**。

2.  按一下 [**設定 VPN 或撥號**。<p>設定 VPN 或撥號精靈會開啟。

3.  按一下**虛擬私人網路 (VPN) 連線**，然後按一下 [**下一步**。

4.  在指定撥號或 VPN 伺服器，在 RADIUS 用戶端中，選取您在上一個步驟加入的 VPN 伺服器的名稱。<p>例如，如果您的 VPN 伺服器 NetBIOS 名稱 RAS1，請按一下**RAS1**。

5.  按 **[下一步]**。

6.  在 [設定驗證方法，請完成下列步驟：

    1.  清除**Microsoft 加密驗證版本 2 (Ms-chapv2)** 核取方塊。

    2.  按一下 [**可延伸驗證通訊協定**核取方塊來選取它。

    3.  在類型 （根據存取和網路設定的方法） 中，按一下**Microsoft： 受保護的 EAP (PEAP)**，按一下 [**設定**。<p>編輯受保護的 EAP 內容] 對話方塊隨即開啟。

    4.  按一下 [**移除**] 來移除安全密碼 (EAP MSCHAP v2) EAP 類型。

    5.  按一下 **\[新增\]**。 新增 EAP] 對話方塊隨即開啟。

    6.  按一下**智慧卡或其他憑證**，然後按一下 **[確定]**。

    7.  按一下 **[確定]** 以關閉編輯受保護的 EAP 內容。

7.  按 **[下一步]**。

8.  在指定的使用者群組，請完成下列步驟：

    1.  按一下 **\[新增\]**。 [選取使用者、 電腦、 服務帳戶或群組對話方塊隨即開啟。

    2.  輸入**VPN 使用者**，然後按一下 **[確定]**。

    3.  按 **[下一步]**。

9.  在指定的 IP 篩選器，按一下 [**下一步**。

10. 在指定的加密設定] 中，按一下 [**下一步**。 不進行任何變更。<p>這些設定僅適用於這個案例中不支援的 Microsoft 點對點加密 (PPTP) 連線。

11. 在指定領域名稱中，按一下 [**下一步**。

12. 按一下 [**完成**] 以關閉精靈。

## 自動註冊 NPS 伺服器憑證

此程序，在您手動重新整理群組原則在本機 NPS 伺服器。 當群組原則重新整理，如果憑證自動註冊設定，並正確運作，在本機電腦是自動註冊憑證由憑證授權單位 (CA)。

>[!NOTE]  
>群組原則自動重新整理重新啟動您的網域成員電腦，或當使用者登入網域成員電腦。 此外，群組原則會定期重新整理。 根據預設，此定期重新整理會發生以隨機位移最多 30 分鐘的時間的每個 90 分鐘。

**系統管理員**或對等中的成員資格的最小才能完成此程序。

**程序：**

1.  在 NPS 上, 開啟 Windows PowerShell。

2.  Windows PowerShell 命令提示字元中，輸入**gpupdate**，並按 enter 鍵。
