# <a name="selecting-hardware-for-your-multipoint-services-system"></a>選取適用於 MultiPoint 服務系統的硬體
當您建立 MultiPoint 服務系統時，您應該選取符合 Windows Server 2016 系統需求的電腦。 如果您決定要選取哪些元件，請考慮下列事項：  
  
-   完整解決方案的目標價格範圍。  
  
-   您可能會預期的 MultiPoint 服務系統使用案例類型，例如使用者是否正在執行多媒體程式、使用文字處理或生產力程式，或流覽網際網路。  
  
-   您的案例是否有大型的處理或記憶體需求。  
  
-   可以同時使用系統的使用者數目。 如果您打算同時在系統上有許多使用者，或使用系統密集程式的使用者，您應該為系統規劃更多運算能力。  
  
-   工作站的類型。 您需要多少 USB 埠或影片埠？  
  
-   未來的擴充計畫。 您是否計畫在稍後將工作站新增至 MultiPoint 服務系統？ 您是否有足夠的視訊卡位置、USB 埠或網路點擊？ 您的硬體需要支援多少個額外的使用者？  
  
-   實體版面配置。 如需詳細資訊，請參閱[MultiPoint 服務網站規劃](MultiPoint-services-Site-Planning.md)。  
  
MultiPoint 服務系統通常包含下列元件：  
  
-   一部執行 MultiPoint 服務的電腦，其中包括 CPU、RAM、硬碟機和視訊卡。  
  
-   適用于每部工作站的監視器、工作站集線器、鍵盤及滑鼠。  
  
-   適用于 MultiPoint 服務工作站的選用週邊裝置，包括僅供工作站使用者使用的喇叭、耳機、麥克風或存放裝置。  
  
-   可供 MultiPoint 服務系統的所有使用者使用的選用周邊裝置，直接連接到主機電腦，例如印表機、外部硬碟機和 USB 存放裝置。  
  
使用下列資訊來進行硬體決策：  
  
-   [選取 CPU](#selecting-a-cpu)  
-   [選取硬體元件](#selecting-hardware-components)  
  
## <a name="selecting-a-cpu"></a>選取 CPU  
MultiPoint 服務系統是多個\-使用者環境，所有使用者都連接到單一主機電腦。 這會增加 CPU 使用量，因為所有使用者都共用相同的電腦。 某些工作（例如多媒體程式\(、媒體播放機或影片\-編輯軟體\)）有較大的處理需求。 因此，請務必選取可處理使用者人數和所需支援之使用者案例類型處理需求的 CPU。  
  
MultiPoint 服務需要以 x64\-為基礎的 CPU，而且必須符合電腦的系統需求，如[硬體需求和效能建議](Hardware-Requirements-and-Performance-Recommendations.md)中所述。  
  
下列的處理器類型已經過測試，可用於具有高\-需求處理常式（例如多媒體程式）的 MultiPoint 服務系統：  
  
-   **雙\-核心處理器：** 最多可支援8個工作站。  
-   **四\-核心處理器：** 最多可支援16個工作站。
-   **具有\-多執行緒的四核心處理器：** 最多可支援20個工作站。      
-   **具有\-多執行緒的六個核心處理器：** 最多可支援24個工作站。  
  
使用此資訊，選取符合 MultiPoint 服務系統處理需求的 CPU。 
> [!NOTE] 
> 如果您正在執行大量影片的應用程式，建議為每部工作站至少有一個核心。

## <a name="selecting-video-hardware"></a>選取影片硬體
您所選取的影片硬體應該支援您想要在 MultiPoint 服務工作站上工作的使用者數目所需的監視器數目。 此外，不同類型的影片硬體可以為圖形\- \-密集的程式（例如多媒體內容）提供較高的效能解決方案。  
  
針對 MultiPoint 服務系統所需的效能類型，選取可支援監視器數目上限的影片硬體。 請確定您已驗證您選擇的影片硬體效能，以確保它符合您的效能需求。  
  
> [!NOTE]  
> 您必須安裝支援將桌面延伸到多個監視器的視頻驅動程式。  
  
影片硬體選項包括：  
  
-   使用 PCI 或 PCIe 匯流排界面的內部視訊卡  
  
-   透過 USB 連接的外部視訊控制器  
  
下列各節說明這些影片硬體類型的各項功能。 您可以結合內部視訊卡和外部視訊控制器，以建立您想要的系統。  
  
### <a name="internal-video-cards"></a>內部視訊卡  
內部視訊卡已插入\-電腦上的主機板。 內部視訊卡是一種解決方案，可協助圖形\-密集多媒體程式的效能。 不過，內部視訊卡需要可用的 PCI 或 PCIe 插槽才能插入\-主機板。 許多高效\-能視訊卡都需要有一個 PCIe 插槽，但在主機板上有有限數量的 pcie 插槽。 您應該知道電腦上有何種視訊卡位置可供使用，以便購買正確的視訊卡類型。  
  
可以連接到每張視訊卡的監視器數目取決於卡片上使用的 GPU 和它支援的埠數目，其範圍通常是從2到6。  
  
當您選取 [內部] 視訊卡時，請選取支援所需監視器數目的視訊卡，以建立所需的直接視頻連接工作站數目。 可支援的監視器數目上限等於連接\-至主機板的內部視訊卡數目，乘以每個視訊卡上的監視器埠數目來表示。 例如，如果您有兩張內部視訊卡，且每張卡片都有兩個監視器埠，則最多可支援四個監視器。    
  
### <a name="external-video-controllers"></a>外部影片控制器  
USB 零用戶端包含外部視訊控制器，可將監視器連接到用戶端。 USB 零用戶端也可能包含耳機、喇叭、麥克風或其他週邊裝置的連線。  
  
如果您想要在不開啟電腦的情況下啟用其他監視的支援，或如果您想要支援的工作站超過可用的影片輸出，請選取 USB 零用戶端。 例如，如果您先前已將四個監視器\-插入內部視訊卡，而您想要新增兩個監視器，可以將兩個\-外部視訊控制器插到電腦上，並有兩個以上監視器的空間。 如此一來，您可以結合 USB 0 用戶端與影片控制器，而不使用主機板上的額外 PCI 或 PCIe 插槽。  
  
## <a name="BKMK_Selectingdirect-video-connectedorUSBzeroclientstationdevices"></a>選取直接\-視頻\-連接或 USB 零用戶端工作站裝置  
MultiPoint 服務站是由一部使用鍵盤和滑鼠插頭\-的「工作站集線器」或「USB 零」用戶端所組成，以及一部插\-到主機電腦或 usb 零用戶端的監視器。 其他週邊裝置可以插入\-至工作站集線器或 USB 零用戶端，但不需要建立 MultiPoint 工作站。 這些其他週邊裝置會在[選取其他工作站的裝置](#selecting-other-station-peripheral-devices)中說明。  
  
您選取要建立 MultiPoint 服務工作站的裝置，應符合使用 MultiPoint 服務的最低需求。 本主題提供有關下列 MultiPoint 服務工作站裝置需求的詳細資料：  
  
-   [選取 USB 集線器](#selecting-usb-hubs)  
-   [選取 USB 零用戶端](#selecting-usb-zero-clients)  
-   [選取鍵盤和滑鼠裝置](#selecting-keyboards-and-mouse-devices)  
-   [選取監視](#selecting-monitors)  
  
### <a name="selecting-usb-hubs"></a>選取 USB 集線器  
MultiPoint 服務系統中所使用的 USB 集線器可以是一般的 USB 集線器。 這類中樞通常會有四個以上的 USB 埠，並允許多個 USB 裝置連接到電腦上的單一 USB 埠。 有些其他裝置（例如鍵盤和視頻監視器）也可以將 USB 集線器納入其設計中。  
  
另一個考慮是使用*外部供電*的集線器，而不是*匯流排\-驅動*的集線器。 透過匯流排\-驅動的集線器，主機電腦提供的目前數量必須足以為插入\-中樞的所有週邊裝置提供電源，而不會降低系統效能。 外部供電的中樞可讓您連接更多的周邊裝置，並為其提供足夠的耗電量。 使用外部供電的集線器有助於防止效能問題、埠失敗和其他間歇問題。  
  
為 MultiPoint 服務系統選取 USB 集線器時，請考慮使用它。 中樞可用來做為*工作站中樞*、*中繼集線器*或*下游中樞*。 請參閱下表，以取得有關每個中樞類型的說明。 我們建議所有 USB 裝置都是 USB 2.0 或更新版本。
  
||啟動|  
|-|-----------|  
|站中樞|可以是匯流排\-電源，除非\-有高速的裝置插入\-電源，或下游中樞會連線到它|  
|中繼中樞 |應為外部電源|  
|下游中樞|可以根據插入\-中樞的裝置，從外部供電或匯流排供電|  
|作用中的 USB 擴充項纜線|包含 USB 集線器的作用中 USB 纜線通常是匯流排供電;因此，不建議將工作站連接到電腦。|  
  
### <a name="selecting-usb-zero-clients"></a>選取 USB 零用戶端  
USB 零用戶端是包含影片輸出的 USB 集線器。 因此，它可讓監視器透過 USB 連線連接到電腦。 如需針對影片使用 USB 零用戶端的詳細資訊，請參閱本檔中的[選取影片硬體](#selecting-video-hardware)。 Usb 零用戶端也可以啟用將各種 usb 和非\-usb 裝置連接到中樞的連線。 USB 零用戶端是由特定硬體製造商所產生，而且需要安裝裝置\-特定驅動程式。  
  
### <a name="selecting-keyboards-and-mouse-devices"></a>選取鍵盤和滑鼠裝置  
您插入\-工作站的鍵盤和滑鼠裝置通常會是 USB 裝置。 有些 USB 零用戶端提供\/ps 2 埠，在此情況下，鍵盤和滑鼠應該使用\/ps 2 來連線到站中樞。 如果\/您要設定 ps\/2 direct\-video\-連線的站，也可以使用 ps 2 鍵盤和滑鼠。  
  
具有內部中樞的鍵盤可用來做為工作站中樞。 不過，所有其他工作站裝置都必須使用鍵盤上的埠連接到內部中樞。 如果這類鍵盤透過另一個集線器連接到電腦，該中樞將被視為中繼中樞。  
  
如果您使用的是\-分割螢幕電臺，您可能會想要考慮使用沒有數位板的迷你鍵盤，讓這兩個鍵盤可以放在監視器前方。  
  
### <a name="selecting-monitors"></a>選取監視  
除非已規劃分割\-畫面，否則每個 MultiPoint 服務站應該都提供一個監視器。 監視器會插到電腦上的視訊卡、USB 零用戶端或以 LAN\-為基礎的用戶端。 可以使用視訊卡、USB 零用戶端或 LAN\-型用戶端支援的任何類型監視，包括 CRT 監視器。  
  
某些特殊的監視包括以內部\-區域網路為基礎的用戶端或 USB 零用戶端。 這類監視器通常會包含音訊\/輸入輸出插孔，以及連接鍵盤和滑鼠的內部 USB 集線器。 它們會透過 USB 或 LAN 連線連接到伺服器。  
  
#### <a name="display-resolution"></a>顯示器解析度  
工作站的顯示區域支援的最低解析度為 512 x 768 圖元。 如果 MultiPoint 服務系統啟動併發現工作站的顯示區域小於最小解析度，則會在該工作站上顯示空白畫面，而且工作站將無法使用。  
  
如果顯示監視器將由兩個工作站共用為分割\-螢幕工作站，則顯示的最低需求為 1024 x 768，因此產生的個別工作站畫面區域至少為 512 x 768。 如需最佳的\-分割畫面使用者體驗，建議使用最少解析度為 1600 x 900 的寬螢幕。  
  
## <a name="selecting-other-station-peripheral-devices"></a>選取其他工作站週邊裝置  
MultiPoint 服務支援連接到工作站集線器、USB 零用戶端或直接連線到電腦的週邊裝置。 插入工作站中樞的裝置將會與該特定工作站相關聯。 當您直接插入電腦時，每個工作站都可以使用其他裝置。 LAN 用戶端也可以支援週邊裝置。  
  
> [!IMPORTANT]  
> 鍵盤無法連接到下游集線器\(，例如，插入到工作站中樞\)的中樞。 如果您將鍵盤插入下游集線器，任何插入\-下游中樞的週邊設備都將無法再供該站使用。 此行為可支援菊花\-鏈式站中樞。  
  
**適用于所有工作站**插到電腦\(的 USB 裝置（例如，不透過工作站集線器\) ）可供所有工作站使用。 視裝置而定，多個使用者一次可以使用，或一次只能有一位使用者存取。 下表說明如何存取 USB 裝置。  
  
> [!NOTE]  
> 資料表中的「連線到主機電腦」資料行，是指執行 MultiPoint 服務的電腦以工作站模式執行時的行為。 如果您是以主控台模式執行，插入的週邊設備的運作方式與在主控台會話中的標準伺服器相同。  
  
||已連接到主機電腦|連線到站中樞或下游中樞|  
|-|------------------------------|----------------------------------------------|  
|鍵盤|無法運作，除非它是 PS/2 站的一部分。 |適用于個別的工作站<br /><br />無法連線到下游中樞|  
|滑鼠|無法運作，除非它是 PS/2 站的一部分。 |適用于個別的工作站|  
|喇叭/耳機|無法運作，除非它是 PS/2 站的一部分。|適用于個別的工作站|  
|USB 存放裝置|適用于所有工作站|適用于個別的工作站|  
|HID 取用者控制項|無法運作|適用于個別的工作站|  
|其他 USB 裝置，例如相機、檔讀取器和 DVD 光碟機|適用于 Windows Server 2012 支援的所有工作站|適用于 Windows Server 2008 R2 支援的所有工作站遠端桌面服務|  
  
## <a name="BKMK_SelectingRDP-over-LAN-connectedstationhardware"></a>選取 RDP\-over\-LAN\-連線的工作站硬體  
任何可以使用遠端桌面通訊協定連接到遠端桌面服務的 LAN 用戶端，都可以成為 MultiPoint 服務站。  
  
如果您想要將 LAN 用戶端僅作為 MultiPoint 工作站使用，您可能會想要「鎖定」您的 LAN 用戶端。 例如，設定您的瘦用戶端，使其只能連線到 MultiPoint 服務會話，或設定桌上型電腦，以便移除桌面圖示和 [開始] 功能表項目（例如網頁瀏覽器）的存取權，以防止直接存取網際網路。 您可以使用 LAN 用戶端設定工具或群組或本機原則來進行這些設定。  
  
## <a name="selecting-audio-devices"></a>選取音訊裝置  
請務必確定當您選取 [音訊裝置] 時，可以將其插入至工作站集線器、USB 零用戶端或 LAN 用戶端。 有些 usb 集線器、usb 零用戶端和 LAN 用戶端都有類比音訊插座，可以搭配使用耳機或 earbuds \( \)等傳統類比音訊裝置。 沒有類比插座的工作站中樞可以使用 USB 音訊裝置。  
  
如果您已使用電腦主機板\/上的\-ps\- \/2 埠來設定 ps 2 direct video 連線站，以用於鍵盤和滑鼠，您必須在電腦的主機板上使用類比音訊當 MultiPoint 服務系統以站模式執行時，音訊裝置可供此工作站使用。  
  
如果您沒有 PS\/2 direct\-video\-連線的工作站，只有在 MultiPoint 服務系統以主控台模式執行時，系統主機板上的主機音訊裝置才會可用。
