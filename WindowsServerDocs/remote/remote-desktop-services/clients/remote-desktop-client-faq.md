## 設定

### 若要可以連線的電腦？

請查看[支援的設定](remote-desktop-supported-config.md)的文章，以便您可以連線到哪些電腦的相關資訊。

### 我該如何針對遠端桌面設定電腦？

我已設定好之後，我的裝置，但認為電腦準備就緒。 協助嗎？

首先，您有看到遠端桌面安裝精靈嗎？ 它會引導您讓您的電腦準備好用於遠端存取。 下載和執行該工具在您的電腦，以取得所有項目上設定。 

否則，如果您想要手動執行動作，請閱讀。

適用於 Windows 10 中，執行下列動作：

1. 在您想要連線到裝置上，開啟 [**設定**]。
2. 選取 [**系統**、**遠端桌面**。
3. 若要啟用遠端桌面使用滑桿。
4. 一般而言，最好保留清醒並搜尋以促進連線的電腦。 按一下以移至您的電腦，您可以在此變更這項設定的電源設定的 [**顯示設定**。
   > [!NOTE]
   > 您無法連接到電腦睡眠或休眠，因此請確定睡眠和休眠遠端電腦上的設定會設定為**永不**。 （不在所有電腦上可用休眠狀態）。


請記下**如何連線到此電腦**的此電腦的名稱。 您將需要此呼叫以用戶端設定。

您可以授與權限來存取此電腦-若要這樣做，請按一下 [**選取的使用者，可以從遠端存取此電腦**的特定使用者。
系統管理員群組的成員會自動有存取權。

適用於 Windows 8.1，請依照下列的指示來[連線到另一個桌面上使用遠端桌面連線](https://support.microsoft.com/en-us/help/17463/windows-7-connect-to-another-computer-remote-desktop-connection#1TC=windows-8)] 中允許遠端連線。



## 連線、 閘道和網路

### 為什麼我無法連線使用遠端桌面？

以下是一些可能的解決方案，嘗試連線到遠端的電腦時，您可能會遇到的常見問題。 如果這些解決方案無法運作，您可以找到更多協助[Microsoft Community 網站](https://go.microsoft.com/fwlink/p/?LinkId=242079)上。

- **找不到遠端電腦。** 請確定您有正確的電腦名稱，然後檢查以查看是否您正確輸入該名稱。 如果您仍然無法連線，請嘗試使用遠端電腦的 IP 位址，而不電腦名稱。
- **沒有網路問題。** 請確定您有網際網路連線。 
- **遠端桌面連接埠可能使用防火牆會封鎖。** 如果您使用的 Windows 防火牆，請依照下列步驟：

   1. 開啟 Windows 防火牆。 
   2. 按一下 [**允許應用程式或功能通過 Windows 防火牆**。 
   3. 按一下 [**變更設定**。 您可能會被要求系統管理員密碼或確認您的選擇。
   4. 在**允許應用程式與功能**，選取 [**遠端桌面**，然後點選或按一下 **[確定]**。

   如果您使用不同的防火牆，請確定已開啟遠端桌面 (通常為 3389) 連接埠。
- **遠端連線可能不是設定在遠端電腦上。** 若要修正這個問題，捲動備份到[如何設定電腦設定遠端桌面？](#how-do-i-set-up-a-pc-for-remote-desktop)本主題中的問題。
- **遠端電腦可能只允許連線至已設定的網路層級驗證的電腦。** 
- **遠端電腦可能已關閉。** 您無法連線到已關閉、 睡眠、 電腦或休眠，因此請確定設定為睡眠和休眠遠端電腦上的設定為**永不**（休眠狀態不是所有的電腦可用的。）。

### 為什麼無法找到或連線到我的電腦嗎？
請檢查以下項目：
- 是在電腦上和清醒？
- 您輸入的正確名稱或 IP 位址未嗎？

   > [!IMPORTANT]
   > 使用電腦名稱，需要您的網路來解析正確地透過 DNS 名稱。 在許多家用網路中，您必須使用而不是主機名稱的 IP 位址，來連線。
- 是不同的網路上的電腦嗎？ 您未設定，讓連線到外部電腦嗎？  如需說明，查看[允許存取您從您的網路以外的電腦](remote-desktop-allow-outside-access.md)。
- 您要連線至支援的 Windows 版本？ 

   > [!NOTE]
   > Windows XP 家用版、 Windows Media Center 版本、 Windows Vista 家用版和 Windows 7 家用版或簡易版不支援無第 3 方軟體。

### 為什麼無法我登入的遠端電腦？
如果您所見遠端電腦的登入畫面，但您無法登入，您可能不已新增至遠端桌面的使用者群組，或遠端電腦上的系統管理員權限的任何群組。 要求您若要這樣做為您的系統管理員。

### 針對公司網路支援哪些連線方法？
如果您想要存取您的 office 桌面從您的公司網路以外，您的公司必須為您提供的遠端存取的一種方法。 RD 用戶端目前支援下列內容：

- 終端機伺服器閘道或遠端桌面閘道
- 遠端桌面 Web 存取
- VPN （透過 iOS 內建的 VPN 選項）

### VPN 無法運作

VPN 問題可以有數個原因。 第一個步驟是確認 VPN 可與您的電腦或 Mac 電腦位於相同網路上運作。 如果您無法使用 Mac 或 PC 測試，您可以嘗試存取公司內部網路的網頁瀏覽器中您的裝置。

若要檢查的其他項目：
- **3g 網路封鎖或損毀 VPN。** 有數個 3g 提供者在世界中的似乎區塊或損毀 3g 流量。 確認 VPN 連線正常運作的時數分鐘。
- **L2TP 或 PPTP Vpn。** 如果您使用 L2TP 或 PPTP 您 VPN 中，請將傳送所有流量 ON VPN 設定。
- **VPN 設定錯誤。** 設定錯誤的 VPN 伺服器可以是為何 VPN 連線永遠不會互相或段時間之後停止運作的原因。 請確定測試與相同網路上 iOS 裝置的網頁瀏覽器，或在電腦或 Mac，如果發生這種情況。

### 如何測試 VPN 是否正常運作？
確認您的裝置上的已啟用 VPN。 您可以藉由在您的內部網路上移至網頁，或使用 web 服務，也就是只透過 VPN 測試您的 VPN 連線。

### 如何設定 L2TP 或 PPTP VPN 連線？
如果您使用 L2TP 或 PPTP 您 VPN 中，請務必**將所有流量都傳送**到**ON**中設定 VPN 設定。

## Web 用戶端

### 可以使用何種瀏覽器？

Web 用戶端支援 Microsoft Edge、 Internet Explorer 11，Mozilla Firefox (v55.0 及更新版本)，狩獵與 Google Chrome。

### 若要存取 web 用戶端可以使用哪些電腦？

Web 用戶端支援 Windows、 macOS、 Linux，以及 ChromeOS。 在此階段不支援行動裝置。

### 可以使用 web 用戶端在遠端桌面部署中沒有閘道嗎？

否。 用戶端需要連線到遠端桌面閘道。 不知道什麼意思？ 您的系統管理員詢問它。

### 遠端桌面 web 用戶端是否會取代遠端桌面 Web 存取頁面？

否。 遠端桌面 web 用戶端會裝載於不同比遠端桌面 Web 存取頁面的 URL。 若要在瀏覽器中檢視遠端資源，您可以使用 web 用戶端或 Web 存取 \] 頁面。

### 可以在另一個網頁中內嵌 web 用戶端？

此時不支援此功能。

### 如何使用我的監視器的所有？
若要使用兩個或多個畫面，執行下列動作：

1. 以滑鼠右鍵按一下您想要啟用多螢幕，遠端桌面，然後按一下 [**編輯**。
2. 啟用**使用所有監視器**和**全螢幕**。

音效上游 （從伺服器上，以麥克風用戶端） 不支援透過遠端桌面用戶端。

登出工作階段不只是中斷連線 (一路登出），並再次登入。

## Mac 用戶端-硬體問題
### 視網膜解析度受支援？
是的遠端桌面用戶端支援視網膜解析度。

### 我該如何啟用次要以滑鼠右鍵按一下？
為了達成的按一下滑鼠右鍵，有三個選項已開啟工作階段內使用：

- 標準 PC 兩個按鈕 USB 滑鼠
- Apple 魔術滑鼠： 若要啟用以按滑鼠右鍵，按一下 [**系統喜好設定**中擴充座中，按一下**滑鼠**，並在啟用**次要按一下**。
- Apple 魔術觸控板或 MacBook 觸控板： 啟用按一下滑鼠右鍵、 按一下 [擴充座中的**系統喜好設定**、 按一下**滑鼠**，並啟用**次要按一下**。

### AirPrint 受支援？
否，遠端桌面用戶端不支援 AirPrint。 （這是適用於 iOS 和 Mac 用戶端，則為 true）。

### 為什麼不正確的字元工作階段中顯示？
如果您使用國際的鍵盤，您可能會看到問題工作階段中顯示的字元進行比對的字元 Mac 鍵盤上輸入。

這可能會發生下列案例：

- 您正在使用的遠端工作階段不會辨識的鍵盤。 當遠端桌面無法辨識的鍵盤時，它預設會與遠端電腦上次使用的語言。
- 您在連線到遠端電腦上的先前已中斷連線的工作階段和遠端電腦，使用不同的鍵盤語言的語言比您目前嘗試使用。

您可以透過手動設定遠端工作階段的鍵盤語言來修正這個問題。 請參閱下一節中的步驟。

### 語言設定如何影響鍵盤遠端工作階段中？
有許多種 Mac 鍵盤配置。 有些是 Mac 特定的配置或自訂版面配置的這完全符合可能無法使用的是到遠端執行功能的 Windows 版本上。 遠端工作階段會將您的鍵盤對應到比對遠端電腦上的鍵盤語言的最佳。 

如果您的 Mac 鍵盤配置設為自己的金鑰應對應正確的語言鍵盤 （例如，法文 – 電腦） 和鍵盤的電腦版本應該正常運作。

如果您的 Mac 鍵盤配置設為鍵盤 （例如，為法文） 的 Mac 版本遠端工作階段會對應您電腦版本的法文語言。 您可用來在 OSX 上使用 Mac 鍵盤快速鍵的一些將無法在遠端 Windows 工作階段中運作。

如果您的鍵盤配置設為一種語言 （例如，加拿大法文） 和遠端工作階段無法將您對應到該確切的變化，遠端工作階段會將您對應到最接近的語言 （例如，為法文）。 您可用來在 OSX 上使用 Mac 鍵盤快速鍵的一些將無法在遠端 Windows 工作階段中運作。

如果您的鍵盤配置設為 [遠端工作階段無法完全相符的版面配置，將會提供您上次使用與該電腦的語言預設遠端工作階段。 在此情況下，或在情況下，您需要變更您的 Mac 鍵盤，使其符合您遠端工作階段的語言，您可以手動設定鍵盤語言的語言，是您想要使用如下所示的一個最接近的相符項目至遠端工作階段中。

使用下列指示來變更遠端桌面工作階段內的鍵盤配置：

**在 Windows 10 或 Windows 8:**

1. 從內遠端工作階段中，開啟 [地區及語言。 按一下 [**開始 > 設定 > 時間與語言**。 開啟 [**地區及語言**。
2. 新增您想要使用的語言。 然後，請關閉 [地區與語言] 視窗。
3. 現在，在遠端工作階段中，您會看到語言之間切換的功能。 （在右側的遠端工作階段，靠近時鐘。）按一下您想要切換至 （例如，**英文**) 的語言。

您可能需要關閉並重新啟動應用程式，您目前使用的鍵盤變更才會生效。


## 特定錯誤

### 為什麼會收到 「 權限不足 」 錯誤？
不允許您存取您想要連線到的工作階段。 最可能的原因是您嘗試連接到系統管理員的工作階段。 只有系統管理員可以連線到主機。 確認主控台參數已關閉在遠端桌面的進階設定。 如果這不是問題的來源，請連絡您的系統管理員取得進一步協助。

### 用戶端為什麼說是沒有 CAL？
當遠端桌面用戶端連線至遠端桌面的伺服器時，伺服器會發出遠端桌面服務用戶端存取使用權 (RDS CAL) 儲存用戶端。 每當用戶端連線一次，它會使用其 RDS CAL 和伺服器將不會發出另一個授權。 如果在裝置上的 RDS CAL 遺失或損毀，伺服器會將會發出另一個授權。 達到授權的裝置數目上限時，伺服器將不會發出新 RDS Cal。 服務以尋求協助，請連絡您的網路管理員。

### 為何我收到 「 拒絕存取 」 錯誤？
「 拒絕存取 」 錯誤是認證的產生的遠端桌面閘道並不正確的結果在嘗試連線所花的期間。 確認您的使用者名稱和密碼。 如果連線之前，且最近發生錯誤，您可能變更您的 Windows 使用者帳戶密碼，還沒有更新它還在遠端桌面設定。

### 「 RPC 錯誤 23014 」 或 「 錯誤 0x59e6 」 平均值什麼？
發生故障時回復**RPC 錯誤 23014**或**錯誤 0x59E6 再試一次在等候幾分鐘的時間後**，RD 閘道伺服器已達到作用中連線的數上限。 視 Windows 版本而定 RD 閘道上執行的連線數上限不同： Windows Server 2008 R2 Standard 的實作會限制為 250 連線數目。 Windows Server 2008 R2 Foundation 實作會限制為 50 連線的數目。 所有其他 Windows 實作可讓不限的數量的連線。

### 「 剖析 NTLM 挑戰失敗 」 錯誤是什麼意思？
此錯誤的原因是由遠端電腦上設定錯誤。 請確定 RDP 的安全性層級設定遠端電腦上設定為 「 用戶端相容 」。 （稍候會以您的系統管理員如果您需要執行此動作的說明。）

### 什麼也不會 」 TS_RAP 不允許您連線到指定的主機 」 表示？
閘道伺服器上的資源授權原則停止您的使用者名稱連線到遠端電腦時，會發生這個錯誤。 這可能會發生在下列情況：

- 遠端電腦名稱會是與閘道的名稱相同。 然後，當您嘗試連線到遠端電腦，連接到閘道相反地，這可能沒有存取權限。 如果您需要連線到閘道，請勿外部閘道名稱做為電腦名稱。 改為使用 「 localhost 」 或 IP 位址 (127.0.0.1) 或內部伺服器名稱。
- 您的使用者帳戶不是遠端存取的使用者群組的成員。
