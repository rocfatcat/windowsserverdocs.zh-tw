# 開始使用遠端桌面在 Mac 上

>適用於：Windows 10、Windows 8.1、Windows Server 2012 R2、Windows Server 2016

您可以使用遠端桌面用戶端的 Mac 可搭配 Windows 應用程式、 資源和桌上型電腦從您的 Mac 電腦。 使用下列資訊以開始使用-並查看[常見問題集](remote-desktop-client-faq.md)，如果您有問題。

>[!Note]
> - 想了解新的 macOS 用戶端的版本嗎？ 請查看[在 Mac 上的遠端桌面的新？](mac-whatsnew.md)
> - Mac 用戶端會執行 macOS 10.10 和更新版本的電腦上執行。
> - 這篇文章中的資訊主要適用於完整版的 Mac 用戶端的 Mac 應用程式 Store 中可用的版本。 藉由下載我們預覽應用程式測試新功能：[搶鮮版用戶端版本資訊](https://go.microsoft.com/fwlink/?LinkID=619698&clcid=0x409)。

## 取得遠端桌面用戶端
請依照下列步驟來開始使用遠端桌面在 Mac 上：

1. 從[Mac 應用程式市集](https://itunes.apple.com/us/app/microsoft-remote-desktop/id1295203466?mt=12)下載 Microsoft 遠端桌面用戶端。
2. [設定您的電腦以接受遠端連線](remote-desktop-client-faq.md#how-do-i-set-up-a-pc-for-remote-desktop)。 （如果您跳過此步驟中，您無法連線到您的電腦。）
3. 新增遠端桌面連線或遠端資源。 您要直接連線到 Windows 電腦和遠端資源使用 RemoteApp 程式、 工作階段為基礎的桌面，使用連線或虛擬桌面發佈在內部使用 RemoteApp 和桌面連線。 這個功能非常通常可以在企業環境中使用。

## Mac 搶鮮版用戶端呢？
我們要測試新功能上 HockeyApp 我們預覽通道上。 想要快來一探嗎？ 移至[Microsoft 遠端桌面的 Mac](https://go.microsoft.com/fwlink/?LinkID=619698) ，然後按一下 [**下載**]。 您不需要先建立帳戶或登入 HockeyApp 下載搶鮮版用戶端。

如果您已經用戶端，您可以檢查以確保您有最新版本的更新。 在搶鮮版用戶端在上方，按一下**Microsoft 遠端桌面搶鮮版**，然後按一下**檢查更新**。 

## 新增遠端桌面連線
若要建立遠端桌面連線：

1. 在 [連線中心]，按一下**+**，然後按一下 [**桌面**。
2. 輸入下列資訊：
   - **電腦名稱**-電腦的名稱。
      - 這可以是 （可在**系統**設定中找到） 的 Windows 電腦名稱、 網域名稱或 IP 位址。
      - 您也可以新增到此名稱，例如*MyDesktop:3389*結尾的連接埠資訊。
   - **使用者帳戶**-新增您用來存取遠端電腦的使用者帳戶。
      - 為 Active Directory (AD) 加入電腦或本機帳戶，可以使用其中一個這些格式：*使用者名稱*、 *domain\user_name*或*user_name@domain.com*。
      - 為 Azure Active Directory (AAD) 加入的電腦，可以使用其中一個這些格式： *AzureAD\user_name*或*Azuread\someone@contoso.onmicrosoft.com user_name@domain.com*。
      - 您也可以選擇是否要要求輸入密碼。
      - 當管理多個具有相同的使用者名稱的使用者帳戶時，設定來區分將帳戶的易記名稱。
      - 管理您的已儲存的使用者帳戶中的應用程式的喜好設定。 

3. 您也可以設定這些連線的選用設定：
   - 設定的易記名稱 
   - 新增閘道
   - 設定聲音輸出
   - 交換滑鼠按鈕
   - 啟用的管理模式
   - 將本機資料夾重新導向至遠端工作階段
   - 正向本機印表機
   - 向前智慧卡
4. 按一下 **\[儲存\]**。

若要開始連線，只要按兩下即可。 相同是適用於遠端資源，則為 true。

### 匯出和匯入連線
您可以匯出遠端桌面連線定義，並在不同的裝置上使用。 遠端桌面會儲存在不同。RDP 檔案。

1. 在 \ [連線中心中以滑鼠右鍵按一下的遠端桌面。
2. 按一下 [**匯出**。
3. 瀏覽至您要儲存的遠端桌面的位置。RDP 檔案。
4. 按一下 **\[確定\]**。

您可以使用下列步驟來匯入遠端桌面。RDP 檔案。

1. 在功能表列上，按一下**檔案 > 匯入**。
2. 瀏覽。RDP 檔案。
3. 按一下 **\[開啟\]**。

## 新增遠端資源
遠端資源是 RemoteApp 程式、 工作階段為基礎的桌面，以及發佈 RemoteApp 和桌面連線使用的虛擬桌面。

- URL 顯示連結到 RD Web 存取伺服器，讓您能夠存取 RemoteApp 和桌面連線。
- 列出已設定 RemoteApp 和桌面連線。

若要新增遠端資源：

1. 在 [連線中心，按一下**+**，然後按一下 [**新增遠端資源**。 
2. 輸入資訊的遠端資源：
   - **摘要的 URL** -RD Web 存取伺服器的 URL。 這會告訴搜尋與您的電子郵件地址相關聯的 RD Web 存取伺服器用戶端您也可以在此欄位 – 輸入您公司的電子郵件帳戶。
   - **使用者名稱**的使用者名稱，以使用您在連線到 RD Web 存取伺服器。
   - **密碼**-使用您在連線到 RD Web 存取伺服器的密碼。
3. 按一下 **\[儲存\]**。


遠端資源將會顯示在連線中心。


## 連線到 RD 閘道來存取內部的資產

遠端桌面閘道 （RD 閘道） 可讓您從任何地方連線至遠端電腦上的公司網路上網際網路。 您可以建立和管理您的應用程式，或在設定新的桌面連線時的喜好設定中的閘道。

若要設定新的閘道的喜好設定︰

1. 在 [連線中心]，按一下**喜好設定 > 閘道**。 
2. 按一下**+** 按鈕底部的表格輸入下列資訊：
  - **伺服器名稱**– 您想要為閘道使用的電腦的名稱。 這可以是 Windows 的電腦名稱、 網際網路網域名稱或 IP 位址。 您也可以新增連接埠資訊的伺服器名稱 (例如： **RDGateway:443**或**10.0.0.1:443**)。
  - **使用者名稱**-的使用者名稱和密碼，以做為您要連線的遠端桌面閘道。 您也可以選擇**使用連線認證**為適用於遠端桌面連線使用的相同的使用者名稱和密碼。


## 管理您的使用者帳戶

當您連接到電腦或遠端資源時，您可以儲存一次從選取的使用者帳戶。 您可以使用遠端桌面用戶端來管理您的使用者帳戶。

若要建立新的使用者帳戶：

1. 在 [連線中心]，按一下 [**設定** > **帳戶**。
2. 按一下 [**新增使用者帳戶**。
3. 輸入下列資訊：
   - **使用者名稱**-使用者儲存與遠端連線搭配使用的名稱。 您可以在任何下列格式輸入使用者名稱： 使用者名稱，domain\user_name，或 user_name@domain.com。
   - **密碼**-您指定的使用者的密碼。 您想要用來進行遠端連線儲存每個使用者帳戶必須有與其關聯的密碼。
   - **易記名稱**-如果您使用不同的密碼，使用相同的使用者帳戶設定來區分那些使用者帳號的易記名稱。
4. 點選 [**儲存**]，然後點選 [**設定**。

## 自訂您的顯示器解析度
您可以指定顯示器解析度，針對遠端桌面工作階段。

1. 在連線中心中，按一下 [**喜好設定**。
2. 按一下**解析度**。 
3. 按一下**+**。
4. 輸入的解析度 height 和 width，然後再按一下**確定。**

若要刪除解析度，並選取它，然後按一下**-**。

**顯示有個別的空間**如果您正在執行 Mac OS X 10.9 並停用 Mavericks**顯示有個別的空間**(**系統喜好設定 > 任務控制項**)，您需要在遠端桌面用戶端使用相同的選項中設定此設定。

### 如需遠端資源的磁碟機重新導向
磁碟機重新導向支援遠端資源，以便您可以儲存與您用在本機的遠端應用程式建立的檔案 重新導向的資料夾一律是您在遠端工作階段中顯示為網路磁碟機的主目錄。

> [!NOTE]
> 若要使用這項功能，系統管理員必須在伺服器上設定適當的設定。


## 在遠端工作階段中使用鍵盤

Mac 鍵盤配置與不同的 Windows 鍵盤配置。 

- Mac 鍵盤命令鍵等於 Windows 鍵。
- 若要執行使用命令按鈕在 Mac 的動作，您必須在 Windows 中使用的控制] 按鈕 (例如： 複製 = Ctrl + C)。
- 可以藉由按下此外 FN 鍵在工作階段中啟用功能鍵 (例如： FN + F1)。
- Alt 鍵以右側的 Mac 鍵盤上的空格鍵等於 Alt Gr/向右 Alt 鍵在 Windows 中。

根據預設，遠端工作階段會為您正在執行的用戶端的作業系統使用相同的鍵盤地區設定。 (如果您的 Mac 執行 en-我們 OS，將遠端工作階段的使用。 如果未使用的作業系統鍵盤地區設定，請檢查鍵盤在遠端電腦上設定，然後變更設定以手動方式。 如需有關鍵盤和地區設定[遠端桌面用戶端常見問題集](remote-desktop-client-faq.md)，請參閱。
