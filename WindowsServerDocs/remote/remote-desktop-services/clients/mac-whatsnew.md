# <a name="whats-new-for-the-remote-desktop-client-on-macos"></a>新功能的遠端桌面用戶端，在 macOS 上？

我們會定期更新[適用於 macOS 的遠端桌面用戶端](remote-desktop-mac.md)、 新增功能和修正問題。 查看最新的更新。

如果您遇到任何問題，您可以一律與我們連絡透過**協助 > 回報問題**。

## <a name="updates-for-version-1029"></a>更新版本，10.2.9
*發行的日期：3/6/2019*

- 在此版本中我們修正了 RD 閘道連線伺服器重新導向時可能發生的放置。
- 我們也會解決因 10.2.8 RD 閘道迴歸更新。

## <a name="updates-for-version-1028"></a>更新版本，10.2.8
*發行的日期：3/1/2019*

- 解決呈現時使用的 RD 閘道的連線問題。
- 已修正連接時所顯示的不正確的憑證警告。
- 解決某些情況下，其中的功能表列和停駐會不必要地隱藏時啟動遠端應用程式。
- 修改位址故障和有已侵犯某些使用者的當機的剪貼簿重新導向程式碼。
- 已修正的 bug 導致不必要地捲動啟動連線時 「 連線中心 」。

## <a name="updates-for-version-1027"></a>10.2.7 版本更新
*發行的日期：2/6/2019*

- 在此版本中，我們已解決圖形 mispaints （伺服器的編碼錯誤所造成） 使用 AVC444 模式時出現的內容。

## <a name="updates-for-version-1026"></a>10.2.6 版本更新
*發行的日期：1/28/2019*

- AVC （420 和 444） 可用的轉碼器，連接到目前的 Windows 10 版本時所新增的支援。
- 在 調整成視窗模式，視窗重新整理現在會之後立即發生一次調整大小以確保該內容會呈現在正確的內插補點層級。
- 修正造成重疊部分使用者摘要的標頭的配置錯誤。
- 清除應用程式喜好設定 UI。
- 完美的新增/編輯桌面 UI。
- 做了許多的調整與完成調整適用於桌上型電腦和摘要的連線中心磚和清單檢視。

>[!NOTE]
>有一個錯誤是 macOS 10.14.0 和可能會導致 「.com.microsoft.rdc.application-data_SUPPORT/_EXTERNAL_DATA"的 10.14.1 資料夾 （巢狀深度 ~/Library 資料夾內） 中耗用大量磁碟空間。 若要解決此問題，請刪除資料夾內容並升級 macos 10.14.2。 請注意，刪除資料夾內容的副作用，會刪除快照集映像指派給書籤。 重新連線到遠端電腦時，將會重新產生這些映像。

## <a name="updates-for-version-1024"></a>10.2.4 版本更新
*發行的日期：12/18/2018*

- 已新增適用於 macOS Mojave 10.14 深色模式支援。
- 如果是空的從 Microsoft 遠端桌面 8 現在匯入的選項會出現在 「 連線中心 」。
- 解決與某些協力廠商企業應用程式的資料夾重新導向相容性。
- 解決問題，其中使用者已取得的 0x30000069 遠端桌面閘道錯誤，因為安全性通訊協定後援的問題。
- 某些使用者所遭遇具有固定的漸進式呈現問題調整成視窗模式。
- 已修正的 bug 導致檔案複製和貼上複製檔案的最新版本。
- 改善滑鼠為基礎的小型捲動差異捲動。

## <a name="updates-for-version-1023"></a>10.2.3 版本更新
*發行的日期：11/06/2018*

- 遠端應用程式案例的 「 remoteapplicationcmdline"RDP 檔案設定已新增的支援。
- 工作階段 視窗的標題現在包含 RDP 檔案 （和伺服器名稱） 的名稱時啟動 RDP 檔案。
- 已修正回報的 RD 閘道的效能問題。
- 已修正回報的 RD 閘道會損毀。
- 已修正的問題，其中的連線會透過 RD 閘道連線時停止回應。
- 更妥善地處理以智慧方式隱藏功能表列和停駐的全螢幕的遠端應用程式。
- 固定的案例，其中遠端應用程式保持隱匿之後啟動。
- 使用 「 調整成視窗大小 」 停用硬體加速時，請解決緩慢呈現的更新。
- 處理用戶端啟動時，權限不正確所造成的資料庫建立錯誤。 
- 已修正的問題的用戶端以一致的方式在啟動時損毀及未啟動的某些使用者。
- 修正案例中，連線已正確匯入為 8 的遠端桌面從全螢幕。

## <a name="updates-for-version-1022"></a>如需 10.2.2 版本更新
*發行的日期：10/09/2018*

- 全新的連線中心支援拖放，手動桌面、 在清單檢視模式、 資料行的排序和比較簡單的群組管理可調整大小的資料行的排列方式。
- 「 連線中心 」 現在會記住最後一個使用中的樞紐 （桌面或摘要） 時關閉應用程式。
- 認證提示 UI 和流程已經過徹底檢查。
- RD 閘道的意見反應已連線的狀態 UI 的一部分。
- 設定匯入從第 8 版用戶端已經過改良。
- RemoteApp 端點所指向的 RDP 檔案現在可以匯入 「 連線中心 」。
- 單一監視器遠端桌面情況的 retina 顯示器最佳化。
- 指定的圖形內插補點層級 （將影響 blurriness） 支援在不使用 Retina 最佳化。
- 若要啟用連線至 Windows 2000 的 256 色支援。
- 已修正裁剪的畫面時，連線到 Windows 7，Windows Server 2008 R2 的右邊緣和下邊緣和更早版本。
- 現在將本機檔案複製到 Outlook （在 遠端工作階段中執行） 做為附件加入的檔案。
- 已修正問題： 如果源自網路共用的檔案，已減緩剪貼板為基礎的檔案傳輸。
- 解決 bug 造成停止回應時重新導向的資料夾上的檔案儲存至 Excel （在 遠端工作階段中執行）。
- 修正導致沒有報告重新導向資料夾的可用空間的問題。
- 已修正的 bug 導致會耗用太多 10.14 在 macOS 上的磁碟儲存體的縮圖。
- 已加入的支援 RD 閘道裝置重新導向原則的強制執行。
- 已修正使用 RD 閘道的連線中斷連線時，防止工作階段視窗無法關閉的問題。
- 如果網路層級驗證 (NLA) 不會強制執行伺服器，您會立即傳送到登入畫面如果您的密碼已過期。
- 已修正呈現時的效能問題在透過網路傳輸大量資料。
- 修正了智慧卡重新導向。
- 支援所有可能的值"EnableCredSspSupport 」 和 「 驗證層級 」 的 RDP 檔案設定如果 ClientSettings.EnforceCredSSPSupport 使用者中的預設金鑰 （com.microsoft.rdc.macos 網域中） 設定為 0。
- NLA 不會交涉時，才會進行 「 提示的認證在用戶端 」 RDP 檔案設定的支援。
- 透過智慧卡在 Winlogon 提示字元時不會交涉 NLA 的重新導向的智慧卡為基礎的登入的支援。
- 已修正防止下載摘要 URL 中有空格的資源的問題。
